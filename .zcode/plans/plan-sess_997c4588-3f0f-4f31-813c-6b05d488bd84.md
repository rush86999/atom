## Goal
Wire MiniMax (latest), GLM 5.2, and Kimi to full end-to-end functionality. The audit found each is partially wired ‚Äî GLM's core client is completely missing, Kimi's fallback models are wrong (Qwen instead of Kimi), and MiniMax has stale cascade/egress config. The fixes follow the same patterns already proven with OpenAI, DeepSeek, and OpenRouter.

## Three providers, organized by severity

### GLM 5.2 ‚Äî core path broken (no client ever created)
**The root cause:** `providers_config` in `byok_handler.py:_initialize_clients` has no `glm` entry, so `self.clients["glm"]` is never constructed even with `GLM_API_KEY` set. The models are registered in auxiliary configs but can't actually be called.

Changes:
1. **`byok_handler.py`** ‚Äî add `"glm": {"base_url": "https://open.bigmodel.cn/api/paas/v4"}` to `providers_config`. Env var `GLM_API_KEY` resolves automatically.
2. **`benchmarks.py`** ‚Äî add GLM models to `MODEL_QUALITY_SCORES`: glm-5.2=97, glm-5=96, glm-4.6=90, glm-4.5=88 (matching the learning router's quality scores). Without this, BPC returns quality 70 (floor) ‚Üí GLM always loses.
3. **`dynamic_pricing_fetcher.py`** ‚Äî add a GLM hardcoded pricing fallback block (mirror the MiniMax block) with `litellm_provider: "glm"`. Add `"glm"` and `"zhipu"` to `get_provider_for_model()`.
4. **`cognitive_tier_service.py`** ‚Äî add `glm`/`zhipu` branch to `_model_to_provider()`.
5. **`sandbox_egress_proxy.py`** ‚Äî add `open.bigmodel.cn` to both `_BASELINE_EGRESS_HOSTS` and `_LLM_PROVIDER_HOSTS`.

### Kimi ‚Äî routing data is wrong (sends Qwen model names to Moonshot API)
**The root cause:** `COST_EFFICIENT_MODELS["moonshot"]` maps to `qwen-3-7b`/`qwen-3-max` ‚Äî model names that will 404 against the Moonshot API.

Changes:
1. **`byok_handler.py`** ‚Äî fix `COST_EFFICIENT_MODELS["moonshot"]` to use Kimi models: `kimi-k2.6` (complex/advanced), `moonshot/kimi-k2-thinking` (simple/moderate).
2. **`benchmarks.py`** ‚Äî add Kimi K2 scores: kimi-k2.6=93, kimi-k2-thinking=91, kimi-k2=89 (matching the learning router's kimi-k2.6 quality_score=0.93).
3. **`dynamic_pricing_fetcher.py`** ‚Äî add Kimi/Moonshot hardcoded pricing fallback with `litellm_provider: "moonshot"`. Add `"kimi"` ‚Üí `"moonshot"` and `"moonshot"` to `get_provider_for_model()`.
4. **`hallucination_config.py`** ‚Äî add `kimi-k2.6` to `FRONTIER_MODELS` and `_FRONTIER_BY_PROVIDER["moonshot"] = "kimi-k2.6"`.
5. **`cache_aware_router.py`** ‚Äî add a `moonshot` entry (the learning router claims Kimi supports cache; honor it: `supports_cache: True`).
6. **`byok_endpoints.py`** ‚Äî add a `moonshot` `AIProviderConfig` to `_initialize_default_providers` (it exists in byok_routes.py but not here ‚Äî the routing engine can't resolve BYOK keys).
7. **`sandbox_egress_proxy.py`** ‚Äî add `api.moonshot.cn` to both host tuples.

### MiniMax ‚Äî stale cascade + egress (smallest delta)
The live routing path already works. Fixes:
1. **`hallucination_config.py`** ‚Äî change `_FRONTIER_BY_PROVIDER["minimax"]` from `minimax-m2.7` to `MiniMax-M3`, and add `MiniMax-M3` to `FRONTIER_MODELS`.
2. **`sandbox_egress_proxy.py`** ‚Äî fix `api.minimaxi.com` ‚Üí `api.minimax.io` (wrong domain, blocks sandboxed agents).
3. **`byok_endpoints.py`** ‚Äî add a `minimax` provider card alongside the existing `minimax_m3` one (so BYOK key lookups via `get_api_key("minimax")` succeed, since the client key is `minimax` not `minimax_m3`).
4. **`byok_routes.py`** ‚Äî update the stale HTTP card (model `minimax-2.5`, base_url `api.minimax.chat/v1` ‚Üí `MiniMax-M3`, `api.minimax.io/v1`).

## Tests ‚Äî `tests/test_provider_wiring.py`
A single test file covering all three providers with the same assertion patterns used for OpenRouter:
- GLM: client registered in `providers_config`, models in `benchmarks.py`, pricing fallback exists, cognitive tier recognizes it
- Kimi: `COST_EFFICIENT_MODELS["moonshot"]` uses Kimi models (not Qwen), K2 models have benchmark scores, hallucination cascade targets kimi-k2.6, moonshot in `byok_endpoints` defaults
- MiniMax: hallucination cascade targets M3 (not M2.7), sandbox egress uses correct domain, `minimax` in `valid_providers`
