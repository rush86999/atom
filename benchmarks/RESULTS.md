## Methodology

- Each latency benchmark runs N iterations; **P50 (median)** and **P95** are reported.
- `chat_round_trip` has no target because it is dominated by the upstream LLM
  provider's latency, which Atom does not control. It is reported for trend
  tracking across runs.
- `formula_eval_correctness` is a correctness check (not just latency): it
  verifies the workbook runtime evaluates `=SUM(A1:A2)` to `300`, not just
  stores the formula string. This exercises the LibreOffice â†’ formulas-lib â†’
  openpyxl fallback chain.
- `canvas_crud_round_trip` times a full createâ†’readâ†’updateâ†’readâ†’delete cycle
  against the CanvasAudit-backed CRUD API.
- `feedback_loop_writes` confirms the learning-router feedback path accepts
  input (the training itself happens asynchronously).

## CI integration

```bash
# Fail the build if any gated target is missed:
python benchmarks/run_benchmarks.py --strict --format json
```

The `--strict` flag exits with code 2 if any benchmark with a target misses it.
`--format json` emits machine-readable output for CI dashboards.
