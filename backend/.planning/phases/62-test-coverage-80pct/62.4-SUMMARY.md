## Problem

The last remaining collection error during full test suite:

```
ERROR collecting tests/test_excel_export_analytics.py
AttributeError: module 'numpy' has no attribute 'short'
```

## Root Cause (Discovered)

**test_cognitive_tier_e2e.py** imports **main_api_app_safe.py** which creates an incomplete mock of numpy:

```python
# main_api_app_safe.py
np_mock = mock_package("numpy")  # Empty module, no .short attribute
sys.modules["numpy.core"] = MagicMock()
sys.modules["numpy._core"] = MagicMock()
sys.modules["numpy.core.multiarray"] = MagicMock()
sys.modules["numpy._core.multiarray"] = MagicMock()
```

When `test_excel_export_analytics.py` imports `discord_analytics_engine`:
1. `discord_analytics_engine` imports `openpyxl`
2. `openpyxl/compat/numbers.py` tries to access `numpy.short`
3. The mock numpy doesn't have `.short` -> `AttributeError`

## Solution

Added module-level cleanup to `test_excel_export_analytics.py`:
1. Detects mocked/incomplete numpy (checks for missing `.short`)
2. Removes mocked module AND all submodules (`numpy.core.*`, `numpy._core.*`)
3. Removes openpyxl modules with cached numpy references
4. Forces clean reimport of real numpy

**Key insight:** Removing just `numpy` wasn't enough because mocked submodules caused import errors.
