## Root Cause Analysis

### The Problem

1. `test_webhook_bridge.py` and `test_browser_agent_ai.py` previously mocked numpy at module level
2. During full test suite collection, some file causes numpy to be in a broken state
3. When `test_excel_export_analytics.py` imports `discord_analytics_engine.py`, openpyxl tries to access `numpy.short` which fails

### Investigation Findings

- test_excel_export_analytics.py works when run alone (14 tests)
- Works with test_browser_agent_ai.py (31 tests)
- Works with many subsets of test files
- Only fails with full test suite collection

This suggests an interaction with a specific file imported late in the collection process.
