## Decisions Made

1. **Do NOT mock numpy/pandas at module level**
   - Rationale: openpyxl imports numpy and accesses numpy.short attribute, which fails when numpy is mocked as MagicMock
   - Impact: test_excel_export_analytics.py can now collect successfully when run with these test files

2. **Use pytest_collection hookwrapper**
   - Rationale: Module-level cleanup happens at import time, but pytest_collection runs before any test collection imports
   - Impact: Ensures numpy/pandas are restored before test collection begins, preventing collection errors

3. **Conditional mocking with hasattr checks**
   - Rationale: Prevents re-mocking modules that are already mocked by other test files
   - Impact: Reduces likelihood of module state corruption during test collection
