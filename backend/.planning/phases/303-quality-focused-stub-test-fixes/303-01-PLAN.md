## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test code → Production code | Test mocks and patches may not reflect real-world behavior |
| Pydantic model validation | Tests verify model validation logic, not business logic |
| Database operations | Tests use mocked sessions, not real database queries |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-303-01 | S | Test fixture mocks | mitigate | Verify mocks match production interface signatures |
| T-303-02 | T | Pydantic validation bypass | accept | Pydantic handles validation, tests verify behavior |
| T-303-03 | I | Model serialization | mitigate | Test both serialization and deserialization |
| T-303-04 | D | Test data exposure | accept | No PII in test data, synthetic workflow parameters only |
| T-303-05 | E | Exception handling in tests | mitigate | Test both valid and invalid inputs, verify validation errors |
