# Phase 297-298 patterns to follow
**Pattern 1: AsyncMock for async service methods**
- Mock async methods: `AsyncMock(return_value=...)`
- Use for: LLM calls, database operations, external API calls
- Example: `mock_llm.generate_structured_response = AsyncMock(return_value=TaskAnalysis(...))`

**Pattern 2: Patch imported modules at their import location**
- Patch where the module is imported, not where it's defined
- Use `@patch` decorator with full import path
- Example: `@patch('core.workflow_engine.LLMService')`

**Pattern 3: Create SQLAlchemy model fixtures with all required fields**
- Set up test data with all required model fields
- Use `pytest.fixture` for reusable test data
- Example: `AgentRegistry(id="...", agent_type="...", maturity_level="...", ...)`

**Pattern 4: Test both success and error handling paths**
- Success path: normal operation, expected results
- Error path: invalid inputs, service failures, edge cases
- Example: `test_analyze_task_success`, `test_analyze_task_failure`

**Pattern 5: Focus on critical business logic over 100% line coverage**
- Target 25-35% coverage per file (not 100%)
- Focus on high-impact code paths
- Skip trivial edge cases initially
