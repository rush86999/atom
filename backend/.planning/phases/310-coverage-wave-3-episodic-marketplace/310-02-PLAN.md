# Phase 310 Plan 02: Fix Patch Paths in test_episode_service.py

**Fix 18 failing tests by correcting patch paths to use import locations instead of module definitions.**

## Objective

Fix the 18 failing tests in `test_episode_service.py` by correcting `patch()` decorators to target the actual import locations rather than the module definition. The episode_service module imports `get_lancedb_handler` from `core.lancedb_service` and `EmbeddingService` from `core.embedding_service`, but tests incorrectly patch `core.episode_service.get_lancedb_handler` and `core.episode_service.get_embedding_service`.

Purpose: Correct patch paths enable proper mocking of dependencies, allowing tests to isolate and verify episode service behavior without external dependencies.

Output: 18 tests passing (pass rate increases from 47.1% to 95%+), test_episode_service.py compliant with Phase 297-298 AsyncMock patterns.

### Gap 1: Incorrect Patch Paths (18 tests failing)

**Current Issue:**
- Tests patch `core.episode_service.get_lancedb_handler` and `core.episode_service.get_embedding_service`
- episode_service.py imports from: `from core.lancedb_service import LanceDBService, get_lancedb_handler`
- episode_service.py imports from: `from core.embedding_service import EmbeddingService`

**Root Cause:**
Patch decorators target the module definition location, but Python imports create new references. The patch must target where the name is used, not where it's defined.

**Correct Patch Paths:**
- `core.lancedb_service.get_lancedb_handler` (not `core.episode_service.get_lancedb_handler`)
- `core.embedding_service.get_embedding_service` or `core.embedding_service.EmbeddingService` (not `core.episode_service.get_embedding_service`)
