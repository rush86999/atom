## Test Procedure

**Step 1: Define fuzz harness**
```python
import sys
import atheris

# Import target
from backend.api.[routes] import [function_name]

@pytest.mark.fuzzing
def test_[target]_fuzzing():
    """
    Fuzz [function_name] to discover input validation bugs.

    Target: [module].[function]
    Input type: [string/bytes/JSON]
    Iterations: 10000
    """
    def fuzz_one_input(data):
        """
        Fuzz harness called by Atheris with random byte arrays.

        Args:
            data: bytes - Random input generated by Atheris
        """
        try:
            # Convert bytes to appropriate input type
            input_data = data.decode('utf-8', errors='ignore')

            # Call target function with fuzzed input
            result = [function_name](input_data)

            # Assertions (Atheris catches crashes/assertion failures)
            assert result is not None, "Function returned None"

        except (ValueError, ValidationError, json.JSONDecodeError):
            # Expected exceptions - input validation working correctly
            pass
        except Exception as e:
            # Unexpected exception - potential bug!
            atheris.FuzzedDataProvider(data)

    # Run fuzzer
    atheris.Setup(sys.argv, fuzz_one_input)
    atheris.Fuzz()
```

**Step 2: Run fuzzing test**
```bash
# Quick smoke test (1000 iterations)
pytest backend/tests/bug_discovery/test_[target]_fuzzing.py -v -k "test_[target]_fuzzing" --fuzzing-iterations=1000

# Standard fuzzing (10000 iterations)
```

**Step 3: Monitor for crashes**

- Crashes are saved to: `backend/tests/bug_discovery/crashes/[target_name]/`
- Each crash includes: crash input, stack trace, reproducer script

## Expected Behavior

**Valid input:**
- Function returns normally with valid result
- No crashes or assertion failures
- No memory leaks

**Invalid input (expected exceptions):**
- Function raises `ValueError` for malformed input
- Function raises `ValidationError` for schema violations
- Function raises `JSONDecodeError` for invalid JSON
- **These are NOT bugs** - input validation working correctly

**Finding (bug discovered):**
- Unexpected crash (segmentation fault, bus error)
- Assertion failure
- Timeout/hang
- Memory leak (detected by valgrind/ASan)
- Any unexpected exception

**Example crash:**
```
==1234==ERROR: AddressSanitizer: heap-buffer-overflow on address 0x...
READ of size 10 at 0x...
    #0 in [function_name]
    #1 in fuzz_one_input
```

## Bug Filing

**Automatic bug filing on crash:**
```python
from tests.bug_discovery.bug_filing_service import BugFilingService
import os

def file_bug_from_crash(crash_path):
    """
    File bug for fuzzing crash with complete metadata.

    Args:
        crash_path: Path to crash input file
    """
    # Read crash input
    with open(crash_path, 'rb') as f:
        crash_data = f.read()

    # Read crash log
    crash_log_path = crash_path.replace('.input', '.log')
    with open(crash_log_path, 'r') as f:
        crash_log = f.read()

    # File bug
    BugFilingService.file_bug(
        test_name="test_[target]_fuzzing",
        error_message=f"Fuzzing crash: {crash_log[:200]}",
        metadata={
            "test_type": "fuzzing",
            "target_function": "[function_name]",
            "target_module": "backend/api/[routes]",
            "fuzz_iterations": 10000,
            "crash_input": crash_data.hex()[:1000],  # First 500 bytes
            "crash_input_path": crash_path,
            "crash_log": crash_log[:500],  # First 500 chars
            "asan_report": "AddressSanitizer report" if "AddressSanitizer" in crash_log else "Generic crash"
        },
        expected_behavior="Function should handle all inputs gracefully (raise ValueError or return normally)",
        actual_behavior=f"Function crashed on input: {crash_data.hex()[:100]}"
    )
```

## TQ Compliance

**TQ-01 (Test Independence):**
- Uses isolated `db_session` fixture (fresh database per test)
- No shared state between fuzzing runs
- Each fuzzing test is self-contained

**TQ-02 (Pass Rate):**
- Fuzzing is non-deterministic by nature (random inputs)
- Crashes are reproducible with the same crash input
- Pass rate not applicable to fuzzing (crashes = bugs)

**TQ-03 (Performance):**
- Fuzzing timeout: 300s (5 minutes) per test
- Quick smoke test: 1000 iterations (~10s)
- Standard fuzzing: 10000 iterations (~60s)
- Use `@pytest.mark.timeout(300)` to enforce limit

**TQ-04 (Determinism):**
- Same crash input always produces same crash (deterministic bugs)
- Atheris provides reproducible crash inputs
- Use `--fuzzing-seed=42` for reproducible fuzzing runs

**TQ-05 (Coverage Quality):**
- Fuzzer targets uncovered code paths (coverage-guided)
- Discovers edge cases not covered by unit tests
- Tests observable behavior (crashes), not implementation

## pytest.ini Marker

Add to `backend/pytest.ini`:
```ini
[pytest]
markers =
    fuzzing: Fuzzing tests (Atheris, slow, may crash)
```

Run only fuzzing tests:
```bash
pytest backend/tests/bug_discovery/ -v -m fuzzing
```

Skip fuzzing tests in fast CI:
```bash
pytest backend/tests/ -v -m "not fuzzing"
```
