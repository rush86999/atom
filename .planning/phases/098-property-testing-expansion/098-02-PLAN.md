    Create FastCheck property tests for API contract round-trip invariants:

    **File:** `frontend-nextjs/tests/property/api-roundtrip-invariants.test.ts`

    **Round-Trip Invariants (5-7 properties):**

    1. **Request Serialization Round-Trip:**
       - Generate arbitrary request objects with `fc.record()`
       - Serialize to JSON
       - Deserialize back to object
       - Verify all fields preserved

    2. **Response Deserialization Integrity:**
       - Generate arbitrary API response objects
       - Test JSON.parse preserves types (dates, numbers, booleans)
       - Verify array ordering preserved
       - Verify nested objects structure preserved

    3. **Error Response Round-Trip:**
       - Generate arbitrary error responses (error codes, messages, details)
       - Serialize then deserialize
       - Verify error structure preserved

    4. **Date/DateTime Field Preservation:**
       - Generate ISO 8601 date strings with `fc.date()`
       - Verify round-trip preserves timezone info
       - Verify milliseconds preserved

    5. **Numeric Field Precision:**
       - Generate numbers with various precision (integers, floats, decimals)
       - Verify JSON round-trip doesn't lose precision
       - Test edge cases: Infinity, NaN, very large numbers

    Use `fc.record()` for complex object generation:
    ```typescript
    const apiRequestGenerator = fc.record({
      id: fc.uuid(),
      method: fc.constantFrom('GET', 'POST', 'PUT', 'DELETE'),
      url: fc.string(),
      headers: fc.record({
        'Content-Type': fc.constantFrom('application/json'),
        'Authorization': fc.option(fc.string()),
      }),
      body: fc.option(fc.object()),
    });

    fc.assert(
      fc.property(apiRequestGenerator, (request) => {
        const serialized = JSON.stringify(request);
        const deserialized = JSON.parse(serialized);
        // Verify round-trip
        expect(deserialized.id).toBe(request.id);
        expect(deserialized.method).toBe(request.method);
      }),
      { numRuns: 100, seed: 23002 }
    );
    ```

    Include VALIDATED_BUG docstrings documenting edge cases found:
    ```typescript
    /**
     * INVARIANT: API request serialization round-trip preserves all fields
     *
     * VALIDATED_BUG: JSON.stringify() converts undefined to null
     * Root cause: JSON spec doesn't support undefined
     * Mitigation: Frontend code treats null and undefined equivalently for API calls
     * Scenario: { field: undefined } -> JSON -> { field: null }
     */
    ```
