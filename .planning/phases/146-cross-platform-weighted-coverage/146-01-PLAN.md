<objective>
Create cross-platform coverage enforcement script with platform-specific minimum thresholds and weighted overall calculation.

Purpose: Enforce different coverage minimums per platform (backend >=70%, frontend >=80%, mobile >=50%, desktop >=40%) while computing an overall weighted score for CI/CD quality gates. This prevents low-coverage platforms from hiding behind high-coverage averages.

Output: cross_platform_coverage_gate.py script that loads all 4 platform coverage reports, enforces per-platform thresholds, computes weighted overall, and provides structured JSON output for CI/CD integration.
</objective>

  <action>
Create cross_platform_coverage_gate.py with the following structure:

1. **Imports and constants** (lines 1-50):
   - argparse, json, sys, logging, pathlib.Path, typing.Dict/Tuple/Optional
   - Platform-specific thresholds:
     ```python
     PLATFORM_THRESHOLDS = {
         "backend": 70.0,
         "frontend": 80.0,
         "mobile": 50.0,
         "desktop": 40.0
     }
     ```
   - Platform weights (from RESEARCH.md recommendation):
     ```python
     PLATFORM_WEIGHTS = {
         "backend": 0.35,
         "frontend": 0.40,
         "mobile": 0.15,
         "desktop": 0.10
     }
     ```
   - Default file paths relative to script location

2. **Coverage loading functions** (lines 51-200):
   - load_backend_coverage(path): Read pytest coverage.json, extract totals.percent_covered
   - load_frontend_coverage(path): Read Jest coverage-final.json, aggregate statements from "s" field
   - load_mobile_coverage(path): Read jest-expo coverage-final.json (same format as Jest)
   - load_desktop_coverage(path): Read tarpaulin coverage.json, aggregate from files[].stats

   Each function must:
   - Return 0.0 with warning log if file not found (not failure)
   - Handle JSON decode errors gracefully
   - Extract covered/total counts for percentage calculation
   - Return dict with: coverage_pct, covered, total, file_path, error (if applicable)

3. **Platform threshold enforcement** (lines 201-280):
   - check_platform_thresholds(coverage_data: Dict[str, float]) -> Tuple[bool, List[str]]
   - Check each platform against PLATFORM_THRESHOLDS
   - Return (all_passed, list_of_failure_messages)
   - Failure message format: "{platform}: {coverage:.2f}% < {threshold:.2f}% (gap: {gap:.2f}%)"

4. **Weighted coverage calculation** (lines 281-350):
   - compute_weighted_coverage(coverage_data: Dict[str, Dict]) -> Dict
   - Multiply each platform coverage by its weight
   - Sum weighted values for overall score
   - Validate weights sum to 1.0 (0.99-1.01 tolerance)
   - Return dict with overall_pct, platform_breakdown, validation_status

5. **CLI interface** (lines 351-450):
   - argparse with:
     - --backend-coverage (default: relative path to pytest coverage.json)
     - --frontend-coverage (default: relative path to Jest coverage-final.json)
     - --mobile-coverage (default: relative path to jest-expo coverage-final.json)
     - --desktop-coverage (default: relative path to tarpaulin coverage.json)
     - --weights (override default weights, comma-separated)
     - --thresholds (override default thresholds, comma-separated key=value pairs)
     - --output-json (path for JSON output)
     - --format (text|json|markdown)
     - --strict (exit 1 if any platform below threshold)

6. **Report generation** (lines 451-550):
   - generate_text_report(data): Human-readable console output
   - generate_json_report(data): Machine-readable JSON for CI/CD
   - generate_markdown_report(data): PR comment format

7. **Main execution** (lines 551-650):
   - Parse args
   - Load all platform coverages
   - Check thresholds
   - Compute weighted overall
   - Generate report
   - Exit 1 if strict mode and any threshold failed
   - Save cross_platform_summary.json

Key implementation notes from research:
- Use Path(__file__).parent.parent.parent for portable path resolution
- Validate weights sum to 1.0 with tolerance for floating point errors
- Log warnings for missing files, don't fail
- Enforce line coverage (not branch) for threshold checking per REQUIREMENTS.md
- Include error field in coverage dicts for graceful degradation
