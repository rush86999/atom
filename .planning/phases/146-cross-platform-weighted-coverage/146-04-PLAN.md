Create docs/CROSS_PLATFORM_COVERAGE.md with the following sections:

1. **Overview** (lines 1-50):
   - Purpose: Enforce platform-specific coverage minimums across backend, frontend, mobile, desktop
   - Problem: Simple averaging masks platform-specific gaps
   - Solution: Per-platform thresholds + weighted overall score
   - Benefits: Prevents low-coverage platforms from hiding, ensures balanced quality

2. **Quick Start** (lines 51-150):
   - Local execution:
     ```bash
     # Run all platform tests
     cd backend && pytest --cov=core --cov=api --cov=tools --cov-report=json
     cd ../frontend-nextjs && npm test -- --coverage
     cd ../mobile && npm test -- --coverage
     cd ../frontend-nextjs/src-tauri && cargo tarpaulin --out Json

     # Generate cross-platform report
     python backend/tests/scripts/cross_platform_coverage_gate.py \
       --backend-coverage backend/tests/coverage_reports/metrics/coverage.json \
       --frontend-coverage frontend-nextjs/coverage/coverage-final.json \
       --mobile-coverage mobile/coverage/coverage-final.json \
       --desktop-coverage frontend-nextjs/src-tauri/coverage-report/coverage.json \
       --format text
     ```
   - CI/CD: Automatic on push/PR, see .github/workflows/cross-platform-coverage.yml

3. **Architecture** (lines 151-250):
   - Components:
     - cross_platform_coverage_gate.py: Main enforcement script
     - update_cross_platform_trending.py: Historical trend tracking
     - cross-platform-coverage.yml: CI/CD orchestration
   - Data flow:
     1. Platform tests generate coverage reports (pytest JSON, Jest JSON, tarpaulin JSON)
     2. CI uploads artifacts with coverage files
     3. Aggregate job downloads all artifacts
     4. cross_platform_coverage_gate.py loads and validates all formats
     5. Platform thresholds enforced, weighted overall computed
     6. PR comment posted with platform breakdown
     7. Trend data updated for historical tracking

4. **Platform Thresholds** (lines 251-350):
   - Backend: >=70% (Python services, mature codebase, high complexity)
   - Frontend: >=80% (React/Next.js, rapid iteration, user-facing)
   - Mobile: >=50% (React Native, newer codebase, platform-specific challenges)
   - Desktop: >=40% (Tauri/Rust, niche platform, hardware-dependent features)
   - Rationale for differences:
     - Backend has lower target (70%) due to large codebase, complex async logic
     - Frontend has highest target (80%) due to UI criticality and testability
     - Mobile has moderate target (50%) due to device testing challenges
     - Desktop has lowest target (40%) due to platform-specific code and hardware dependencies

5. **Weight Distribution** (lines 351-450):
   - Formula: Overall = (backend * 0.35) + (frontend * 0.40) + (mobile * 0.15) + (desktop * 0.10)
   - Rationale based on business impact:
     - Frontend: 40% (user-facing, highest impact on UX)
     - Backend: 35% (core logic, data integrity)
     - Mobile: 15% (important but smaller user base)
     - Desktop: 10% (niche platform, supplementary experience)
   - Weight validation: Must sum to 1.0 (0.99-1.01 tolerance)
   - Customization: --weights flag allows override for specific contexts

6. **Coverage File Formats** (lines 451-550):
   - Backend (pytest coverage.json):
     ```json
     {
       "totals": {
         "percent_covered": 75.0,
         "covered_lines": 1500,
         "num_statements": 2000
       }
     }
     ```
   - Frontend/Mobile (Jest coverage-final.json):
     ```json
     {
       "src/file.ts": {
         "s": {"1": 10, "2": 5},
         "b": {"1": [10, 5]},
         "f": {"1": 10}
       }
     }
     ```
   - Desktop (tarpaulin coverage.json):
     ```json
     {
       "files": {
         "src/main.rs": {
           "stats": {
             "covered": 50,
             "coverable": 100,
             "percent": 50.0
           }
         }
       }
     }
     ```

7. **CLI Reference** (lines 551-650):
   - Script: cross_platform_coverage_gate.py
   - Arguments:
     - --backend-coverage: Path to pytest coverage.json
     - --frontend-coverage: Path to Jest coverage-final.json
     - --mobile-coverage: Path to jest-expo coverage-final.json
     - --desktop-coverage: Path to tarpaulin coverage.json
     - --weights: Override default weights (comma-separated)
     - --thresholds: Override default thresholds (key=value pairs)
     - --format: Output format (text|json|markdown|pr-comment)
     - --output-json: Save JSON output to file
     - --strict: Exit 1 if any platform below threshold
   - Exit codes: 0 (success), 1 (threshold failure), 2 (execution error)

8. **Troubleshooting** (lines 651-750):
   - Missing coverage files:
     - Symptom: "WARNING: frontend coverage file not found"
     - Cause: Platform tests didn't run or failed
     - Solution: Check platform-specific job logs, ensure tests pass
   - Threshold failures:
     - Symptom: "Backend: 65.00% < 70.00% (gap: 5.00%)"
     - Cause: Coverage below minimum threshold
     - Solution: Add tests for uncovered code, see remediation steps in PR comment
   - Weight validation errors:
     - Symptom: "ERROR: Weights must sum to 1.0"
     - Cause: Custom weights don't sum correctly
     - Solution: Adjust weights to sum to 1.0 (0.99-1.01 tolerance)
   - JSON parse errors:
     - Symptom: "ERROR: Invalid backend coverage schema"
     - Cause: Coverage file corrupted or wrong format
     - Solution: Re-run platform tests to regenerate coverage
   - CI/CD integration issues:
     - Symptom: Artifacts not downloaded
     - Cause: Artifact upload failed or paths don't match
     - Solution: Verify artifact names, check download patterns

9. **CI/CD Integration** (lines 751-850):
   - Workflow: .github/workflows/cross-platform-coverage.yml
   - Jobs: backend-tests, frontend-tests, mobile-tests, desktop-tests, aggregate-coverage
   - Triggers: push to main/develop, pull_request, workflow_dispatch
   - Artifact retention: 30 days
   - PR comments: Automatic platform breakdown with trend indicators
   - Main branch enforcement: Build fails if any platform below threshold
   - PR enforcement: Warning only (allows development flexibility)

10. **Trend Tracking** (lines 851-900):
    - Script: update_cross_platform_trending.py
    - Storage: cross_platform_trend.json
    - Metrics: Per-platform coverage over time, trend deltas
    - Indicators: ↑ (upward >1%), ↓ (downward <-1%), → (stable)
    - Retention: 30 days of historical data
    - Usage: Automatic in CI, manual via --summary flag

11. **Best Practices** (lines 901-950):
    - Run locally before pushing to catch coverage gaps early
    - Focus on high-impact files first (governance, episodic memory, LLM services)
    - Use platform-specific test strategies (unit tests for backend, component tests for frontend)
    - Monitor trends to detect regressions early
    - Adjust thresholds as codebase matures (incremental increases)

12. **Related Documentation** (lines 951-1000):
    - FRONTEND_COVERAGE.md: Frontend-specific coverage strategies
    - DESKTOP_COVERAGE.md: Desktop Tauri coverage patterns
    - ROADMAP.md: Phase 146 completion status
    - RESEARCH.md: Technical research and rationale
