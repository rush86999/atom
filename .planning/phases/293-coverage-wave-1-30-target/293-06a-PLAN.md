# Plan 293-06a: Frontend Coverage Push Group A (4 Integrations + 2 Lib Files)

## Objective

Test 4 High-tier integration components (Group A: Monday, GoogleWorkspace, Slack, Notion) and extend 2 lib utility test files (integrationUtils, apiClient) to increase frontend coverage from 17.77% toward the 30% target.

**Purpose:** The 293-03 plan achieved only +2.63pp (17.77% total). Analysis shows 3,206 additional lines needed to reach 30%. At ~20 lines per test, ~160 tests required. This plan (Group A) adds ~80 tests across 4 integration components and 2 lib utility files. Group B (293-06b) will add the remaining ~80 tests.

**Calculated Test Count for Group A:**
- Current coverage: 17.77% (4,671 of 26,275 lines)
- Target coverage: 30% = 7,882 lines
- Gap: 3,206 lines needed
- Average coverage per test: ~20 lines
- Tests needed: 3,206 / 20 = ~160 tests total
- Group A portion: ~80 tests (40-56 integration + 30-50 lib)
- Group B portion (293-06b): ~80 tests (remaining integrations + lib files)

**Output:** 4 new/extended integration test files (~40-56 tests), 2 extended lib test files (~30-50 tests), updated combined coverage measurement showing progress toward 30% target.

**Dependency on 293-05:** This plan depends on 293-05 (Frontend Async Timeout Fixes) because the timeout fixes are required for the new integration tests to pass reliably. Without the 10-second timeout configuration and proper waitFor handling, the new integration tests will fail with timeout errors.
