    ```markdown
    # Frontend Test Fix Patterns - Phase 291

    This guide documents common test failure patterns and their fixes for the Atom frontend test suite.

    ## Pattern Catalog

    ### 1. MSW Network Error (Node.js Incompatibility)

    **Symptom**: Tests fail with "Network Error" or MSW throws errors in Node.js environment
    **Root Cause**: `res.networkError()` doesn't work in Node.js/jsdom (MSW 1.x limitation)
    **Fix**: Use HTTP status codes instead

    ```typescript
    // WRONG (broken in Node.js):
    server.use(
      rest.post('/api/test', (req, res) => {
        return res.networkError('Connection failed');
      })
    );

    // CORRECT (works in Node.js):
    server.use(
      rest.post('/api/test', (req, res, ctx) => {
        return res(
          ctx.status(503),
          ctx.json({ error: 'Service unavailable', code: 'SERVICE_UNAVAILABLE' })
        );
      })
    );
    ```

    **Files Affected**: All test files using MSW handlers
    **Tests Fixed**: 300-500 tests

    ### 2. Axios Retry Cascading Failures

    **Symptom**: Tests show "Retry attempt 2 of 3" repeated many times, timeouts
    **Root Cause**: Axios retry logic (via @lifeomic/attempt) retries failed requests
    **Fix**: Disable retries in tests or mock apiClient

    ```typescript
    // Option A: Disable retry per request
    await apiClient.get('/api/test', { retry: false });

    // Option B: Mock apiClient
    jest.mock('@/lib/api', () => ({
      apiClient: {
        get: jest.fn(() => Promise.resolve({ data: {} })),
        post: jest.fn(() => Promise.resolve({ data: {} })),
      },
    }));

    // Option C: Suppress console output
    beforeEach(() => {
      jest.spyOn(console, 'log').mockImplementation(() => {});
    });
    ```

    **Files Affected**: lib/__tests__/api/*.test.ts
    **Tests Fixed**: 100-200 tests

    ### 3. Error Response Shape Mismatches

    **Symptom**: Tests expect `error.response?.status` but receive `undefined`
    **Root Cause**: Network errors don't have response objects, only API errors do
    **Fix**: Check error type before asserting

    ```typescript
    // WRONG (fails for network errors):
    expect(error.response?.status).toBe(404);

    // CORRECT (handles both error types):
    if (error.code === 'ERR_NETWORK' || error.code === 'SERVICE_UNAVAILABLE') {
      expect(error.message).toMatch(/network|service unavailable/i);
    } else {
      expect(error.response?.status).toBe(404);
    }
    ```

    **Error Codes**:
    - `ERR_NETWORK`: Network error (no response)
    - `SERVICE_UNAVAILABLE`: 503 response
    - `ECONNABORTED`: Timeout
    - `error.response?.status`: HTTP status code (404, 500, etc.)

    **Files Affected**: All API error handling tests
    **Tests Fixed**: 200-300 tests

    ### 4. Import Path Errors

    **Symptom**: "Cannot find module '@/core/models/agent'" or similar
    **Root Cause**: Pydantic v2 migration changed type locations
    **Fix**: Update to new import paths

    ```typescript
    // OLD (broken):
    import { AgentDTO } from '@/core/models/agent';
    import { IntegrationConfig } from '@/api/generated';

    // NEW (working):
    import { AgentDTO, IntegrationConfig } from '@/types/api-generated';
    ```

    **Files Affected**: Integration component tests
    **Tests Fixed**: 300+ tests

    ### 5. Test Isolation Issues

    **Symptom**: Tests pass in isolation but fail when run together
    **Root Cause**: State leaks between tests (MSW handlers, mocks, localStorage)
    **Fix**: Add proper cleanup

    ```typescript
    // Add to each test file:
    beforeEach(() => {
      jest.clearAllMocks();
      server.use(...defaultHandlers);
    });

    afterEach(() => {
      server.resetHandlers();
      jest.restoreAllMocks();
    });
    ```

    **Files Affected**: Tests using MSW or shared state
    **Tests Fixed**: 100-200 tests

    ## Quick Reference

    | Symptom | Pattern | Fix |
    |---------|---------|-----|
    | "Network Error" | MSW networkError | Use ctx.status(503) |
    | "Retry attempt" logged | Axios retry | Disable with { retry: false } |
    | error.response is undefined | Error shape | Check error.code first |
    | "Cannot find module" | Import path | Use @/types/api-generated |
    | Tests fail together | Isolation | Add beforeEach/afterEach |

    ## Before Writing New Tests

    1. **MSW**: Always use `ctx.status()` for errors, never `res.networkError()`
    2. **API Client**: Disable retries with `{ retry: false }` or mock apiClient
    3. **Error Handling**: Check `error.code` for network errors, `error.response` for API errors
    4. **Imports**: Use `@/types/api-generated` for DTOs
    5. **Isolation**: Always add `beforeEach` cleanup
    ```
