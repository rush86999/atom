**Test Groups (600+ lines):**

1. **Required Field Edge Cases** (10 tests, 150 lines)
   - Empty string: `''` should fail
   - Whitespace only: `'   '` should fail
   - Null value: `null` should fail
   - Undefined: `undefined` should fail
   - Zero: `0` should PASS (not empty)
   - False: `false` should PASS (checkbox case)
   - Single space: `' '` should fail
   - Tab character: `'\t'` should fail
   - Newline only: `'\n'` should fail
   - Mixed whitespace: `'  \t\n  '` should fail

2. **Boundary Value Tests for Numbers** (12 tests, 200 lines)
   - Min-1: should fail
   - Min: should pass
   - Min+1: should pass
   - Max-1: should pass
   - Max: should pass
   - Max+1: should fail
   - Negative numbers when min=0
   - Decimal values at boundaries (e.g., 17.999 vs 18.0)
   - Very large numbers (Number.MAX_SAFE_INTEGER)
   - Very small decimals
   - NaN: should fail
   - Infinity: should fail

3. **Boundary Value Tests for String Length** (10 tests, 150 lines)
   - Empty string for min=1
   - At min length exactly
   - Below min by 1
   - At max length exactly
   - Above max by 1
   - Unicode character boundaries (emoji counts as 1+ chars)
   - Multibyte characters
   - Very long strings (10000+ chars)

4. **Format Validation Edge Cases** (8 tests, 100 lines)
   - Email with trailing dot: `'test@example.com.'`
   - Email with multiple @: `'test@@example.com'`
   - Email with IP address domain: `'test@127.0.0.1'`
   - URL without protocol: `'example.com'`
   - URL with fragment: `'https://example.com#section'`
   - Phone with extensions: `'123-456-7890 ext 123'`
   - Pattern with special regex chars escaping

5. **Type Coercion Edge Cases** (6 tests, 80 lines)
   - String number input to number field: `'123'` should parse to 123
   - Empty string to number field: should parse to NaN or 0
   - Boolean checkbox true/false values
   - Select field with empty string value

Create `frontend-nextjs/lib/__tests__/validation-edge-cases.test.ts` with comprehensive edge cases for validation utilities:

**Test Structure:**
```typescript
import {
  validateEmail,
  validateRequired,
  validateLength,
  validateUrl,
  validatePhone,
  validatePasswordStrength,
  validateRange,
  validatePattern
} from '../validation';
```

**Test Groups (400+ lines):**

1. **validateEmail Edge Cases** (15 tests, 120 lines)
   - Valid: Internationalized domain names (IDN)
   - Valid: Subdomain with multiple levels
   - Valid: Plus addressing (user+tag@domain.com)
   - Invalid: Leading/trailing dots
   - Invalid: Consecutive dots
   - Invalid: Missing TLD
   - Edge: Very long email (1000 chars)
   - Edge: Email with Unicode characters
   - Edge: Email with quoted local part
   - Edge: IP address literal format

2. **validateRequired Edge Cases** (10 tests, 80 lines)
   - Empty array `[]` should fail
   - Empty object `{}` should pass (truthy)
   - Negative zero `-0` should pass
   - Empty string after trim
   - Whitespace variations
   - Null/undefined handling
   - Boolean false should pass
   - Number 0 should pass

3. **validateLength Edge Cases** (12 tests, 100 lines)
   - Unicode string length (emoji = 2 chars but 1 grapheme)
   - Surrogate pairs
   - Combining characters
   - Zero-width characters
   - Min/max with same value
   - Min greater than max (should always fail)
   - Negative length values (should handle gracefully)

4. **validateUrl Edge Cases** (10 tests, 80 lines)
   - Valid with port number
   - Valid with auth (user:pass@host)
   - Valid with query params
   - Invalid with missing protocol
   - Invalid with invalid protocol
   - Edge: javascript: URL (security - should reject)
   - Edge: data: URL
   - Edge: Very long URL (2000+ chars)

5. **validatePhone Edge Cases** (8 tests, 70 lines)
   - International formats with + prefix
   - Formats with extensions
   - Empty string after stripping non-digits
   - Mixed format separators (dots, dashes, spaces)
   - Less than 10 digits (US minimum)
   - More than 15 digits (unlikely valid)

6. **validatePasswordStrength Edge Cases** (8 tests, 70 lines)
   - Exactly 8 chars meeting all rules
   - 7 chars with all rules (should fail length)
   - Uppercase + numbers (no lowercase)
   - Lowercase + numbers (no uppercase)
   - Only special chars (no letters)
   - Common passwords (password123) should fail
   - Repeated characters (aaaaaaaa1)

7. **validateRange Edge Cases** (10 tests, 80 lines)
   - Exactly at min boundary
   - Exactly at max boundary
   - NaN handling
   - Infinity handling
   - Negative numbers in positive range
   - Floating point precision (0.1 + 0.2 issues)
   - Min > max (invalid config)
   - Negative range values

8. **validatePattern Edge Cases** (8 tests, 70 lines)
   - Empty string against pattern
   - Pattern with special regex chars
   - Case-sensitive vs insensitive
   - Unicode matching
   - Null/undefined handling
   - Invalid regex object (should throw or handle)

**Pattern:** Use expect().toBe(true/false) for boolean return values, test actual behavior not implementation
