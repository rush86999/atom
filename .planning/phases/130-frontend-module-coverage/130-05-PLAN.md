<task type="auto">
  <name>Task 1: Create coverage enforcement script</name>
  <files>frontend-nextjs/scripts/check-module-coverage.js</files>
  <action>
Create `frontend-nextjs/scripts/check-module-coverage.js` that:

1. **Reads coverage data** from coverage/coverage-summary.json

2. **Loads thresholds** from coverage/thresholds.json (create if missing with defaults)

3. **Validates each module** against its threshold:
   - Global threshold: 80% (default for all modules)
   - Directory overrides: e.g., components/ 60%, lib/ 80%
   - Module-specific overrides: e.g., specific file 50%

4. **Reports violations**:
   - List modules below threshold
   - Show current vs target coverage
   - Calculate lines needed to reach threshold

5. **Exit codes**:
   - 0: All modules meet thresholds
   - 1: One or more modules below threshold
   - 2: Coverage data missing

6. **Options**:
   - `--threshold`: Override global threshold (e.g., --threshold 70)
   - `--format`: Output format (text, json, markdown)
   - `--fail-only`: Only output failing modules
   - `--fix-me`: Generate test file stubs for failing modules

Example output:
```
Checking module coverage against 80% threshold...

FAILING MODULES (12):
components/Dashboard.tsx: 0% (target: 80%, need: 51 lines)
components/AsanaIntegration.tsx: 0% (target: 80%, need: 114 lines)
lib/api.ts: 38.54% (target: 80%, need: 41 lines)
...

PASSING MODULES (45):
lib/validation.ts: 100%
hooks/useWebSocket.ts: 100%
...

Exit: 1
```

Script should be pure Node.js, handle missing files gracefully, and support all options.

<task type="auto">
  <name>Task 2: Create threshold configuration</name>
  <files>frontend-nextjs/coverage/thresholds.json</files>
  <action>
Create `frontend-nextjs/coverage/thresholds.json` with:

1. **Global default threshold** (80%)

2. **Directory-level overrides**:
   - pages/ : 40% (API routes harder to test, focus on components)
   - components/ : 80% (main target)
   - lib/ : 80% (utilities should be well-tested)
   - hooks/ : 70% (hooks can be tricky to test)

3. **Module-specific overrides** (for exceptional cases):
   - Complex integration components: 60%
   - Legacy code: 50%
   - Third-party wrappers: 40%

Format:
```json
{
  "global": 80,
  "directories": {
    "pages/": 40,
    "components/": 80,
    "lib/": 80,
    "hooks/": 70
  },
  "modules": {
    "components/integrations/AsanaIntegration.tsx": 60,
    "components/integrations/JiraIntegration.tsx": 60
  },
  "exclude": [
    "*.test.tsx",
    "*.test.ts",
    "*.spec.tsx",
    "*.spec.ts",
    "__tests__/**"
  ]
}
```

Include comments explaining threshold rationale.
