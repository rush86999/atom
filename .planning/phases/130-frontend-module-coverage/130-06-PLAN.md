Create `.github/workflows/frontend-coverage.yml` that:

1. **Triggers**: Run on pull_request to main branch

2. **Job steps**:
   - Checkout code
   - Setup Node.js (v20)
   - Install dependencies (npm ci)
   - Run tests with coverage (npm run test:ci)
   - Run coverage enforcement (npm run check:coverage)
   - Upload coverage report as artifact
   - Comment on PR with coverage results

3. **Coverage trend tracking**:
   - Store coverage metrics in GitHub Actions cache
   - Compare with previous run
   - Show trend (+/- percentage points)

4. **PR comment format**:
```markdown
## Frontend Coverage Report

**Overall Coverage**: X.XX% (lines)
**Change**: +Y.YY% from baseline

### Modules Below Threshold (80%):
- components/Dashboard.tsx: 0% (need: 51 lines)
- components/AsanaIntegration.tsx: 0% (need: 114 lines)
...

### Modules Above Threshold:
- lib/validation.ts: 100%
- hooks/useWebSocket.ts: 100%
...
```

5. **Optional**: Fail build if coverage regresses significantly

Reference existing ci.yml workflow for patterns. Use actions/checkout, actions/setup-node, actions/upload-artifact.
