  <files>frontend-nextjs/scripts/coverage-gap-analysis.js</files>
  <action>
Create `frontend-nextjs/scripts/coverage-gap-analysis.js` that:

1. **Reads baseline data** from coverage/baseline-130.json (created in Plan 01)

2. **Categorizes modules** by business value:
   - **Tier 1 - Critical**: Integration components (Asana, Jira, Slack, Notion, GitHub)
   - **Tier 2 - High**: Dashboard, CommunicationHub, CalendarManagement, TaskManagement
   - **Tier 3 - Medium**: Settings pages, Debugging components, Collaboration features
   - **Tier 4 - Low**: UI primitives (button, input - already have good coverage)

3. **Calculates coverage gap** for each module:
   - gap = (80 - current_coverage) * line_count / 100
   - This estimates lines of test code needed to reach 80%

4. **Prioritizes** by: (business_tier * 100) + line_count
   - Higher tier modules get priority
   - Within tier, larger files get priority (more impact)

5. **Groups** for test planning:
   - **Group A**: Top 10 integration components (Plans 03)
   - **Group B**: Dashboard + core features (Plan 04)
   - **Group C**: Settings pages (future phase)
   - **Group D**: Low-priority UI components (future phase)

6. **Outputs**:
   - test-priorities.json with prioritized modules
   - Console report with grouping and estimated effort
   - Coverage gap totals by tier

Script should be pure Node.js with clear output formatting.
