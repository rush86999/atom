<task type="auto">
  <name>Task 2: Create new code coverage enforcement script</name>
  <files>backend/tests/scripts/new_code_coverage_gate.py</files>
  <action>
Create backend/tests/scripts/new_code_coverage_gate.py with:

1. New file detection (get_new_files function):
   - Run: git diff --name-only --diff-filter=A origin/main...HEAD
   - Filter for Python files (*.py)
   - Skip test files (exclude paths containing 'test' or 'tests')
   - Return list of new file paths

2. Coverage checking (check_new_file_coverage function):
   - Load coverage.json from pytest-cov
   - For each new Python file:
     - Extract coverage from coverage_data["files"][file_path]["summary"]["percent_covered"]
     - If coverage < 80.0: Print ERROR and fail
     - If coverage >= 80.0: Print PASS
     - If file not in coverage data: Print WARNING and fail (0% coverage)
   - Return True if all files pass, False otherwise

3. CLI interface:
   ```bash
   python new_code_coverage_gate.py --coverage-file coverage.json --base-branch origin/main
   ```

4. Error handling:
   - No new files → exit 0 with "✅ No new files detected"
   - Coverage file not found → exit 2 with error
   - New files below 80% → exit 1 with list of failing files
   - All files pass → exit 0

Reference RESEARCH.md Example 5 for complete implementation. Use subprocess.check=True for git command to handle errors gracefully.
  </action>
