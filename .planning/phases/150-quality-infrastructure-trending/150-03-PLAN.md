<task type="auto">
  <name>Create CI/CD trending workflow</name>
  <files>.github/workflows/coverage-trending.yml</files>
  <action>
Create .github/workflows/coverage-trending.yml (150+ lines) with the following structure:

1. **Workflow triggers**
   - on:
     - push: branches: [main, develop]
     - pull_request: branches: [main, develop]
     - workflow_dispatch: (manual trigger for on-demand trending)

2. **Permissions**
   - permissions:
     - contents: read
     - actions: write (for job summaries)

3. **Job: trending-analysis**
   - name: Coverage Trending Analysis
   - runs-on: ubuntu-latest
   - needs: [test-platform] (from unified-tests-parallel.yml)
   - if: always() (run even if tests fail to track coverage drops)

   **Steps:**

   a. Checkout code
      - uses: actions/checkout@v4

   b. Setup Python
      - uses: actions/setup-python@v5
      - with: python-version: '3.11'

   c. Download all coverage artifacts
      - Pattern: 4 download steps for backend/frontend/mobile/desktop
      - uses: actions/download-artifact@v4
      - continue-on-error: true (allow partial results)

   d. Run cross-platform coverage gate
      - working-directory: ./backend
      - run: |
          python tests/scripts/cross_platform_coverage_gate.py \
            --backend-coverage ../coverage/backend/coverage.json \
            --frontend-coverage ../coverage/frontend/coverage-final.json \
            --mobile-coverage ../coverage/mobile/coverage-final.json \
            --desktop-coverage ../coverage/desktop/coverage.json \
            --format json \
            --output-json tests/coverage_reports/metrics/cross_platform_summary.json

   e. Update trending data with commit tracking
      - env:
          COMMIT_SHA: ${{ github.sha }}
          BRANCH: ${{ github.ref_name }}
      - working-directory: ./backend
      - run: |
          python tests/scripts/update_cross_platform_trending.py \
            --summary tests/coverage_reports/metrics/cross_platform_summary.json \
            --trending-file tests/coverage_reports/metrics/cross_platform_trend.json \
            --commit-sha $COMMIT_SHA \
            --branch $BRANCH \
            --prune

   f. Detect regressions
      - working-directory: ./backend
      - run: |
          python tests/scripts/coverage_trend_analyzer.py \
            --trending-file tests/coverage_reports/metrics/cross_platform_trend.json \
            --regression-threshold 1.0 \
            --output tests/coverage_reports/metrics/coverage_regressions.json \
            --format json > regression_output.json
      - continue-on-error: true

   g. Generate trend dashboard
      - working-directory: ./backend
      - run: |
          python tests/scripts/generate_coverage_dashboard.py \
            --trending-file tests/coverage_reports/metrics/cross_platform_trend.json \
            --output tests/coverage_reports/dashboards/coverage_trend_30d.html \
            --days 30

   h. Upload dashboard artifact
      - uses: actions/upload-artifact@v4
      - with:
          name: coverage-trend-dashboard
          path: backend/tests/coverage_reports/dashboards/coverage_trend_30d.html
          retention-days: 30

   i. Upload trending data artifact
      - uses: actions/upload-artifact@v4
      - with:
          name: coverage-trending-data
          path: |
            backend/tests/coverage_reports/metrics/cross_platform_trend.json
            backend/tests/coverage_reports/metrics/coverage_regressions.json
            backend/tests/coverage_reports/metrics/cross_platform_summary.json
          retention-days: 30

   j. Post trend report to job summary
      - working-directory: ./backend
      - run: |
          python tests/scripts/update_cross_platform_trending.py \
            --summary tests/coverage_reports/metrics/cross_platform_summary.json \
            --trending-file tests/coverage_reports/metrics/cross_platform_trend.json \
            --periods 7 \
            --format markdown >> $GITHUB_STEP_SUMMARY

   k. Post regression alerts to job summary
      - working-directory: ./backend
      - run: |
          echo "### Coverage Regressions" >> $GITHUB_STEP_SUMMARY
          echo "" >> $GITHUB_STEP_SUMMARY
          if [ -s regression_output.json ]; then
            cat regression_output.json | jq -r '.regressions[] | "- **\(.platform)**: \(.previous_coverage)% → \(.current_coverage)% (\(.delta)%)"' >> $GITHUB_STEP_SUMMARY
          else
            echo "No regressions detected ✓" >> $GITHUB_STEP_SUMMARY
          fi

   l. Dashboard link in job summary
      - run: |
          echo "" >> $GITHUB_STEP_SUMMARY
          echo "### Artifacts" >> $GITHUB_STEP_SUMMARY
          echo "" >> $GITHUB_STEP_SUMMARY
          echo "- [Coverage Trend Dashboard](https://github.com/${{ github.repository }}/actions/runs/${{ github.run_id }})" >> $GITHUB_STEP_SUMMARY
          echo "- [Trending Data (JSON)](https://github.com/${{ github.repository }}/actions/runs/${{ github.run_id }})" >> $GITHUB_STEP_SUMMARY

4. **Conditional failure on critical regressions**
   - Add step that checks regression severity
   - Exit 1 if critical regressions detected (delta < -5%)
   - Only applies to main branch, PRs get warnings

**Workflow design principles:**
- if: always() ensures trending runs even if tests fail
- continue-on-error: true for downloads allows partial results
- Artifact retention 30 days matches trending data retention
- Job summaries provide immediate visibility
- Dashboard link allows easy access to visual trends
</action>

  <files>backend/tests/scripts/ci_trending_integration.py</files>

Create ci_trending_integration.py (100+ lines) with:

1. **Purpose**
   - Orchestrates all trending steps in CI environment
   - Handles missing coverage files gracefully
   - Provides structured output for job summaries

2. **Functions**

   - orchestrate_trending(
       coverage_dir: Path,
       output_dir: Path,
       commit_sha: str,
       branch: str
     ) -> Dict:
     - Run cross_platform_coverage_gate.py
     - Run update_cross_platform_trending.py with commit tracking
     - Run coverage_trend_analyzer.py for regression detection
     - Run generate_coverage_dashboard.py for visual trends
     - Return dict with:
       - success: bool
       - summary_path: Path
       - dashboard_path: Path
       - regression_count: int
       - critical_regressions: List[str]

   - check_coverage_files_exist(coverage_dir: Path) -> Dict[str, bool]:
     - Check for backend/frontend/mobile/desktop coverage files
     - Return dict mapping platform -> file_exists (bool)
     - Log warnings for missing files

   - generate_ci_summary(results: Dict) -> str:
     - Create markdown summary for job posting
     - Include: platform status, regression alerts, dashboard link
     - Return formatted markdown string

   - main():
     - argparse with:
       - --coverage-dir PATH (default: ../coverage/)
       - --output-dir PATH (default: tests/coverage_reports/)
       - --commit-sha STR (from GITHUB_SHA)
       - --branch STR (from GITHUB_REF_NAME)
       - --format json|markdown (default: json)
     - Call orchestrate_trending()
     - Print results in specified format
     - Exit 1 if critical regressions detected

3. **Error handling**
   - Wrap each script execution in try/except
   - Continue on missing files (log warning, don't fail)
   - Return partial results if some steps fail
   - Provide clear error messages for CI debugging

4. **CLI output**
   - JSON format for machine parsing
   - Markdown format for job summaries
   - Console logging for CI output
