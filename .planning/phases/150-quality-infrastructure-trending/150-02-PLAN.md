Create generate_coverage_dashboard.py (250+ lines) with the following structure:

1. **Imports and Configuration**
   - Import argparse, json, sys from standard library
   - Import Path, Dict, List from pathlib/typing
   - Import matplotlib.pyplot as plt
   - Import matplotlib.dates as mdates
   - Import io, base64 for embedding images
   - Import datetime for date handling
   - Define TREND_FILE, OUTPUT_DIR constants
   - Define CHART_COLORS for platforms:
     - backend: "#3B82F6" (blue)
     - frontend: "#10B981" (green)
     - mobile: "#F59E0B" (orange)
     - desktop: "#8B5CF6" (purple)
     - overall: "#111827" (dark gray)
   - Define DEFAULT_DAYS = 30

2. **Data Loading**
   - load_trend_data(trend_file: Path) -> Dict:
     - Reuse load_trending_data() from update_cross_platform_trending.py
     - Return trending data dict with history list

3. **Data Preparation**
   - prepare_chart_data(trending_data: Dict, days: int = 30) -> Dict:
     - Filter history entries to last N days
     - Extract timestamps and coverage values for each platform
     - Return dict with:
       - timestamps: list of datetime objects
       - platforms: {platform: [coverage_values]}
       - overall: [overall_coverage_values]

4. **Chart Generation**
   - create_line_chart(data: Dict, title: str = "Coverage Trend (30 Days)") -> bytes:
     - Create figure with figsize=(12, 6)
     - Filter data to last 30 entries (or N days parameter)
     - Plot overall coverage line (thick, dark)
     - Plot each platform line (thinner, colored)
     - Add legend with platform names
     - Add grid lines for readability
     - Format x-axis with dates (MM-DD format)
     - Set y-axis range 0-100 with "% " suffix
     - Add title and axis labels
     - Use tight_layout() to prevent clipping
     - Save to BytesIO buffer as PNG
     - Return base64-encoded bytes for HTML embedding
     - Close figure to prevent memory leak

   - create_platform_charts(data: Dict) -> Dict[str, bytes]:
     - Create 4 subplots (2x2 grid) for individual platforms
     - Each subplot shows single platform trend with threshold line
     - Color code: above threshold green, below threshold red
     - Return dict mapping platform name -> base64 image

5. **HTML Template**
   - generate_html_template(chart_base64: str, platform_charts: Dict[str, str], data: Dict) -> str:
     - Create self-contained HTML with embedded CSS
     - Include responsive design (works on mobile/desktop)
     - Structure:
       - Header with title and timestamp
       - Main chart (all platforms)
       - Platform-specific charts (2x2 grid)
       - Summary statistics table (current, min, max, avg)
       - Trend indicators (↑↓→)
       - Footer with generation time
     - CSS for:
       - Clean typography (system fonts)
       - Responsive grid layout
       - Card-style containers
       - Color coding (green/red for thresholds)
     - Embedded JavaScript for:
       - Image toggle (hover to zoom)
       - Simple interactivity (show/hide platforms)
     - Return complete HTML string

6. **Statistics Calculation**
   - calculate_statistics(data: Dict) -> Dict:
     - For each platform and overall:
       - current: latest value
       - min: minimum value
       - max: maximum value
       - avg: average value
       - trend: "up"/"down"/"stable" (compare first vs last)
     - Return dict with platform breakdown

7. **CLI Interface**
   - argparse with:
     - --trending-file PATH (default: cross_platform_trend.json)
     - --output PATH (default: coverage_trend_30d.html)
     - --days INT (default: 30)
     - --width INT (default: 1200 pixels)
     - --height INT (default: 600 pixels)

8. **main() function**
   - Parse CLI arguments
   - Load trend data via load_trend_data()
   - Prepare chart data via prepare_chart_data()
   - Generate main chart via create_line_chart()
   - Generate platform charts via create_platform_charts()
   - Calculate statistics via calculate_statistics()
   - Generate HTML via generate_html_template()
   - Write HTML to output file
   - Print confirmation with file path

**Matplotlib configuration:**
- Use 'Agg' backend for non-interactive environments (CI/CD)
- Set dpi=100 for reasonable file size
- Use tight_layout() to prevent label clipping
- Close figures after saving to prevent memory leaks

**HTML design principles:**
- Self-contained (no external CSS/JS references)
- Responsive (viewport meta tag, flexible grid)
- Accessible (alt text, semantic HTML)
- Fast (inline base64 images, no external requests)
