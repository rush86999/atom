  <files>backend/tests/scripts/coverage_trend_analyzer.py</files>
  <action>
Create coverage_trend_analyzer.py (300+ lines) with the following structure:

1. **Imports and Configuration**
   - Import argparse, json, logging, sys from standard library
   - Import Path, Dict, List, Optional from pathlib/typing
   - Import validate from jsonschema for data integrity
   - Import datetime for timestamp handling
   - Define TREND_FILE, REGRESSION_OUTPUT constants
   - Define REGRESSION_THRESHOLD = 1.0 (1% change triggers alert)
   - Configure logging with INFO level

2. **Schema Definition**
   - Define JSON schema for cross_platform_trend.json validation:
     - history: array of trend entries (required)
     - latest: latest entry dict (required)
     - platform_trends: dict mapping platform->trend (required)
     - computed_weights: dict mapping platform->weight (required)
   - Entry schema: timestamp (string), overall_coverage (number), platforms (object), thresholds (object), commit_sha (string, optional), branch (string, optional)

3. **Data Validation**
   - validate_trend_data(trending_data: Dict) -> bool:
     - Use jsonschema.validate() against schema
     - Return True if valid, False if validation fails
     - Log warnings for missing keys, invalid types
     - Handle malformed timestamps gracefully

4. **Regression Detection**
   - detect_regressions(trending_data: Dict, threshold: float = 1.0) -> List[Dict]:
     - Load history from trending_data["history"]
     - Require at least 2 entries for comparison
     - For each platform (backend, frontend, mobile, desktop):
       - Compare current (history[-1]) vs previous (history[-2])
       - Calculate delta = current - previous
       - If delta < -threshold (negative drop exceeds threshold):
         - Create regression dict with:
           - platform: str
           - current_coverage: float
           - previous_coverage: float
           - delta: float (negative value)
           - severity: "critical" if delta < -5.0 else "warning"
           - timestamp_current: str
           - timestamp_previous: str
           - commit_sha: str (from history entries)
     - Return list of regression dicts (empty if none)

5. **Trend Analysis**
   - analyze_trends(trending_data: Dict, periods: int = 7) -> Dict:
     - Call compute_trend_delta() from update_cross_platform_trending.py
     - Calculate moving average (3-period) for each platform
     - Identify trends: "improving" (consistent positive deltas), "declining" (consistent negative deltas), "stable" (mixed deltas)
     - Return dict with:
       - platform_trends: {platform: {trend, delta, moving_avg}}
       - regression_count: int
       - improvement_count: int

6. **Regression Logging**
   - log_regressions(regressions: List[Dict], output_file: Path) -> None:
     - Create output_file parent directory if needed
     - Load existing regressions if file exists
     - Append new regressions with detected_at timestamp
     - Prune old entries (keep last 90 days)
     - Write updated JSON with indent=2

7. **CLI Interface**
   - argparse with:
     - --trending-file PATH (default: cross_platform_trend.json)
     - --regression-threshold FLOAT (default: 1.0)
     - --output PATH (default: coverage_regressions.json)
     - --periods INT (default: 7 for week-over-week)
     - --format text|json|markdown (default: text)

8. **Report Generation**
   - generate_text_report(regressions, trends) -> str:
     - Human-readable report with regression count, severity breakdown, platform summary
   - generate_json_report(regressions, trends) -> str:
     - Machine-readable JSON for CI/CD parsing
   - generate_markdown_report(regressions, trends) -> str:
     - PR comment format with tables and indicators

9. **main() function**
   - Parse CLI arguments
   - Load trending data via load_trending_data()
   - Validate data structure via validate_trend_data()
   - Detect regressions via detect_regressions()
   - Analyze trends via analyze_trends()
   - Log regressions via log_regressions()
   - Generate report based on --format
   - Exit 1 if critical regressions detected (severity="critical")

**Pattern reuse from existing code:**
- Use TrendDelta dataclass from update_cross_platform_trending.py
- Use load_trending_data() pattern for error handling
- Use logging pattern (logger.info, logger.error, logger.warning)
- Use Path.parent.mkdir(parents=True, exist_ok=True) for directory creation
