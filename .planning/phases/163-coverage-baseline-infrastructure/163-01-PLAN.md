# Plan 163-01: Branch Coverage Configuration and Baseline Generation

## Objective
Enable branch coverage measurement in pytest configuration and generate comprehensive baseline report with actual line and branch coverage metrics (not service-level estimates).

**Purpose:** Ensure team has accurate baseline coverage measurement to prevent false confidence from service-level estimation errors discovered in Phases 160-162.

**Output:** Validated baseline with actual line + branch coverage from coverage.py execution data.

## Tasks

<task id="1" type="auto">
  <name>Verify and enhance pytest.ini for branch coverage</name>
  <files>
    - backend/pytest.ini
  </files>
  <action>
    Ensure pytest.ini has branch coverage enabled by verifying:
    - `--cov-branch` flag is present in addopts for branch coverage measurement
    - `--cov-report=json` is present for JSON output to coverage.json
    - `--cov-report=term-missing` is present for terminal output showing missing lines
    - `--cov-report=html` is present for HTML reports for detailed inspection
    - Coverage target is `backend` module (not `backend/core` or other submodules that would cause service-level aggregation)
  </action>
  <verify>
    Run: `grep -E "(--cov-branch|--cov-report)" backend/pytest.ini`
    Expected output shows all four flags present
  </verify>
  <done>
    pytest.ini contains --cov-branch, --cov-report=json, --cov-report=term-missing, --cov-report=html flags
  </done>
</task>

<task id="2" type="auto">
  <name>Create standardized baseline generation script</name>
  <files>
    - backend/tests/scripts/generate_baseline_coverage_report.py
  </files>
  <action>
    Create script at backend/tests/scripts/generate_baseline_coverage_report.py that:
    - Runs `pytest --cov=backend --cov-branch --cov-report=json --cov-report=html --cov-report=term-missing`
    - Parses coverage.json to extract overall metrics (line_covered/total_lines, branch_covered/total_branches)
    - **Validates coverage.json has `files` array with per-file data** by checking:
      - `coverage_json['files']` exists and is a list
      - Each file has `summary` dict with `covered` and `num_statements` keys
      - `coverage_json['totals']` exists with line and branch coverage data
      - Raises error if only high-level totals exist without per-file breakdown (indicates service-level aggregation)
    - Creates baseline summary report in markdown at backend/tests/coverage_reports/backend_163_baseline.md
    - Saves coverage.json to backend/tests/coverage_reports/backend_163_baseline.json

    Script must verify this is actual line execution data from coverage.py, not service-level estimates.
  </action>
  <verify>
    Run: `python backend/tests/scripts/generate_baseline_coverage_report.py`
    Expected: Script executes without errors, produces coverage.json and baseline summary markdown
    Additional verification: `python -c "import json; d=json.load(open('backend/tests/coverage_reports/backend_163_baseline.json')); assert 'files' in d and len(d['files']) > 0"`
  </verify>
  <done>
    Script generates coverage.json with files array containing per-file line + branch metrics; baseline summary markdown created
  </done>
</task>

3. **Branch coverage is enabled**: Baseline includes branch coverage metrics from --cov-branch flag
4. **Baseline prevents false confidence**: Methodology documentation clearly states this is actual line execution data, not service-level estimates
