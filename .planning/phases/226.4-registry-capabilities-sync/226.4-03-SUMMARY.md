## Implementation Details

### ProviderScheduler Architecture

**Class Structure:**
```python
class ProviderScheduler:
    _instance: Optional["ProviderScheduler"] = None

    def __init__(self):
        self.scheduler = AsyncIOScheduler()
        self.auto_discovery = get_auto_discovery()
        self.health_monitor = get_provider_health_monitor()

    @classmethod
    def get_instance(cls) -> "ProviderScheduler":
        if cls._instance is None:
            cls._instance = ProviderScheduler()
        return cls._instance

    def start(self):
        # Schedule auto-sync every 24 hours if enabled
        self.scheduler.add_job(
            self._sync_with_error_handling,
            'interval',
            hours=24,
            id='provider_auto_sync',
            max_instances=1,
            coalesce=True,
            replace_existing=True
        )
        self.scheduler.start()

    async def _sync_with_error_handling(self):
        try:
            result = await self.auto_discovery.sync_providers()
            self.health_monitor.record_call("provider_registry", True, 0)
        except Exception as e:
            self.health_monitor.record_call("provider_registry", False, 0)
```

**Job Scheduling Parameters:**
- **Trigger:** interval (every 24 hours)
- **Job ID:** 'provider_auto_sync'
- **max_instances:** 1 (prevent overlapping executions)
- **coalesce:** True (combine missed executions)
- **replace_existing:** True (replace job on restart)

**Health Monitoring Integration:**
- Sync success recorded as `health_monitor.record_call("provider_registry", True, 0)`
- Sync failure recorded as `health_monitor.record_call("provider_registry", False, 0)`
- Latency not measured for sync operations (0ms placeholder)

## Decisions Made

- **AsyncIOScheduler over BackgroundScheduler:** Critical decision because sync_providers is an async function. BackgroundScheduler doesn't support async job functions, while AsyncIOScheduler is designed for async/await patterns.

- **24-hour polling interval:** Balances freshness (daily updates) with API rate limits from LiteLLM/OpenRouter. More frequent polling would provide minimal value since provider/model changes are infrequent.

- **max_instances=1:** Prevents overlapping sync jobs. If a sync takes longer than 24 hours (unlikely but possible with network issues), the next scheduled execution won't start until the previous one completes.

- **coalesce=True:** Combines missed executions if server was down. If the server is offline for 3 days, only 1 sync runs on restart (not 3 accumulated syncs), reducing load and API usage.

- **Environment variable toggle:** Provides operational control without code changes. Useful for maintenance windows, debugging, or disabling sync in specific environments (staging, development).

- **Health monitor integration:** Tracks sync operations as 'provider_registry' provider for operational visibility. Enables monitoring sync health alongside actual provider health metrics.
