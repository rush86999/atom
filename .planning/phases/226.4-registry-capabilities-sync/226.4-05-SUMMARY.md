## API Endpoints

### 1. GET /api/providers/health

**Overall provider registry health status**

**Response:**
```json
{
  "status": "healthy",
  "timestamp": "2026-03-22T20:30:00.000Z",
  "total_providers": 108,
  "healthy_providers": 108,
  "unhealthy_providers": 0,
  "providers": [
    {
      "provider_id": "openai",
      "health_score": 0.95,
      "model_count": 50
    }
  ]
}
```

**Features:**
- Total provider count
- Healthy provider count (health_score >= 0.5)
- Unhealthy provider count
- Per-provider health scores and model counts
- ISO timestamp format

### 2. GET /api/providers/{provider_id}/health

**Per-provider health details**

**Response:**
```json
{
  "provider_id": "openai",
  "name": "OpenAI",
  "health_score": 0.95,
  "is_healthy": true,
  "last_updated": "2026-03-22T19:00:00.000Z",
  "is_active": true,
  "supports_vision": true,
  "supports_tools": true
}
```

**Features:**
- Provider ID and name
- Health score (0.0-1.0)
- is_healthy flag (true if score >= 0.5)
- Last updated timestamp
- Active status
- Capability flags (vision, tools)

**Error Responses:**
- 404: Provider not found

### 3. POST /api/providers/sync

**Manual provider registry sync trigger**

**Response:**
```json
{
  "success": true,
  "timestamp": "2026-03-22T20:30:00.000Z",
  "providers_synced": 108,
  "models_synced": 2595
}
```

**Features:**
- Triggers immediate sync from DynamicPricingFetcher to ProviderRegistry
- Returns counts of providers and models synced
- ISO timestamp format

**Error Responses:**
- 500: Sync failed

## Decisions Made

- **datetime.now(timezone.utc) instead of datetime.utcnow():** Fixed deprecation warning for Python 3.14+ compatibility. All timestamps now use timezone-aware datetime objects.

- **ProviderScheduler environment toggle:** Scheduler only starts when PROVIDER_AUTO_SYNC_ENABLED=true (default: true). This allows disabling auto-sync in API-only replicas.

- **Health endpoints without authentication:** Provider health endpoints are operational monitoring endpoints that don't require authentication. This follows the pattern of existing health check endpoints (/health/live, /health/ready).

- **Integration tests with real database:** Tests use real database connections instead of mocks for simplicity. This provides more realistic integration testing and avoids complex mock setup.

- **Graceful shutdown handling:** ProviderScheduler is stopped gracefully in the lifespan shutdown function to ensure clean resource cleanup.
