  - Improves query performance for filtering specialized models
  - Enables efficient WHERE exclude_from_general_routing = False queries

## Capability Detection Logic

### Auto-Detection Rules

1. **Default capabilities:** ["chat"] for all models
2. **Vision capability:** Added if mode="vision" or supports_vision=True
3. **Tools capability:** Added if supports_function_calling=True
4. **Computer use capability:** Added for LUX models (model_name.startswith("lux"))
5. **Browser use capability:** Added for LUX models
6. **LUX specialization:** LUX models get ["computer_use", "browser_use"] WITHOUT "chat"
7. **Vision-only models:** Remove "chat" if mode="vision" and no chat support

### Examples

- **lux-1.0:** ["computer_use", "browser_use"] → exclude_from_general_routing=True
- **gpt-4o:** ["chat", "vision", "tools"] → exclude_from_general_routing=False
- **claude-3.5-sonnet:** ["chat", "vision", "tools"] → exclude_from_general_routing=False
- **gemini-2.0-flash:** ["chat", "vision", "tools"] → exclude_from_general_routing=False

### Fallback Hierarchy

1. **Exact match:** MODEL_CAPABILITY_SCORES[capability][model_id]
2. **Partial match:** Search for model_id substring in capability scores
3. **General quality:** MODEL_QUALITY_SCORES[model_id]
4. **Heuristics:** Model name-based heuristics (reasoner, flash, mini, etc.)
5. **Floor:** 70 for completely unknown models
