<objective>
**Set up type generation infrastructure** installing openapi-typescript, exporting the backend OpenAPI spec, and generating initial TypeScript types for use across frontend, mobile, and desktop platforms.

**Purpose:** The current codebase lacks automated TypeScript type generation from the FastAPI OpenAPI specification. Developers manually maintain API types which drift from the backend contract, causing type mismatches and runtime errors. This plan establishes the type generation infrastructure using openapi-typescript, the industry-standard tool with 1M+ weekly downloads.

**Output:**
- openapi-typescript installed in frontend-nextjs devDependencies
- Backend OpenAPI spec exported to backend/openapi.json (OpenAPI 3.0.3)
- Initial TypeScript types generated at frontend-nextjs/src/types/api-generated.ts
- npm scripts for type regeneration (generate:api-types, generate:api-types:watch)
- Generated types file with DO NOT EDIT header comment
</objective>

<task type="auto">
  <name>Task 1: Install openapi-typescript and add npm scripts</name>
  <files>frontend-nextjs/package.json</files>
  <action>
Install openapi-typescript and configure npm scripts:

1. Install openapi-typescript as dev dependency:
   ```bash
   cd frontend-nextjs
   npm install -D openapi-typescript@latest
   ```

2. Add npm scripts to package.json scripts section:
   ```json
   "generate:api-types": "npx openapi-typescript ../backend/openapi.json -o src/types/api-generated.ts",
   "generate:api-types:watch": "npx openapi-typescript http://localhost:8000/openapi.json -o src/types/api-generated.ts --watch"
   ```

3. Place scripts alphabetically after "generate:types" if it exists, or near other generate:* scripts

4. Verify installation: `npm info openapi-typescript version` should show 7.13.0 or later

Do NOT use openapi-generator-cli or orval (per RESEARCH.md, they add framework coupling)
  </action>
  <verify>
```bash

grep "generate:api-types" frontend-nextjs/package.json
npm list openapi-typescript
```
  </verify>
  <done>
openapi-typescript installed in devDependencies with generate:api-types and generate:api-types:watch scripts

  <action>
Generate TypeScript types from OpenAPI spec:

1. Create types directory if not exists:
   ```bash
   mkdir -p frontend-nextjs/src/types
   ```

2. Run type generation:
   ```bash
   cd frontend-nextjs
   npm run generate:api-types
   ```

3. Verify generated file:
   - Exists at frontend-nextjs/src/types/api-generated.ts
   - Contains export interface paths (endpoint definitions)
   - Contains export interface components (schema definitions)
   - Contains export interface operations (operation definitions)
   - File size should be substantial (1000+ lines expected for 100+ endpoints)

4. Add file header comment (edit the file, add at top):
   ```typescript
   /**
    * Auto-generated from backend OpenAPI spec
    * DO NOT EDIT - Regenerate with: npm run generate:api-types
    * Source: backend/openapi.json
    * Generated: {timestamp}
    */
   ```

Do NOT use --namespace flag yet (decide in Plan 02 if needed based on file size/collisions)
