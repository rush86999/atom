<task type="auto">
  <name>Create mobile tests CI workflow</name>
  <files>.github/workflows/mobile-tests.yml</files>
  <action>
    Create .github/workflows/mobile-tests.yml with:

    **Workflow triggers:**
    - push: branches: [main]
    - pull_request: branches: [main]
    - workflow_dispatch: (for manual triggering)

    **Jobs structure:**

    1. **test-mobile job**:
       - runs-on: macos-latest (for iOS compatibility) OR ubuntu-latest (faster, for Android)
       - timeout-minutes: 15

    **Steps:**
    1. Checkout repository: actions/checkout@v4
    2. Setup Node.js: actions/setup-node@v4 with node-version: 18 or 20
    3. Cache node_modules: actions/cache@v4 with key: ${{ runner.os }}-mobile-${{ hashFiles('mobile/package-lock.json') }}
    4. Install dependencies: working-directory: ./mobile, run: npm ci
    5. Run tests with coverage: working-directory: ./mobile, run: npm run test:coverage -- --maxWorkers=2
    6. Upload coverage artifact: uses: actions/upload-artifact@v4 with:
       - name: mobile-coverage
       - path: mobile/coverage/coverage-final.json
       - retention-days: 7
    7. Upload coverage HTML (optional): uses: actions/upload-artifact@v4 with:
       - name: mobile-coverage-html
       - path: mobile/coverage/lcov-report
       - retention-days: 7

    **Optional: Matrix for platform testing** (if time permits):
    - strategy.matrix.os: [macos-latest, ubuntu-latest]
    - This ensures tests work on both platforms

    Follow patterns from unified-tests.yml (095-03) for consistency.
  </action>
  <verify>cat .github/workflows/mobile-tests.yml | grep -E "(runs-on|npm run test|upload-artifact)"</verify>
  <done>mobile-tests.yml workflow exists with test and coverage upload steps</done>
</task>
