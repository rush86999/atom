## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frontend→auth tests | User authentication flows tested with mocked services |
| frontend→automation tests | Complex ReactFlow components with mocked dependencies |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-255-01 | Spoofing | Auth test mock data | accept | Test authentication flows with mocked credentials; no real authentication in tests |
| T-255-02 | Tampering | ReactFlow component state | mitigate | Use shallow rendering and mock ReactFlow dependencies to prevent state manipulation |
| T-255-03 | Repudiation | Test coverage measurement | accept | Coverage measurement is read-only analysis; no impact on production code |
| T-255-04 | Information Disclosure | Component props in tests | accept | Tests use dummy data; no sensitive information exposure |
