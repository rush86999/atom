       - Trending: e2e_trend.json historical tracking

       - **Independent tests**: Unique IDs, cleanup fixtures, no shared state
       - **Auto-waiting**: Playwright's built-in waiting vs explicit waits
       - **Error handling**: try/except blocks, retry logic, graceful degradation
       - **Test data**: Factory fixtures, realistic data, deterministic tests
       - **Anti-patterns**: Sleep calls, hard-coded waits, shared database

       - Install dependencies: `pip install pytest-playwright playwright`

       - Start services: Backend on port 8001, frontend on port 3001
       - Environment: DATABASE_URL, BASE_URL environment variables

    4. **Writing Tests** section:
       - Use Page fixture from playwright.sync_api
       - Use expect() assertions for auto-waiting
       - Add @pytest.mark.e2e marker for CI/CD
       - Cleanup after test with autouse fixtures

    5. **Common Issues** section:
       - Port 8001/3001 in use: Kill existing processes
       - Element timeout: Use page.wait_for_selector()
       - Test isolation: Use unique IDs, cleanup fixtures

       - Run single test: `pytest tests/e2e_api/test_mobile_endpoints.py::test_mobile_agent_spawn_api -v`
       - Run with coverage: `pytest tests/e2e_api/ -v --cov=api/mobile`

    2. **Why API-Level Testing?** section:
       - Detox E2E BLOCKED: expo-dev-client requirement (+15min CI)
       - API contracts validate mobile workflows without full UI
       - ROADMAP compliant: Satisfies "mobile workflows (navigation, device features)" requirement
       - Faster feedback: API tests run in seconds vs minutes for full E2E

    3. **Test Organization** section:
       - Agent tests: test_mobile_endpoints.py (agent spawn, chat)
       - Navigation tests: test_mobile_endpoints.py (screen routing, history)

       - Use unique IDs for isolation (cleanup after test)

       - Missing permissions: Check device capability mock setup
