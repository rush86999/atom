  <action>
  Add concurrent payment safety mechanisms to StripeService:

  Modify `backend/integrations/stripe_service.py`:
  1. Add imports: `import threading`, `from functools import wraps`

  2. Add per-customer locks for preventing concurrent charges to same customer:
     ```python
     # Class-level lock registry
     _customer_locks = {}
     _lock_registry_lock = threading.Lock()

     @classmethod
     def _get_customer_lock(cls, customer_id: str) -> threading.Lock:
         """Get or create lock for specific customer (prevents double-charging)"""
         with cls._lock_registry_lock:
             if customer_id not in cls._customer_locks:
                 cls._customer_locks[customer_id] = threading.Lock()
             return cls._customer_locks[customer_id]
     ```

  3. Add `@synchronized_payment` decorator for critical sections:
     ```python
     def synchronized_payment(func):
         """Decorator to prevent concurrent payment processing for same customer"""
         @wraps(func)
         def wrapper(self, *args, **kwargs):
             # Extract customer_id from kwargs or args
             customer_id = kwargs.get('customer') or (args[1] if len(args) > 1 else None)
             if customer_id:
                 lock = self._get_customer_lock(str(customer_id))
                 with lock:
                     return func(self, *args, **kwargs)
             else:
                 return func(self, *args, **kwargs)
         return wrapper
     ```

  4. Apply `@synchronized_payment` decorator to `create_payment()` method:
     - Prevents concurrent charges to same customer from causing double-charges

  5. Add `cleanup_old_locks()` classmethod:
     - Remove locks for customers not seen in >1 hour (prevents memory leak)

  IMPORTANT: Client-side locks are NOT a substitute for database transactions. This is a defense-in-depth layer. Real concurrency control happens at Stripe API level (idempotency keys).
