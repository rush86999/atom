    Extend `backend/tests/scripts/generate_coverage_dashboard.py` (483 lines, existing) with new functions:

    **`generate_trend_dashboard(trend_data)`** - Generate markdown dashboard with ASCII charts:
    - Reads from coverage_trend_v5.0.json (Phase 100 format)
    - Uses existing `generate_visualization()` from coverage_trend_tracker.py
    - Creates ASCII line chart showing last 30 snapshots
    - Marks baseline (B), current (C), and target (80%) line

    **`generate_module_charts(trend_data)`** - Per-module ASCII charts:
    - Extract module history (core, api, tools) from trend data
    - Generate separate ASCII chart for each module
    - Format: small charts (40 chars wide) side-by-side

    **`calculate_forecast_to_target(trend_data, target=80.0)`** - Timeline estimation:
    - Calculate average coverage gain per snapshot
    - Generate 3 scenarios: optimistic (130%), realistic (100%), pessimistic (70%)
    - Return estimated days/phases remaining to 80%
    - Format: "X days" or "Phase NNN"

    **Dashboard markdown structure**:
    ```markdown

    ## Overall Coverage Trend
    [ASCII chart with 30 history points, target marker]

    ## Module Breakdown
    ### Core (XX.XX%)
    [ASCII chart for core module]

    ### API (XX.XX%)
    [ASCII chart for api module]

    ### Tools (XX.XX%)
    [ASCII chart for tools module]

    ## Progress to 80% Goal
    - Current: XX.XX%
    - Remaining: XX.XX%
    - Progress: XX.X%

    ## Forecast to 80%
    - Optimistic: X days (130% rate)
    - Realistic: X days (100% rate)
    - Pessimistic: X days (70% rate)

    ## Recent Snapshots
    | Date | Coverage | Delta | Commit |
    |------|----------|-------|--------|
    ...
    ```
