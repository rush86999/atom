  <name>Task 1: Create PR coverage comment bot script</name>
  <files>backend/tests/scripts/pr_coverage_comment_bot.py</files>
  <action>
    Create `backend/tests/scripts/pr_coverage_comment_bot.py` with the following functions:

    **`generate_pr_comment_payload(coverage_file, base_branch, commit_hash)`**:
    - Reads coverage.json for current coverage
    - Reads baseline coverage from git history of base branch
    - Calculates delta (current - baseline)
    - Identifies files with coverage drops (>1% decrease)
    - Returns JSON payload with comment body structured for GitHub API

    **`get_file_by_file_delta(coverage_file, base_branch, changed_files)`**:
    - Uses `diff-cover` library for accurate coverage delta calculation
    - Accepts list of changed files from `git diff --name-only`
    - Returns list of files with: {path, before_pct, after_pct, delta_pct}
    - Only includes files that decreased in coverage

    **`main()` CLI entry point**:
    - Accepts args: `--coverage-file`, `--base-branch`, `--output-file`
    - Generates comment payload JSON
    - Outputs to stdout or file for GitHub Actions consumption
    - Exit codes: 0 (success), 1 (error), 2 (no coverage data)

    **Comment format requirements**:
    ```markdown
    ## Coverage Report
    | Metric | Value |
    |--------|-------|
    | Coverage | XX.XX% |
    | Delta | +X.XX% |
    | Target | 80% |

    ### Files with Coverage Drops
    | File | Before | After | Change |
    |------|--------|-------|--------|
    | path/to/file.py | 85% | 72% | -13% |
    ```

    Use `subprocess.run()` for git commands (get baseline coverage, changed files).
    Handle missing files gracefully (return warning message, don't fail).
  </action>

<task type="auto">
  <name>Task 2: Extend coverage-report.yml workflow with PR comment step</name>
  <files>.github/workflows/coverage-report.yml</files>
  <action>
    Extend `.github/workflows/coverage-report.yml` to add PR comment generation:

    **After "Run tests with coverage" step, add new step**:

    ```yaml
    - name: Generate PR comment payload
      if: github.event_name == 'pull_request'
      run: |
        cd backend
        python tests/scripts/pr_coverage_comment_bot.py \
          --coverage-file tests/coverage_reports/metrics/coverage.json \
          --base-branch origin/main \
          --output-file pr_comment_payload.json

    - name: Post PR comment with coverage delta
      if: github.event_name == 'pull_request'
      uses: actions/github-script@v7
      with:
        script: |
          const fs = require('fs');
          const payload = JSON.parse(
            fs.readFileSync('backend/pr_comment_payload.json', 'utf8')
          );

          // Find existing comment to avoid duplicates
          const { data: comments } = await github.rest.issues.listComments({
            owner: context.repo.owner,
            repo: context.repo.repo,
            issue_number: context.issue.number,
          });

          const botComment = comments.find(comment =>
            comment.user.type === 'Bot' &&
            comment.body.includes('## Coverage Report')
          );

          if (botComment) {
            // Update existing comment
            await github.rest.issues.updateComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              comment_id: botComment.id,
              body: payload.body
            });
          } else {
            // Create new comment
            await github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              body: payload.body
            });
          }
    ```

    Preserve existing workflow functionality (trending, regression check, artifacts).
    Use `actions/github-script@v7` (already used in workflow line 82).
    Implement comment update logic to avoid spam (find existing bot comment, update instead of creating new).
