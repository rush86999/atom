    Create `backend/tests/scripts/per_commit_report_generator.py` with the following functions:

    **`generate_commit_report(coverage_file, commit_hash, timestamp)`** - Generate report dict:
    ```python
    def generate_commit_report(coverage_file, commit_hash, timestamp=None):
        """Generate per-commit coverage report.

        Returns:
            Dict with: commit, timestamp, overall_coverage, branch_coverage,
            module_breakdown, files_below_80, top_uncovered_files
        """
        # Read coverage.json
        # Extract totals (percent_covered, covered_lines, num_statements)
        # Calculate module breakdown (core, api, tools)
        # Find files below 80% threshold
        # Get top 10 uncovered files (sorted by missing_lines descending)
    ```

    **`extract_module_breakdown(coverage_data)`** - Extract module-level coverage:
    - Iterate through coverage_data["files"]
    - Group by module (core/, api/, tools/, skills/)
    - Calculate per-module: covered, total, percent
    - Return: {module: {covered, total, percent}}

    **`get_files_below_threshold(coverage_data, threshold=80.0)`** - Find low-coverage files:
    - Filter files where percent_covered < threshold
    - Sort by uncovered_lines descending
    - Return: [{path, percent_covered, uncovered_lines, total_lines}]

    **`store_commit_report(report, commits_dir)`** - Store JSON report:
    - Creates commits_dir if not exists
    - Filename: `{short_hash}_coverage.json` (first 8 chars of commit hash)
    - Include full hash in report["commit"] field
    - Timestamp in UTC ISO format with 'Z' suffix

    **`cleanup_old_reports(commits_dir, days_to_keep=90)`** - Remove old reports:
    - Iterate through *.json files in commits_dir
    - Parse timestamp from report["timestamp"]
    - Delete files older than days_to_keep
    - Return count of deleted files

    **`main()` CLI entry point**:
    - Args: `--coverage-file`, `--commits-dir`, `--commit`, `--retention-days`
    - Generates report, stores to file, runs cleanup
    - Returns exit code 0 on success, 1 on error, 2 if no coverage data
    - Supports `--list` flag to list existing reports

    Report JSON structure:
    ```json
    {
      "commit": "abc123def456...",
      "short_hash": "abc123de",
      "timestamp": "2026-03-01T12:34:56Z",
      "overall_coverage": 65.43,
      "covered_lines": 12345,
      "total_lines": 18876,
      "branch_coverage": 58.32,
      "module_breakdown": {
        "core": {"percent": 62.1, "covered": 5000, "total": 8050},
        "api": {"percent": 71.5, "covered": 4000, "total": 5594},
        "tools": {"percent": 58.2, "covered": 3345, "total": 5747}
      },
      "files_below_80": 45,
      "top_uncovered_files": [
        {"path": "core/workflow_engine.py", "uncovered": 543, "total": 1089, "percent": 50.1},
        ...
      ]
    }
    ```

    Target file size: ~1-2KB per report (limit top_uncovered_files to 10 entries).

<task type="auto">
  <name>Task 3: Add per-commit report generation to CI workflow</name>
  <files>.github/workflows/coverage-report.yml</files>
  <action>
    Add per-commit report generation step to `.github/workflows/coverage-report.yml`:

    **After "Run tests with coverage" step (line ~34)**:

    ```yaml
    - name: Generate per-commit coverage report
      if: always()  # Run even if tests fail (coverage still useful)
      run: |
        cd backend
        COMMIT_HASH="${{ github.sha }}"
        python3 tests/scripts/per_commit_report_generator.py \
          --coverage-file tests/coverage_reports/metrics/coverage.json \
          --commits-dir tests/coverage_reports/commits \
          --commit "$COMMIT_HASH" \
          --retention-days 90

    - name: Upload commit reports as artifacts
      if: always()
      uses: actions/upload-artifact@v4
      with:
        name: coverage-commit-reports
        path: backend/tests/coverage_reports/commits/*.json
        retention-days: 90
        if-no-files-found: ignore

    - name: Commit new reports to repository
      if: github.ref == 'refs/heads/main' && always()
      run: |
        git config user.name "github-actions[bot]"
        git config user.email "github-actions[bot]@users.noreply.github.com"
        git add backend/tests/coverage_reports/commits/*.json
        git add backend/tests/coverage_reports/commits/README.md
        git diff --staged --quiet || git commit -m "chore: add coverage report for ${{ github.sha }}"
        git push
      continue-on-error: true  # Don't fail if push conflict
    ```

    Use `GITHUB_SHA` environment variable for commit hash (auto-provided by GitHub Actions).
    Run with `if: always()` to ensure reports even on test failures.
    Store reports as workflow artifacts (backup for git commit failures).
    Commit to main branch only (not PRs).
    Use `continue-on-error: true` for commit step (avoid failures on concurrent runs).
  </action>
