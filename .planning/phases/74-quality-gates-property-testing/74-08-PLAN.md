    ## Writing New Property Tests

    1. Identify a critical invariant (something that MUST always be true)
    2. Use `@given` decorator with strategies to generate random inputs
    3. Set `max_examples=50-200` for thoroughness vs speed tradeoff
    4. Add `VALIDATED_BUG` docstring documenting bugs found/prevented
    5. Use `suppress_health_check=[HealthCheck.function_scoped_fixture]` for DB tests

    Example:
    ```python
    from hypothesis import given, settings, HealthCheck
    from hypothesis import strategies as st

    @given(
        value=st.integers(min_value=0, max_value=100)
    )
    @settings(max_examples=100)
    def test_value_always_positive(self, value):
        '''INVARIANT: Value should never be negative after processing.'''
        result = process(value)
        assert result >= 0, f"Result {result} should be non-negative"
    ```
