    Fix the flaky test `test_appointment_time_range` in TestAppointmentModel class.

    **Root cause:** The test uses datetime.utcnow() twice which creates microsecond differences:
    - start = datetime.utcnow() + timedelta(days=1, hours=10)
    - end = datetime.utcnow() + timedelta(days=1, hours=11)
    - These are evaluated at different times, creating ~15 microsecond drift
    - Assertion expects exact 3600 seconds but gets 3600.000015

    **Fix approach (2 options, choose ONE):**

    Option A (Recommended): Use a base datetime and calculate both times from it
    ```python
    base_time = datetime.utcnow().replace(microsecond=0)
    start = base_time + timedelta(days=1, hours=10)
    end = base_time + timedelta(days=1, hours=11)
    # OR use timezone-aware:
    from datetime import timezone
    base_time = datetime.now(timezone.UTC).replace(microsecond=0)
    start = base_time + timedelta(days=1, hours=10)
    end = base_time + timedelta(days=1, hours=11)
    ```

    Option B: Use approximate comparison for seconds
    ```python
    assert abs((appointment.end_time - appointment.start_time).total_seconds() - 3600) < 0.001
    ```
