    Refactor BYOKHandler to move inline imports to module level:

    **Current Issue (from 194-04):**
    - Inline imports inside methods prevent proper mocking
    - Coverage stuck at 36.4% due to unmockable imports
    - Example: `import openai` inside `call_openai()` method

    **Refactoring Steps:**
    1. Identify all inline imports in BYOKHandler methods
    2. Move imports to module level (top of file)
    3. Wrap imports in try/except for optional dependencies
    4. Update method logic to handle missing imports gracefully
    5. Ensure backward compatibility (no API changes)

    **Pattern:**
    ```python
    # Before (inline import - blocks mocking):
    def call_openai(self, ...):
        import openai  # <-- Cannot mock this
        ...

    # After (module-level import - allows mocking):
    try:
        import openai
    except ImportError:
        openai = None

    def call_openai(self, ...):
        if openai is None:
            raise ImportError("openai not installed")
        ...
    ```

    **Files to Modify:**
    - backend/core/llm/byok_handler.py

      - Add try/except for optional dependencies
      - Update methods to check for import availability
