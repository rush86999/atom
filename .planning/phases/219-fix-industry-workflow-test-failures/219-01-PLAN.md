# Plan 219-01: Remove Duplicate Test File and Fix Industry Workflow Tests

## Objective

Fix failing industry workflow endpoint tests by removing the duplicate test file and correcting the remaining issues in the unit test file.

**Purpose:** The duplicate test file `tests/api/services/test_industry_workflow_endpoints.py` was created in Phase 219 but uses incorrect fixture-based mocking that doesn't intercept FastAPI's dependency injection. The correct `tests/unit/test_industry_workflow_endpoints.py` file uses `@patch` decorators but has some test issues.

**Output:** All 17 tests passing, no duplicate files.

## Problem Analysis

### Duplicate Files Identified
1. `tests/unit/test_industry_workflow_endpoints.py` (358 lines) - Uses correct `@patch` pattern
2. `tests/api/services/test_industry_workflow_endpoints.py` (441 lines) - Uses incorrect fixture pattern

### Root Causes of Test Failures

**tests/api/services/test_industry_workflow_endpoints.py:**
- Uses `@pytest.fixture` mock which doesn't intercept FastAPI dependency injection
- Tests get real data (7 industries, actual template IDs) instead of mocked data
- Result: 14/19 tests failing

**tests/unit/test_industry_workflow_endpoints.py:**
- Uses correct `@patch` decorator pattern
- Failures are due to:
  1. Template ID mismatch (tests expect "test_template_1" but real templates exist like "healthcare_patient_onboarding")
  2. ROI tests fail with 422 (Pydantic validation error on path template_id vs body template_id)
  3. Some tests call real implementation instead of mock
