1. **Request validation** - API request body validation
2. **Schema validation** - Complex object schema validation
3. **Type conversion** - String to type coercion
4. **SQL injection prevention** - Database query safety
5. **XSS prevention** - Output encoding
6. **Path traversal prevention** - File system safety

## Notes

**Test Quality:**
- Focus on validation that prevents attacks
- Test with real attack patterns (OWASP Top 10)
- Verify attacks are blocked, not just logged
- Ensure error messages don't leak sensitive info

**Security Testing:**
- Use real attack patterns from OWASP
- Test SQL injection with various techniques
- Test XSS with various script vectors
- Test path traversal with various encodings
- Test authentication/authorization bypass

**Validation Best Practices:**
- Validate all inputs (never trust user input)
- Use whitelist validation (allow known good)
- Reject invalid input early (fail fast)
- Sanitize output (escape for context)
- Use parameterized queries (prevent SQL injection)
- Encode output (prevent XSS)

**Security Best Practices:**
- Defense in depth (multiple layers)
- Principle of least privilege
- Fail secure (default deny)
- Don't leak info in error messages
- Rate limit authentication endpoints
- Use strong encryption (HTTPS, signed JWTs)
