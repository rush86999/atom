## Context

State machines and transitions are critical for system correctness. This plan focuses on:

1. **Agent maturity transitions** - STUDENT → INTERN → SUPERVISED → AUTONOMOUS
2. **Episode lifecycle** - active → completed → archived
3. **Workflow states** - initialized → running → completed/failed
4. **Execution states** - pending → running → completed/failed/cancelled
5. **Transition validation** - Valid vs invalid transitions

## Target Areas

Based on coverage gap analysis from Phase 260, the following state machines need coverage:

### High Priority State Machines
1. **Agent Maturity State Machine** - `core/agent_graduation_service.py`
   - States: STUDENT, INTERN, SUPERVISED, AUTONOMOUS
   - Transitions: Forward only (with graduation criteria)
   - Invalid: Reverse transitions, skip levels

2. **Episode Lifecycle State Machine** - `core/episode_lifecycle_service.py`
   - States: active, completed, archived, deleted
   - Transitions: active → completed → archived → deleted
   - Invalid: active → archived, completed → active

3. **Workflow Execution State Machine** - `core/workflow_engine.py`
   - States: initialized, running, completed, failed, cancelled
   - Transitions: initialized → running → (completed | failed | cancelled)
   - Invalid: completed → running, failed → running

4. **Agent Execution State Machine** - `core/agent_execution.py`
   - States: pending, running, completed, failed, cancelled
   - Transitions: pending → running → (completed | failed | cancelled)
   - Invalid: completed → running

5. **Training Session State Machine** - `core/student_training_service.py`
   - States: proposed, approved, in_progress, completed, cancelled
   - Transitions: proposed → approved → in_progress → completed
   - Invalid: in_progress → proposed, completed → in_progress
