## Database Schema

### Table: provider_registry
```sql
CREATE TABLE provider_registry (
    provider_id VARCHAR(50) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description VARCHAR(500),
    litellm_provider VARCHAR(50),
    base_url VARCHAR(500),
    supports_vision BOOLEAN DEFAULT FALSE,
    supports_tools BOOLEAN DEFAULT FALSE,
    supports_cache BOOLEAN DEFAULT FALSE,
    supports_structured_output BOOLEAN DEFAULT FALSE,
    reasoning_level INTEGER,
    quality_score FLOAT,
    is_active BOOLEAN DEFAULT TRUE,
    discovered_at DATETIME,
    last_updated DATETIME,
    provider_metadata JSON
);
CREATE INDEX ix_provider_registry_is_active ON provider_registry(is_active);
```

### Table: model_catalog
```sql
CREATE TABLE model_catalog (
    model_id VARCHAR(100) PRIMARY KEY,
    provider_id VARCHAR(50) NOT NULL,
    name VARCHAR(200),
    description VARCHAR(500),
    input_cost_per_token FLOAT,
    output_cost_per_token FLOAT,
    max_tokens INTEGER,
    max_input_tokens INTEGER,
    context_window INTEGER,
    mode VARCHAR(50),
    source VARCHAR(50),
    discovered_at DATETIME,
    last_updated DATETIME,
    model_metadata JSON,
    FOREIGN KEY(provider_id) REFERENCES provider_registry(provider_id) ON DELETE CASCADE
);
CREATE INDEX ix_model_catalog_provider_id ON model_catalog(provider_id);
```

## Technical Decisions

### Decision 1: Renamed metadata Columns
**Context:** SQLAlchemy reserves `metadata` for Declarative API internals

**Options:**
- A. Use `metadata` anyway (would cause errors)
- B. Rename to `provider_metadata` / `model_metadata` (chosen)
- C. Use `meta` (less descriptive)

**Rationale:** Option B preserves clarity while avoiding SQLAlchemy conflicts. Explicit naming is better than abbreviations.

**Impact:** Migration updated, models use new column names.

---

### Decision 2: Singleton Pattern for Services
**Context:** ProviderRegistryService and ProviderAutoDiscovery need global access

**Options:**
- A. Pass instances through function calls (verbose)
- B. Global singleton with `get_xxx()` functions (chosen)
- C. Class methods only (harder to test)

**Rationale:** Option B balances convenience with testability (accepts optional `db_session` for testing).

**Impact:** Both services use singleton pattern with dependency injection support.
