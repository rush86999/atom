       ```yaml
         workflow_dispatch:  # Allow manual trigger
       ```

    2. **Job: Load Testing**:
       ```yaml
       jobs:
         load-tests:
           runs-on: ubuntu-latest
           steps:
             - uses: actions/checkout@v3
             - name: Set up Node.js
               uses: actions/setup-node@v3
               with:
                 node-version: '18'
             - name: Install k6
               run: npm install -g k6
             - name: Start backend server
               run: |
                 cd backend
                 python -m uvicorn main:app &
                 sleep 10
             - name: Run baseline load test
               run: |
                 cd backend/tests/load
                 k6 run test_api_load_baseline.js --out json=baseline-results.json
             - name: Run moderate load test
               run: |
                 cd backend/tests/load
                 k6 run test_api_load_moderate.js --out json=moderate-results.json
             - name: Upload load test results
               uses: actions/upload-artifact@v3
               with:
                 name: load-test-results
                 path: backend/tests/load/*-results.json
       ```

    3. **Job: Network Simulation**:
       - Run network tests (slow 3G, offline mode)
       - Use Playwright with network throttling
       - Upload test results as artifacts

    4. **Job: Memory Leak Detection**:
       - Run memory leak tests (CDP heap snapshots)
       - Upload heap snapshots and comparison results

    5. **Job: Result Aggregation**:
       - Download all artifacts
       - Run stress test scheduler to aggregate results
       - Generate summary report
       - Send alert if tests failed

    Use cron syntax for nightly scheduling (0 2 * * * = 2 AM UTC).
    Run on ubuntu-latest with Node.js 18 and Python 3.11.
    Upload artifacts for each test type (load results, screenshots, logs).

<task type="auto">
  <name>Task 2: Create weekly stress tests CI/CD workflow</name>
  <files>.github/workflows/weekly-stress-tests.yml</files>
  <action>
    Create `.github/workflows/weekly-stress-tests.yml`:

    1. **Workflow trigger**:
       ```yaml
       on:
         schedule:
           - cron: '0 3 * * 0'  # Run at 3 AM UTC every Sunday
         workflow_dispatch:  # Allow manual trigger
       ```

    2. **Job: All Stress Tests**:
       ```yaml
       jobs:
         comprehensive-stress-tests:
           runs-on: ubuntu-latest
           steps:
             - uses: actions/checkout@v3
             - name: Set up Python
               uses: actions/setup-python@v4
               with:
                 python-version: '3.11'
             - name: Install dependencies
               run: |
                 pip install -r backend/requirements.txt
                 pip install pytest pytest-playwright
                 playwright install chromium
             - name: Run all stress tests
               run: |
                 cd backend
                 pytest tests/load/ tests/e2e_ui/tests/test_network_*.py \
                        tests/e2e_ui/tests/test_memory_leak_detection.py \
                        tests/mobile_api/ tests/desktop/ \
                        frontend-nextjs/tests/visual/ frontend-nextjs/tests/accessibility/ \
                        -v --junitxml=stress-test-results.xml
             - name: Upload test results
               uses: actions/upload-artifact@v3
               with:
                 name: stress-test-results
                 path: backend/stress-test-results.xml
             - name: Aggregate results
               run: |
                 python backend/tests/stress/test_scheduler.py aggregate \
                   --results-file backend/stress-test-results.xml \
                   --output backend/stress-test-summary.json
             - name: Send summary report
               env:
                 SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
               run: |
                 python backend/tests/stress/test_scheduler.py report \
                   --summary-file backend/stress-test-summary.json \
                   --slack
             - name: File bugs for failures
               if: failure()
               env:
                 GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
                 GITHUB_REPOSITORY: ${{ github.repository }}
               run: |
                 python backend/tests/bug_discovery/file_bugs_from_artifacts.py
       ```

    Use cron syntax for weekly scheduling (0 3 * * 0 = 3 AM UTC every Sunday).
    Run all stress tests: load, network, memory, mobile, desktop, visual, a11y.
    Use pytest with JUnit XML output for result aggregation.
    Send Slack summary report (if SLACK_WEBHOOK_URL configured).
    Trigger automated bug filing on any test failure.
  </action>
  <verify>cat .github/workflows/weekly-stress-tests.yml | grep -E "(on:|schedule:|cron:|comprehensive-stress-tests|file_bugs)"</verify>
