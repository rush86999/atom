## Deviations from Plan

### Rule 1 - Bug Fix: Correct Mock Patch Path

**Found during:** Task 3 verification

**Issue:** test_lux_client_fallback_to_byok failed with AssertionError: expected call not found. Expected: get_api_key('lux'), Actual: get_api_key('qwen').

**Root Cause:** Mock patch path was 'core.lux_config.lux_config.get_anthropic_key', but byok_handler imports lux_config at module level. The patch needs to happen where the object is imported, not where it's defined.

**Fix:** Changed patch decorator from `@patch('core.lux_config.lux_config.get_anthropic_key')` to `@patch('core.llm.byok_handler.lux_config')`. Also updated assertion from `assert_called_with("lux")` to `assert_any_call("lux")` for robustness.

if "lux" in providers_config:
    # LUX uses Anthropic API key via lux_config or BYOK fallback
    api_key = lux_config.get_anthropic_key() or self.byok_manager.get_api_key("lux")
    if api_key:
        try:
            self.clients["lux"] = OpenAI(api_key=api_key)
            if AsyncOpenAI:
                self.async_clients["lux"] = AsyncOpenAI(api_key=api_key)
            logger.info("Initialized LUX provider with Anthropic client")
        except Exception as e:
            logger.error(f"Failed to initialize LUX client: {e}")
