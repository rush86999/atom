  <name>Task 1: Create centralized retry policy configuration</name>
  <files>backend/tests/scripts/retry_policy.py</files>
  <action>
Create retry_policy.py with:

1. RetryPolicy dataclass:
   @dataclass
   class RetryPolicy:
       max_retries: int = 3
       retry_delay: float = 1.0
       test_timeout: float = 30.0
       flaky_threshold: float = 0.3  # 30% failure rate
       detection_runs: int = 10
       min_stable_runs: int = 20
       auto_quarantine: bool = True
       warn_quarantine: bool = True
       platform_overrides: dict = None

       def get_policy_for_platform(self, platform: str) -> 'RetryPolicy':
           if self.platform_overrides and platform in self.platform_overrides:
               overrides = self.platform_overrides[platform]
               return RetryPolicy(**{**self.__dict__, **overrides})
           return self

2. DEFAULT_RETRY_POLICY instance:
   DEFAULT_RETRY_POLICY = RetryPolicy(
       max_retries=3,
       retry_delay=1.0,
       test_timeout=30.0,
       flaky_threshold=0.3,
       detection_runs=10,
       min_stable_runs=20,
       auto_quarantine=True,
       warn_quarantine=True,
       platform_overrides={
           "backend": {"test_timeout": 60.0, "max_retries": 2},
           "frontend": {"flaky_threshold": 0.2, "detection_runs": 15},
           "mobile": {"max_retries": 5, "retry_delay": 2.0},
           "desktop": {"max_retries": 2, "retry_delay": 0.5, "test_timeout": 15.0}
       }
   )

3. get_policy() function:
   - Load DEFAULT_RETRY_POLICY
   - Allow override via environment variables (FLAKY_THRESHOLD, MAX_RETRIES)
   - Return RetryPolicy instance

4. CLI for testing:

<task type="auto">
  <name>Task 2: Update unified-tests-parallel.yml with flaky detection</name>
  <files>.github/workflows/unified-tests-parallel.yml</files>
  <action>
Modify .github/workflows/unified-tests-parallel.yml to add:

1. After test execution steps (continue-on-error: true):
   - name: Run flaky detection (backend)
     if: matrix.platform == 'backend'
     working-directory: ./backend
     run: |
       python tests/scripts/flaky_test_detector.py \
         --platform backend \
         --runs 3 \
         --quarantine-db tests/coverage_reports/metrics/flaky_tests.db \
         --output tests/coverage_reports/metrics/backend_flaky_tests.json
     continue-on-error: true

   - name: Run flaky detection (frontend)
     if: matrix.platform == 'frontend'
     working-directory: ./frontend-nextjs
     run: |
       node scripts/jest-retry-wrapper.js \
         --platform frontend \
         --runs 3 \
         --output coverage/frontend_flaky_tests.json
     continue-on-error: true

   - name: Run flaky detection (mobile)
     if: matrix.platform == 'mobile'
     working-directory: ./mobile
     run: |
       node scripts/jest-retry-wrapper.js \
         --platform mobile \
         --runs 3 \
         --output test-results/mobile_flaky_tests.json
     continue-on-error: true

2. Aggregate flaky test reports (after all platform jobs):
   - name: Calculate reliability scores
     if: always()
     working-directory: ./backend
     run: |
       python tests/scripts/reliability_scorer.py \
         --backend-flaky ../results/backend/backend_flaky_tests.json \
         --frontend-flaky ../results/frontend/frontend_flaky_tests.json \
         --mobile-flaky ../results/mobile/mobile_flaky_tests.json \
         --quarantine-db tests/coverage_reports/metrics/flaky_tests.db \
         --output ../results/reliability_score.json
     continue-on-error: true

3. Upload artifacts:
   - name: Upload flaky test reports
     uses: actions/upload-artifact@v4
     if: always()
     with:
       name: flaky-test-reports
       path: |
         backend/tests/coverage_reports/metrics/*_flaky_tests.json
         backend/tests/coverage_reports/metrics/flaky_tests.db
         results/reliability_score.json
       retention-days: 30

4. PR comment (add to aggregation job):
   - name: Comment reliability score on PR
     if: github.event_name == 'pull_request'
     uses: actions/github-script@v7
     with:
       script: |
         const fs = require('fs');
         const reliability = JSON.parse(fs.readFileSync('backend/tests/coverage_reports/metrics/reliability_score.json', 'utf8'));
         const score = (reliability.overall_score * 100).toFixed(1);
         const delta = reliability.score_change || '';
         const icon = score >= 90 ? '🟢' : score >= 80 ? '🟡' : '🔴';
         const comment = `## Test Reliability Report\n\n${icon} Overall Score: ${score}% ${delta}\n\nPlatform Breakdown:\n- Backend: ${(reliability.platform_scores.backend * 100).toFixed(1)}%\n- Frontend: ${(reliability.platform_scores.frontend * 100).toFixed(1)}%\n- Mobile: ${(reliability.platform_scores.mobile * 100).toFixed(1)}%\n- Desktop: ${(reliability.platform_scores.desktop * 100).toFixed(1)}%\n\n[Full Report](https://github.com/${context.repo.owner}/${context.repo.repo}/actions/runs/${context.runId})`;
         github.rest.issues.createComment({ ...context.issue, body: comment });

DO NOT modify existing test execution steps, caching, or aggregation job structure.
  </action>

  <name>Task 3: Create flaky test quarantine documentation</name>
  <files>backend/tests/docs/FLAKY_TEST_QUARANTINE.md</files>
  <action>
Create backend/tests/docs/FLAKY_TEST_QUARANTINE.md with:

1. Overview section:
   - Purpose: Track and manage flaky tests across all platforms
   - Scope: Backend pytest, frontend Jest, mobile jest-expo, desktop cargo
   - Goal: Reduce flaky tests to <5% of total test suite

2. Quarantine workflow:
   - Detection: Multi-run verification (10 runs, 30% flaky threshold)
   - Recording: SQLite database (flaky_tests.db) with failure history
   - Classification: stable (0% failures), flaky (1-99% failures), broken (100% failures)
   - Auto-quarantine: Tests with flaky_rate > 0.3 automatically marked

3. Auto-removal policy:
   - Weekly cron job re-runs all quarantined tests
   - Auto-remove from quarantine if 20 consecutive passes
   - Send PR notification when test becomes stable
   - Maintain "fixed_flaky_tests" log for accountability

4. Retry configuration:
   - Reference retry_policy.py for platform-specific settings
   - Backend: 2 retries, 60s timeout (async tests)
   - Frontend: 3 retries, 30s timeout, 20% flaky threshold
   - Mobile: 5 retries, 2s delay (network flakiness)
   - Desktop: 2 retries, 0.5s delay (fast Rust tests)

5. Developer guide:
   - How to check if a test is quarantined: sqlite3 flaky_tests.db "SELECT * FROM flaky_tests WHERE test_path LIKE '%test_name%'"
   - How to fix a flaky test: Review failure_history JSON for patterns (timing, race conditions, shared state)
   - How to remove from quarantine: Fix underlying issue, wait for weekly auto-removal or manual UPDATE
   - How to verify fix: Run flaky_test_detector.py --runs 10 on fixed test

6. CI/CD integration:
   - unified-tests-parallel.yml runs detection after test execution
   - PR comments show reliability score with trend indicators
   - Artifact uploads preserve flaky test reports for 30 days
   - Aggregation job combines results across platforms

7. Troubleshooting:
   - False positive flaky detection: Increase detection_runs or check test isolation
   - Database bloat: Prune records older than 90 days
   - Slow detection: Use --runs 3 for quick check, --runs 10 for deep analysis

Reference existing FLAKY_TEST_GUIDE.md for consistency.
  </action>
