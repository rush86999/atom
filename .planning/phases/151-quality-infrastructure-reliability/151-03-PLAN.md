# Reliability scoring formula
# Individual test: reliability = 1.0 - flaky_rate
# Platform score: average of all test reliabilities
# Overall score: weighted average (backend 35%, frontend 40%, mobile 15%, desktop 10%)
# Reference: Phase 146 weighted coverage distribution (based on business impact)

<task type="auto">
  <name>Task 1: Create reliability scorer with platform aggregation</name>
  <files>backend/tests/scripts/reliability_scorer.py</files>
  <action>
Create reliability_scorer.py with:

1. calculate_reliability_score(flaky_rate: float) -> float:
   - Return max(0.0, 1.0 - flaky_rate)
   - Clamp to [0.0, 1.0] range
   - Handle edge cases (None, negative, >1.0)

2. calculate_platform_score(flaky_tests: List[Dict]) -> float:
   - Filter by platform (backend/frontend/mobile/desktop)
   - For each test: reliability = calculate_reliability_score(test.flaky_rate)
   - Return average of all reliabilities
   - Return 1.0 if no flaky tests (perfect reliability)

3. aggregate_platform_scores(platform_flaky_data: Dict) -> Dict:
   - Input: {
       "backend": [flaky test objects],
       "frontend": [flaky test objects],
       "mobile": [flaky test objects],
       "desktop": [flaky test objects]
     }
   - Calculate each platform score using calculate_platform_score
   - Apply weights from Phase 146 (backend: 0.35, frontend: 0.40, mobile: 0.15, desktop: 0.10)
   - Return: {
       "overall_score": weighted_average,
       "platform_scores": {backend: X, frontend: Y, mobile: Z, desktop: W},
       "weights_used": {backend: 0.35, frontend: 0.40, mobile: 0.15, desktop: 0.10}
     }

4. load_flaky_data(file_path: str, platform: str) -> List[Dict]:
   - Load JSON file from detector output
   - Add platform field to each test entry
   - Return list of flaky test objects

5. CLI interface:
   --backend-flaky: Path to backend flaky JSON
   --frontend-flaky: Path to frontend flaky JSON
   --mobile-flaky: Path to mobile flaky JSON
   --desktop-flaky: Path to desktop flaky JSON
   --quarantine-db: Path to flaky_tests.db (alternative to JSON files)
   --output: Path to write reliability_score.json

Reference ci_status_aggregator.py for aggregation pattern.
