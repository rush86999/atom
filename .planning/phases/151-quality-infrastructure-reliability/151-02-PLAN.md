<task type="auto">
  <name>Task 1: Create Jest retry wrapper for frontend</name>
  <files>frontend-nextjs/scripts/jest-retry-wrapper.js</files>
  <action>
Create frontend-nextjs/scripts/jest-retry-wrapper.js with:

1. CLI argument parsing (yargs or minimist):
   --testPattern: Jest test name pattern (required)
   --runs: Number of runs (default: 10)
   --output: JSON output path (default: frontend_flaky_tests.json)
   --platform: Platform name (default: "frontend")
   --verbose: Enable verbose logging

2. runTestMultipleTimes function:
   - Accept testPattern, runs, verbose
   - Loop N times: execSync(`jest --testNamePattern="${testPattern}" --passWithNoTests`)
   - Track failures array: [{run: 0, failed: true/false}]
   - Return: {failureCount, failures, flakyRate, classification}

3. classifyFlakiness function:
   - If failureCount === 0: return "stable"
   - If failureCount === runs: return "broken"
   - If 0 < failureCount < runs: return "flaky"

4. JSON export format (matching Python detector):
   {
     "scan_date": "ISO timestamp",
     "platform": "frontend",
     "test_pattern": "...",
     "total_runs": N,
     "failures": M,
     "flaky_rate": 0.3,
     "classification": "flaky",
     "failure_details": [{run: 0, failed: true}, ...]
   }

5. Error handling:
   - Catch execSync errors (test failures), treat as failed run
   - Handle timeout errors (30s default per run)
   - Exit codes: 0 (stable), 1 (flaky found), 2 (execution error)

Reference RESEARCH.md Pattern 1 (Jest multi-run example).
  </action>
