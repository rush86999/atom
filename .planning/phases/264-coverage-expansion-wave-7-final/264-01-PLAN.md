# Plan 264-01: Fix Import & Execution Issues

**Phase:** 264 - Coverage Expansion Wave 7 (FINAL)
**Plan:** 01 - Fix Import & Execution Issues
**Type:** Auto (Autonomous Execution)
**Duration:** 30-45 minutes
**Dependencies:** Phase 263 (Wave 6 complete)

---

## Objective

Fix import issues, schema mismatches, and async/await problems that are blocking test execution from previous coverage waves. Enable all ~854 existing tests to execute successfully.

---

## Context

From previous coverage waves (260-263), we created hundreds of tests but many cannot execute due to:

1. **Import Issues:** llm_service, CanvasTool, and other module imports failing
2. **Schema Mismatches:** Database models don't match test expectations
3. **Async Issues:** Missing async/await keywords in service tests
4. **Marker Issues:** Missing pytest markers (e.g., 'visual' marker)
5. **Syntax Errors:** Unterminated strings or invalid syntax

These issues prevent test collection and execution, blocking coverage measurement.

### Task 3: Fix Import Issues in Integration Tests
**Type:** auto
**Target:** Integration test files with import errors

**Known Issues:**
- `llm_service` import errors (module path changes)
- `CanvasTool` import errors (tool registry changes)
- `tool_registry` import errors (module restructuring)

**Steps:**
1. Find all import errors: `pytest --collect-only 2>&1 | grep -i "import"`
2. Fix import paths:
   - Update `from core.llm.llm_service import` to correct path
   - Update `from tools.canvas_tool import CanvasTool` to correct path
   - Update `from tools.registry import tool_registry` to correct path
3. Verify imports: `python3 -c "from tests.integration.test_xxx import"`
4. Re-run test collection

**Expected Outcome:** All import errors resolved, test modules load successfully

### Task 4: Fix Database Schema Mismatches
**Type:** auto
**Target:** Test files expecting schema that doesn't exist

**Known Issues:**
- `WorkflowBreakpoint` model missing
- `WorkflowDebugSession` model missing
- `AgentMaturity` enum mismatch
- Column name mismatches in models

**Steps:**
1. Identify schema errors from test failures

**Expected Outcome:** Tests run without schema-related errors

### Task 5: Fix Async/Await Issues
**Type:** auto
**Target:** Service tests with async issues

**Known Issues:**
- Missing `await` keywords for async calls
- Missing `@pytest.mark.asyncio` decorators
- Incorrect async fixture usage

**Steps:**
1. Find async errors: `pytest -v 2>&1 | grep -i "async"`
2. Fix async patterns:
   - Add `await` before async function calls
   - Add `@pytest.mark.asyncio` to async test functions
   - Use `async with` for async context managers
3. Verify: Run affected tests individually

**Expected Outcome:** Async tests execute without coroutine/syntax errors
