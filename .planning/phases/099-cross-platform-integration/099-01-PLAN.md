<objective>
Create cross-platform integration tests that verify shared workflows (authentication, agent execution, canvas presentations) work identically across web, mobile, and desktop platforms. This is the foundation for Phase 099, establishing the test patterns that mobile and desktop E2E tests will follow.

Purpose: Cross-platform consistency is critical for user experience — users should be able to switch between web, mobile, and desktop apps with identical functionality. These tests verify that the backend APIs support all platforms equally and that shared workflows produce consistent results.

Output: Cross-platform integration test suite with shared workflow tests, feature parity validation, and API consistency checks. These tests run on web (Playwright) during this plan, then serve as templates for mobile (Detox) and desktop (tauri-driver) E2E tests in subsequent plans.
</objective>
