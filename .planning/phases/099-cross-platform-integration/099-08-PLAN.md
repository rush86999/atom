# E2E Testing Guide

## Overview

Atom uses E2E (End-to-End) tests to verify complete user workflows across web, mobile, and desktop platforms. E2E tests run on merge to main (not on PRs) to avoid blocking development.

## Quick Start

### Web E2E Tests (Playwright)

```bash
# Install dependencies
cd backend
pip install -r requirements-testing.txt
playwright install chromium

# Run web E2E tests
pytest tests/e2e_ui/tests/ -v

# Run specific test file
pytest tests/e2e_ui/tests/cross-platform/test_shared_workflows.py -v
```

### Mobile E2E Tests (Detox)

```bash
cd mobile

# Build for testing
npm run e2e:build:ios

# Run mobile E2E tests
npm run e2e:test:ios
```

**Note:** Detox requires Expo dev client. See Plan 02 SUMMARY for setup details.

### Desktop E2E Tests (WebDriverIO)

```bash
cd frontend-nextjs

# Start Tauri dev server
npm run tauri dev &

# Run desktop E2E tests
npm run test:e2e
```

**Note:** tauri-driver provides WebDriver support. See Plan 03 SUMMARY for setup details.

## Test Structure

### Cross-Platform Tests

All platforms test the same shared workflows:

| Workflow | Web | Mobile | Desktop |
|----------|-----|--------|---------|
| Authentication | ✅ | ✅ | ✅ |
| Agent Execution | ✅ | ✅ | ✅ |
| Canvas Presentations | ✅ | ✅ | ✅ |
| Skill Execution | ✅ | ✅ | ✅ |
| Feature Parity | ✅ | ✅ | ✅ |

### File Locations

- **Web:** `backend/tests/e2e_ui/tests/cross-platform/`
- **Mobile:** `mobile/e2e/`
- **Desktop:** `frontend-nextjs/wdio/specs/`

## CI/CD Integration

### When E2E Tests Run

- **On push to main:** All platform E2E tests run in parallel
- **On PR:** E2E tests do NOT run (only unit/integration tests)
- **Manual trigger:** Available via `workflow_dispatch`

### Workflows

- `e2e-ui-tests.yml`: Web E2E (Playwright)
- `e2e-mobile.yml`: Mobile E2E (Detox)
- `e2e-desktop.yml`: Desktop E2E (WebDriverIO)
- `e2e-unified.yml`: Aggregates all platform results
- `lighthouse-ci.yml`: Performance regression tests

### Results

E2E test results are uploaded as GitHub Actions artifacts:
- `e2e-test-report`: Web HTML report
- `e2e-mobile-artifacts`: Mobile test artifacts
- `e2e-desktop-artifacts`: Desktop test artifacts
- `e2e-unified-results`: Aggregated results

## Writing E2E Tests

### Pattern: Shared Workflow

When adding a new shared workflow:

1. Implement on web first (Playwright)
2. Mirror on mobile (Detox)
3. Mirror on desktop (WebDriverIO)
4. Update feature parity tests

Example:

```python
# Web: backend/tests/e2e_ui/tests/cross-platform/test_shared_workflows.py
def test_new_workflow(self, browser, authenticated_user):
    workflow_page = SharedWorkflowPage(browser.new_page())
    workflow_page.navigate()
    workflow_page.do_action()
    assert workflow_page.is_complete()
```

```typescript
// Mobile: mobile/e2e/workflows/newWorkflow.e2e.ts
it('should complete new workflow', async () => {
  await element(by.id('new-workflow-screen')).tap();
  await element(by.id('do-action-button')).tap();
  await expect(element(by.id('complete-indicator'))).toBeVisible();
});
```

```typescript
// Desktop: frontend-nextjs/wdio/specs/workflows/newWorkflow.e2e.ts
it('should complete new workflow', async () => {
  await driver.navigate('/new-workflow');
  await driver.findByTestId('do-action-button').click();
  await driver.waitForVisible('complete-indicator');
});
```

### Selectors

Use `data-testid` attributes for selectors (resilient to CSS/class changes):

```python
# Good: Resilient
page.get_by_test_id("login-submit-button").click()

# Bad: Fragile
page.locator(".btn.btn-primary").click()
```

## Performance Testing

### Lighthouse CI

Performance audits run on every PR to detect regressions:

```bash
# Run locally
cd frontend-nextjs
npm run lighthouse:collect
npm run lighthouse:assert
```

### Budgets

- Performance score: > 90
- First Contentful Paint: < 2s
- Time to Interactive: < 5s
- Cumulative Layout Shift: < 0.1
- Total Blocking Time: < 300ms

See `frontend-nextjs/LIGHTHOUSE.md` for details.

## Troubleshooting

### Flaky E2E Tests

If E2E tests fail intermittently:

1. Check for timing issues: Use explicit waits, not hard-coded sleeps
2. Verify test data: Ensure tests create their own data
3. Check for race conditions: Use `waitFor*` methods
4. Review CI logs: Check for network/timeout issues

### Mobile Tests Failing

If Detox tests fail:

1. Verify Expo dev client is installed
2. Check iOS simulator is available
3. Rebuild app: `npm run e2e:build:ios`
4. Check Plan 02 SUMMARY for known issues

### Desktop Tests Failing

If WebDriverIO tests fail:

1. Verify Tauri dev server is running
2. Check tauri-driver is installed
3. Verify port 1430 is available
4. Check Plan 03 SUMMARY for known issues

## Maintenance

### Adding New E2E Tests

1. Write test on web platform first
2. Verify test passes locally
3. Mirror test to mobile/desktop
4. Update documentation
5. Commit with pattern: `test(e2e): add {workflow} E2E test`

### Updating E2E Dependencies

- Playwright: `backend/requirements-testing.txt`
- Detox: `mobile/package.json`
- WebDriverIO: `frontend-nextjs/package.json`

Update dependencies quarterly or when major releases include breaking changes.
