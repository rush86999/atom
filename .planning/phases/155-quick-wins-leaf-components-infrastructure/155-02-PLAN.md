   - test_validate_phone_valid: Accepts valid phone numbers (1234567890)
   - test_validate_phone_invalid_length: Rejects invalid lengths
   - test_validate_phone_international: Validates international formats (+1...)
   - test_validate_phone_formatting: Validates formatted numbers ((123) 456-7890)

4. **UUID validators**:
   - test_validate_uuid_valid: Accepts valid UUID v4 format
   - test_validate_uuid_invalid_format: Rejects malformed UUIDs
   - test_validate_uuid_version: Validates UUID version (v4 vs v1)

5. **General validators**:
   - test_validate_boolean: Boolean string validation ("true"/"false")
   - test_validate_integer: Integer validation with ranges
   - test_validate_float: Float validation with precision
   - test_validate_json: JSON string validation

6. **Edge cases** (use pytest.mark.parametrize):
   - Empty strings, null values
   - Boundary values (min/max lengths)
   - Special characters (Unicode, emojis)
   - Malformed inputs (typos, truncated)

Pattern example:
```python
@pytest.mark.parametrize("invalid_email", [
    "plainaddress",
    "@missingusername.com",
    "username@.com",
    "username@com",
    "",
    None,
])
def test_validate_email_invalid(self, invalid_email):
    with pytest.raises(ValidationError):
        validate_email(invalid_email)
```

Test validation logic - both valid and invalid inputs, with clear error messages.
