Create comprehensive tests for Button component using React Testing Library:

1. **Rendering tests**:
   - Renders default button with correct classes (bg-primary-600)
   - Renders destructive variant (bg-red-500)
   - Renders outline variant (border)
   - Renders ghost variant (transparent background)

   - Renders default size (h-10)
   - Renders sm size (h-9)
   - Renders lg size (h-11)
   - Renders icon size (h-10 w-10, square)

3. **User interaction tests**:
   - Calls onClick handler when clicked (use userEvent.click())
   - Does not call onClick when disabled
   - Prevents default behavior for form submissions

4. **Accessibility tests**:
   - Has accessible name (getByRole with name)
   - Forwards ARIA attributes (aria-label, aria-describedby)
   - Has correct button role
   - Keyboard navigation (Enter key triggers click)

5. **Edge cases**:
   - Renders with children (text, icons)
   - Renders with asChild for polymorphic behavior
   - Handles loading state (if spinner/disable-on-click)
   - Renders with custom className

Pattern from research:
```tsx
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { Button } from '../button';

describe('Button Component', () => {
  it('renders default button', () => {
    render(<Button>Click me</Button>);
    const button = screen.getByRole('button', { name: 'Click me' });
    expect(button).toBeInTheDocument();
    expect(button).toHaveClass('bg-primary-600');
  });

  it('calls onClick when clicked', async () => {
    const user = userEvent.setup();
    const handleClick = jest.fn();

    render(<Button onClick={handleClick}>Click me</Button>);

    const button = screen.getByRole('button');
    await user.click(button);

    expect(handleClick).toHaveBeenCalledTimes(1);
  });
});
```

Focus on USER BEHAVIOR, not component internals. Use getByRole, getByLabelText, not getByTestId.

   - Renders with placeholder text
   - Renders with default value
   - Renders with disabled state

2. **User interaction tests**:
   - Allows typing (use userEvent.type())
   - Calls onChange handler with value
   - Prevents input when disabled
   - Handles focus/blur events

3. **Validation tests**:
   - Renders with error state (error styling)
   - Shows error message when invalid
   - Enforces required attribute
   - Enforces min/max/step for number inputs
   - Enforces pattern validation

4. **Accessibility tests**:
   - Has accessible label (getByLabelText)
   - Forwards ARIA attributes (aria-invalid, aria-describedby)
   - Has correct role (textbox or specific input type)
   - Keyboard navigation (Tab, Shift+Tab)

5. **Edge cases**:
   - Handles empty input
   - Handles long input (truncation if applicable)
   - Handles special characters (Unicode, emojis)
   - Handles controlled vs uncontrolled state

Pattern example:
```tsx
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { Input } from '../input';

describe('Input Component', () => {
  it('renders input with label', () => {
    render(<Input label="Email" placeholder="user@example.com" />);
    const input = screen.getByLabelText('Email');
    expect(input).toBeInTheDocument();
    expect(input).toHaveAttribute('placeholder', 'user@example.com');
  });

  it('calls onChange when typing', async () => {
    const user = userEvent.setup();
    const handleChange = jest.fn();

    render(<Input label="Name" onChange={handleChange} />);

    const input = screen.getByLabelText('Name');
    await user.type(input, 'John Doe');

    expect(handleChange).toHaveBeenCalled();
  });
});
```

Test user interactions (typing, focus, blur), not component state.

import { render, screen } from '@testing-library/react';
import { Badge } from '../badge';

describe('Badge Component', () => {

    const badge = screen.getByText('New');
    expect(badge).toBeInTheDocument();
    expect(badge).toHaveClass('bg-primary');

    const badge = screen.getByText('Error');

  it('renders notification count', () => {
    render(<Badge>99+</Badge>);
    const badge = screen.getByText('99+');
    expect(badge).toBeInTheDocument();
  });
});
