<task type="auto">
  <name>Create unified test orchestrator workflow</name>
  <files>.github/workflows/unified-tests.yml</files>
  <action>
    Create GitHub Actions workflow file with:

    1. **Workflow triggers**:
       - on.push: branches [main, develop]
       - on.pull_request: branches [main, develop]
       - workflow_dispatch for manual runs

    2. **Three parallel jobs** (no dependencies between them):
       - backend-test: runs-on ubuntu-latest, timeout 30min
         - Checkout, setup Python 3.11, install dependencies
         - Run pytest with coverage JSON output
         - Upload artifact: backend-coverage (backend/tests/coverage_reports/metrics/coverage.json)

       - frontend-test: runs-on ubuntu-latest, timeout 15min
         - Checkout, setup Node 20, install dependencies
         - Run npm run test:ci
         - Upload artifact: frontend-coverage (frontend-nextjs/coverage/coverage-final.json)

       - type-check: runs-on ubuntu-latest, timeout 10min
         - Run TypeScript type check on frontend
         - Run lint check

    3. **Aggregate coverage job** (depends on all test jobs):
       - needs: [backend-test, frontend-test]
       - runs-on ubuntu-latest
       - Download artifacts: actions/download-artifact@v4 for backend-coverage and frontend-coverage
       - Run aggregator: python backend/tests/scripts/aggregate_coverage.py --format json --output unified/coverage.json
       - Upload unified artifact: actions/upload-artifact@v4 with name: unified-coverage, path: backend/tests/scripts/coverage_reports/unified/
       - Check quality gate: overall.coverage_pct >= 80%

    4. **PR comment on failure**:
       - If pass_rate < 98% or coverage < 80%: create PR comment with breakdown
       - Show platform-specific coverage and pass rates
       - Include remediation steps

    Use matrix strategy if more platforms added in future phases.
  </action>
  <verify>cat .github/workflows/unified-tests.yml | grep -E "(jobs|needs:|aggregate_coverage|quality-gate)"</verify>
  <done>unified-tests.yml workflow exists with parallel backend/frontend jobs and aggregation job</done>
