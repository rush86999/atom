### Task 5: Document Common Pitfalls and Solutions

**Status:** pending
**Estimated Time:** 2-3 hours
**File to Update:** `backend/docs/TDD_WORKFLOW.md`

**Pitfalls to Document:**

1. **Writing Tests After Code**
   - Problem: Tests become verification, not specification
   - Solution: Always write test first
   - Example from Phase 256 (tests not passing)

2. **Testing Implementation Details**
   - Problem: Fragile tests, hard to refactor
   - Solution: Test behavior, not implementation
   - Example from Phase 256 (component tests)

3. **Skipping the Refactor Phase**
   - Problem: Code quality degrades
   - Solution: Always refactor after green
   - Example from Phase 250 (test quality improvements)

4. **Writing Too Many Tests at Once**
   - Problem: Overwhelming, hard to debug
   - Solution: One test at a time, small batches
   - Example from Phase 256 (839 tests created)

5. **Ignoring Failing Tests**
   - Problem: Test suite loses trust
   - Solution: Fix immediately or mark as skip
   - Example from Phase 250 (fixing 1,504 failures)

6. **Not Running Tests Frequently**
   - Problem: Failures accumulate
   - Solution: Run tests after every change
   - Example from Phase 249 (rapid iteration)

**Content Requirements:**
- Each pitfall explained
- Real examples from Atom codebase
- Concrete solutions provided
- Prevention strategies

**Acceptance Criteria:**
- [ ] 6+ pitfalls documented
- [ ] All pitfalls have real examples
- [ ] All pitfalls have solutions
- [ ] All pitfalls have prevention strategies
- [ ] Clear, actionable advice
