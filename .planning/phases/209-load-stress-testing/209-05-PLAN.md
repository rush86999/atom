1. run_load_tests.sh: Execute Locust with configurable users, spawn rate, duration
2. run_soak_tests.sh: Run pytest with soak marks, extended timeouts
3. generate_load_report.py: Convert Locust JSON to HTML report
4. compare_performance.py: Compare current results to baseline, fail on regression

# Locust Command Pattern
```bash
locust -f tests/load/locustfile.py \
  --headless \
  -u 100 \          # users
  -r 10 \           # spawn rate (users/sec)
  -t 5m \           # duration
  --html report.html \
  --json results.json
```

# Performance Regression Detection
- Compare P95 response times
- Compare requests/second throughput
- Threshold: 15% regression = fail
- Exit code 1 on regression

2. Function `load_results(filepath)` returning parsed JSON
3. Function `compare_metrics(baseline, current, threshold)`:
   - Compare P95 response times for each endpoint
   - Compare RPS (requests per second)
   - Calculate % change
   - Return list of regressions (exceeding threshold)
4. Regression detection logic:
```python
if baseline_p95 > 0:
    p95_change = ((current_p95 - baseline_p95) / baseline_p95) * 100
    if p95_change > threshold:
        regressions.append({
            'endpoint': name,
            'metric': 'p95_response_time',
            'change_percent': p95_change
        })
```
5. Main function:
   - Parse args: baseline.json, current.json, --threshold (default 15)
   - Load both files
   - Compare metrics
   - Print results with icons (checkmark / X)
   - Exit code 1 if regressions found
   - Exit code 0 if no regressions

6. Output format:
```
✅ No performance regressions detected (threshold: 15%)
```
or
```
❌ PERFORMANCE REGRESSION DETECTED (3 regressions)
🔴 /api/v1/agents - P95: +25%
```
