# Plan 307-07: Complete Remaining Test Files with PostgreSQL Infrastructure

## Objective

Set up PostgreSQL test infrastructure and complete the remaining 5 test files from Phase 307 to achieve 80%+ coverage for critical backend files. This plan addresses the PostgreSQL dependency blockers identified in Plan 307-06.

## Purpose

Phase 307-06 created comprehensive test structure but encountered PostgreSQL compatibility issues:
- `EntityTypeDefinition` model uses `JSONBColumn` (PostgreSQL-specific)
- `GraphRAGEngine` uses recursive CTEs (PostgreSQL-specific)
- 5 test files incomplete due to infrastructure gaps

This plan creates the necessary PostgreSQL test infrastructure and completes all remaining test files to achieve the 80% backend coverage target.

## Context

**Current State (from 307-06-SUMMARY.md):**
- 9 test files created (6,000+ lines, 250+ test functions)
- Coverage: ~48% backend (target: 50%+, gap: 2pp)
- 5 files incomplete due to PostgreSQL dependencies:
  1. `test_graphrag_engine.py` - Recursive CTEs require PostgreSQL
  2. `test_intent_classifier.py` - LLM integration requires test database
  3. `test_atom_meta_agent.py` - Complex orchestration requires full stack
  4. `test_fleet_admiral.py` - Multi-agent coordination requires test infrastructure
  5. `test_agent_governance_routes.py` - API integration requires test database

**Infrastructure Available:**
- `docker-compose.prod.yml` already has PostgreSQL 15 configuration
- `.env` has `DATABASE_URL=postgresql://atom:atom_password@127.0.0.1:5433/atom_db`
- `pytest.ini` configured for unit tests
- `conftest.py` exists for test fixtures

## Risks and Mitigations

**Risk 1: PostgreSQL Setup Complexity**
- **Mitigation**: Use existing docker-compose.prod.yml as template
- **Fallback**: Use PostgreSQL Docker container directly

**Risk 2: Test Database Performance**
- **Mitigation**: Connection pooling, test isolation
- **Fallback**: Run tests sequentially if too slow

**Risk 3: Schema Migration Failures**
- **Mitigation**: Use alembic upgrade head in fixture setup
- **Fallback**: Manual schema creation if migrations fail

**Risk 4: LLM Mocking Complexity**
- **Mitigation**: Use AsyncMock for LLM service responses
- **Fallback**: Integration tests can skip LLM-dependent tests
