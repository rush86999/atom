<objective>
Assign business impact scores to all source files for coverage prioritization.

**Purpose**: Not all uncovered lines are equally important. A governance bug is 10x more critical than a UI utility bug. Business impact scoring weights coverage gaps by business criticality so we prioritize testing where it matters most. This enables the COVR-02 requirement: "rank files by (lines * impact / current_coverage)".

**Output**: Python script that scores files based on keyword patterns (critical/high/medium/low), produces JSON mapping files to scores, and documents the methodology.
</objective>
