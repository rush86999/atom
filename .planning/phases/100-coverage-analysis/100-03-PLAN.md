<task type="auto">
  <name>Create prioritization script with impact-weighted scoring</name>
  <files>backend/tests/scripts/prioritize_high_impact_files.py</files>
  <action>
    Create Python script implementing priority ranking:

    1. **calculate_priority_score(file_data, impact_score)** function:
       ```python
       priority_score = (uncovered_lines * impact_score) / (current_coverage_pct + 1)
       ```
       - Add +1 to avoid division by zero for 0% coverage files
       - Higher score = higher priority (more impact per test)
       - Example: 100 uncovered lines * 10 impact / (20% + 1) = 47.6

    2. **rank_files(baseline_data, impact_scores)** function:
       - Merge coverage baseline with impact scores by filepath
       - Calculate priority_score for each file below 80%
       - Sort by priority_score descending
       - Return ranked list with: file, coverage_pct, uncovered_lines, impact_score, priority_score, tier

    3. **estimate_tests_needed(file_data)** function:
       - Target: 50% coverage (proven sustainable from Phase 8.6)
       - Lines_to_cover = uncovered_lines * 0.5
       - Estimated_tests = max(10, lines_to_cover / 20) (20 lines per test average)
       - Return integer estimate

    4. **group_by_phase(ranked_files)** function:
       - Phase 101 (Backend Core): Top 20 Critical + High tier files
       - Phase 102 (Backend API): Next 15 High + Medium tier API files
       - Phase 103 (Backend Property): Files with complex logic (engine, handler, coordinator)
       - Phase 104 (Backend Error): Files with error handling focus
       - Phase 105-109: Frontend files (from frontend coverage when available)
       - Return dict with phase assignments

    5. **identify_quick_wins(ranked_files)** function:
       - Filter: 0% coverage AND (Critical or High tier)
       - Sort by uncovered_lines descending
       - Return top 20 quick wins

    6. **write_json_report()** - Save to prioritized_files_v5.0.json
    7. **write_markdown_report()** - Generate human-readable report

    Reference HIGH_IMPACT_FILES.md for priority_score formula examples.
  </action>
