    2. **identify_below_threshold()** - Find all files below 80% coverage

       - Sort by uncovered_lines descending (largest gaps first)

    3. **generate_baseline_metrics()** - Compute baseline statistics
       - Overall: percent_covered, covered_lines, total_lines, coverage_gap
       - By module: core, api, tools breakdown
       - Threshold analysis: files_below_80, files_below_50, files_below_20
       - Distribution: count of files in each coverage tier (0-20%, 21-50%, 51-80%, 80%+)

    4. **write_json_report()** - Save baseline metrics to JSON
       - Output: backend/tests/coverage_reports/metrics/coverage_baseline.json
       - Include: timestamp, overall, modules, files_below_threshold, distribution
       - Use datetime.utcnow().isoformat() + 'Z' for UTC timestamp

    5. **write_markdown_report()** - Generate human-readable report
       - Output: backend/tests/coverage_reports/COVERAGE_BASELINE_v5.0.md
       - Sections: Executive Summary, Module Breakdown, Files Below 80%, Distribution
       - Include table of top 50 files by uncovered lines

    6. **main()** - Orchestrate report generation
       - Parse command line args: --coverage-file (default), --output-dir, --threshold
       - Call all generation functions
       - Print summary statistics to stdout
