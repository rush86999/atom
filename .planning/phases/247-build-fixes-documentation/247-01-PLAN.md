# Known Issues from Investigation

The file `backend/integrations/asana_service.py` has the following syntax errors:

1. **Line 127-147**: Circuit breaker check opens `try:` block but never closes it
2. **Line 148**: New `try:` block starts before previous one closed
3. **Line 177-196**: Same pattern - circuit breaker `try:` without `except:`
4. **Line 197**: New `try:` block before previous closed
5. **Line 418-437**: Circuit breaker `try:` without `except:`
6. **Line 438**: New `try:` block before previous closed
7. **Lines 635-656**: Orphaned code after module-level `asana_service = AsanaService()` assignment

The pattern is: circuit breaker checks were added with `try:` but missing the corresponding `except:` block. The actual API call has its own `try:` which creates the syntax error.

# Fix Strategy

Remove the orphaned `try:` statements before the API call `try:` blocks. The circuit breaker/rate limiter checks should not have their own try block - they should be part of the same try block as the API call, or use if statements without try.

**Correct structure:**
```python
# Start audit logging
audit_ctx = log_integration_attempt("asana", "method_name", locals())
try:
    # Check circuit breaker
    if not await circuit_breaker.is_enabled("asana"):
        logger.warning(f"Circuit breaker is open for asana")
        log_integration_complete(audit_ctx, error=Exception("Circuit breaker open"))
        raise HTTPException(status_code=503, detail="Asana integration temporarily disabled")

    # Check rate limiter
    is_limited, remaining = await rate_limiter.is_rate_limited("asana")
    if is_limited:
        logger.warning(f"Rate limit exceeded for asana")
        log_integration_complete(audit_ctx, error=Exception("Rate limit exceeded"))
        raise HTTPException(status_code=429, detail="Rate limit exceeded for asana")

    # Actual API call
    result = self._make_request(...)

    return {...}
except Exception as e:
    logger.error(f"Failed to ...: {e}")
    log_integration_complete(audit_ctx, error=e)
    return {"ok": False, "error": str(e)}
```
