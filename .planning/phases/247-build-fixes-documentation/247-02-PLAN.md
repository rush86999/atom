# Known Issues from Investigation

1. **SWC Version Mismatch**: `@next/swc-darwin-x64` is version 15.5.7 but Next.js is 15.5.11
   - Warning: "Mismatching @next/swc version, detected: 15.5.7 while Next.js is on 15.5.11"
   - This mismatch can cause minification errors

2. **Minification Error**: During build, SWC minifier produces corrupted code:
   ```
   ReferenceError: erator is not defined
   at <unknown> (.next/server/pages/automations.js:1:43648)
   ```
   - "erator" is likely a corrupted "operator" variable name
   - This happens during page data collection for /automations

3. **Build Command**: `npm run build` in `frontend-nextjs/` directory

# Fix Strategy

The version mismatch needs to be resolved. The SWC package version should match Next.js version exactly.

Steps:
1. Check current package versions: `npm list next @next/swc-darwin-x64`
2. Update @next/swc packages to match Next.js version
3. Clean build artifacts: `rm -rf .next node_modules/.cache`
4. Rebuild: `npm run build`
5. If build still fails, investigate the automations page for code patterns that might cause minification issues

# Note on Architecture

The system is darwin/x64 (Intel Mac), so the correct package is `@next/swc-darwin-x64`, not the ARM variant.
