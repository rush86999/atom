## Migration Patterns Established

### Pattern 1: Simple Migration (event_sourced_architecture.py)
```python
# OLD
from core.llm.byok_handler import BYOKHandler
handler = BYOKHandler()
response = await handler.generate_response(prompt=..., **kwargs)

# NEW
from core.llm_service import get_llm_service
service = get_llm_service()
response = await service.generate(prompt=..., **kwargs)
```

### Pattern 2: Infrastructure Layer with Tool Calling (workflow_engine.py)
```python
# OLD
from core.llm.byok_handler import BYOKHandler
llm_handler = BYOKHandler(self.db)
response = await llm_handler.chat_completion(messages=..., tools=...)

# NEW
from core.llm_service import get_llm_service
llm_service = get_llm_service(db=self.db)
response = await llm_service.handler.chat_completion(messages=..., tools=...)
```

**Rationale:** Infrastructure layer can access LLMService.handler directly for advanced features (tool calling, function calling) that aren't exposed in the LLMService API yet.
