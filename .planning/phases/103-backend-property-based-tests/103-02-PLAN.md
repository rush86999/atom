**Test Structure (TDD RED-GREEN-REFACTOR):**

A. Filtered Retrieval Invariants (max_examples=200 - CRITICAL):
   - test_filter_by_agent_id
     Invariant: Episodes filtered by agent_id return only that agent's episodes
     Strategy: st.lists of episodes with different agent_ids
     All returned episodes have agent_id == filter_value

   - test_filter_by_workspace_id
     Invariant: Episodes filtered by workspace_id respect workspace isolation
     Strategy: st.lists of episodes across multiple workspaces
     Workspace A filter returns only Workspace A episodes

   - test_filter_by_time_range
     Invariant: Time range filter returns episodes within bounds (inclusive)
     Strategy: st.tuples(start_time, end_time, episode_timestamps)
     Retrieved episodes: start <= episode.started_at <= end

   - test_filter_combination_consistent
     Invariant: Multiple filters applied together are consistent (AND logic)
     Strategy: st.dictionaries with filter combinations
     All returned episodes match ALL filter conditions

   - test_empty_filter_returns_all
     Invariant: Empty filters (no constraints) return all accessible episodes
     Strategy: st.lists of episodes with empty filter dict
     Empty filter = return all (subject to pagination)

B. Semantic Retrieval Invariants (max_examples=100 - STANDARD):
   - test_similarity_scores_descending
     Invariant: Semantic retrieval returns results sorted by decreasing similarity
     Strategy: st.lists of (episode, similarity_score) tuples
     similarity[i] >= similarity[i+1] for all i

   - test_similarity_scores_normalized
     Invariant: Similarity scores are in [0.0, 1.0] range
     Strategy: st.lists of similarity calculations
     0.0 <= score <= 1.0 for all results

   - test_semantic_query_idempotent
     Invariant: Same semantic query returns same results (deterministic)
     Strategy: st.tuples(query_text, embedding_model)
     Same query executed 10x = identical ranked results

   - test_vector_search_closest_neighbors
     Invariant: Vector search returns episodes with highest cosine similarity
     Strategy: st.lists of (query_vector, episode_vectors) tuples
     Top K results are K nearest neighbors (by cosine similarity)

C. Hybrid Retrieval Invariants (PostgreSQL + LanceDB) (max_examples=100 - STANDARD):
   - test_hybrid_retrieval_consistent
     Invariant: Hybrid retrieval (hot + cold) returns consistent results
     Strategy: st.tuples(hot_limit, cold_limit, total_episodes)
     Results from hot + cold = consistent with single query

   - test_hierarchical_storage_migration
     Invariant: Episodes migrate from hot (PostgreSQL) to cold (LanceDB) correctly
     Strategy: st.lists of episodes with varying ages
     Old episodes accessible from cold storage, young from hot

   - test_retrieval_fallback_on_cold_failure
     Invariant: If cold storage unavailable, fallback to hot only (graceful degradation)
     Strategy: st.tuples(cold_storage_available, hot_storage_available)
     Cold unavailable = return hot results only (no crash)

**Test Structure (TDD RED-GREEN-REFACTOR):**

A. Memory Consolidation Invariants (max_examples=200 - CRITICAL):
   - test_consolidation_reduces_segment_count
     Invariant: Consolidation reduces or maintains segment count (never increases)
     Strategy: st.lists of segments with varying gaps
     segments_consolidated <= segments_original

   - test_consolidation_preserves_content
     Invariant: Consolidated episode preserves all original content
     Strategy: st.lists of (segment_content, timestamp) tuples
     All unique content from original segments in consolidated result

   - test_consolidation_merges_adjacent_segments
     Invariant: Adjacent segments below threshold are merged
     Strategy: st.lists of segments with small gaps
     Segments with gap < merge_threshold merged into single segment

   - test_consolidation_respects_chronological_order
     Invariant: Consolidated segments maintain chronological order
     Strategy: st.lists of segments with timestamps
     Merged segments maintain time ordering (first to last)

B. Episode Archival Invariants (max_examples=100 - STANDARD):
   - test_archived_episodes_read_only
     Invariant: Archived episodes cannot be modified (immutable)
     Strategy: st.sampled_from(episode_states)
     If state == "archived": modifications blocked

   - test_archival_age_threshold
     Invariant: Episodes archived only after age threshold (e.g., 30 days)
     Strategy: st.lists of episodes with varying ages
     Episodes younger than threshold remain active

   - test_archival_preserves_access_count
     Invariant: Archival preserves access_count for analytics
     Strategy: st.integers for access counts (0 to 1000)
     access_count_archived == access_count_pre_archive

   - test_archived_episodes_retrievable
     Invariant: Archived episodes remain retrievable (read access)
     Strategy: st.lists of episodes (active and archived)
     Archived episodes appear in historical queries

C. Decay Calculation Invariants (max_examples=100 - STANDARD):
   - test_decay_score_non_negative
     Invariant: Decay score always in [0.0, 1.0]
     Strategy: st.integers for episode age in days (0 to 365)
     0.0 <= decay_score <= 1.0 for any age

   - test_decay_monotonic_decrease
     Invariant: Decay score decreases (or stays same) as episode ages
     Strategy: st.lists of (age_1, age_2) where age_2 > age_1
     If age increases: decay(age_2) <= decay(age_1)

   - test_decay_never_negative
     Invariant: Decay calculation never produces negative values
     Strategy: st.integers for episode age (0 to 1000 days)
     decay_score >= 0 for all valid ages

   - test_decay_formula_consistent
     Invariant: Decay formula is consistent across multiple calculations
     Strategy: st.lists of episode_ages
     Same age calculated 10x = identical decay_score
