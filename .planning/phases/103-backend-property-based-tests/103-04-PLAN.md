Create backend/tests/property_tests/STRATEGIC_MAX_EXAMPLES_GUIDE.md:

**Document Structure:**

1. Overview Section
   - Purpose: Guide for choosing max_examples in Hypothesis property tests
   - Research findings from Phase 100 (200 examples ~10 hours vs 50 ~30 minutes)
   - Diminishing returns: 95%+ bugs found with 50-100 examples

2. Criticality Categories
   - **CRITICAL (max_examples=200)**: State machines, financial calculations, security boundaries
   - **STANDARD (max_examples=100)**: Business logic, data transformations, validation rules
   - **IO_BOUND (max_examples=50)**: Database queries, API calls, file operations

3. Selection Criteria
   - Use CRITICAL when:
     - Invariant relates to money (financial calculations)
     - Invariant relates to security (auth, permissions, governance)
     - Invariant is a state machine transition (maturity levels, lifecycle states)
     - Bug would cause data corruption or loss

   - Use STANDARD when:
     - Invariant is business logic (permissions, retrieval, categorization)
     - Invariant involves data transformation (formatting, serialization)
     - Invariant has clear input/output mapping

   - Use IO_BOUND when:
     - Test involves database queries (each example = DB roundtrip)
     - Test involves file I/O (each example = file read/write)
     - Test involves network calls (each example = HTTP request)

4. Execution Time Targets
   - CRITICAL property test: <30 seconds for 200 examples
   - STANDARD property test: <15 seconds for 100 examples
   - IO_BOUND property test: <10 seconds for 50 examples
   - Full suite execution: <5 minutes for all property tests

5. Example Configurations
   ```python
   # CRITICAL: Financial precision
   @given(amount=money_strategy('0.01', '10000.00'))
   @settings(max_examples=200, suppress_health_check=[...])
   def test_money_preserves_precision(self, amount):
       # Financial calculations need extensive coverage
       ...

   # STANDARD: Permission checks
   @given(maturity=sampled_from(MATURITY_LEVELS))
   @settings(max_examples=100)
   def test_permission_deterministic(self, maturity):
       # Business logic, moderate coverage sufficient
       ...

   # IO_BOUND: Database queries
   @given(agent_count=integers(min_value=1, max_value=20))
   @settings(max_examples=50)
   def test_cache_lookup_performance(self, db_session, agent_count):
       # DB-bound, fewer examples to keep test fast
       ...
   ```

6. Trade-off Analysis
   - Higher max_examples = more bug finding BUT longer execution
   - 200 examples finds 5% more bugs than 100 examples but takes 2x time
   - 50 examples catches 95%+ of bugs for most properties
   - Choose based on invariant impact vs. execution time budget
