<objective>
Fix response structure assertions for error handling tests

**Purpose:**

**Impact:**
- 2 tests fail due to incorrect response structure assumptions
- Error response pattern consistent across all admin routes
- Tests must access nested detail["error"]["message"] for string operations

**Output:**
- Fixed assertions in 2 error handling tests
- Reusable pattern for error response assertions
</objective>
