<task type="auto">
  <name>Define test quality standards (TQ-01 through TQ-05)</name>
  <files>backend/docs/TEST_QUALITY_STANDARDS.md</files>
  <action>
Create comprehensive test quality standards documentation:

1. Document TQ-01: Test Independence
   - No shared state between tests
   - Tests must pass in random order
   - Each test creates and cleans up its own data
   - Use fixtures for setup/teardown
   - Examples and anti-patterns

2. Document TQ-02: Pass Rate
   - 98%+ pass rate across 3 consecutive runs
   - Flaky tests must be fixed or disabled
   - No tolerated failures in CI/CD
   - Retry logic for transient failures
   - Pass rate monitoring and alerts

3. Document TQ-03: Performance
   - Full suite <60 minutes
   - No single test >30 seconds
   - Unit tests <1s per test
   - Integration tests <5s per test
   - E2E tests <30s per test
   - Performance profiling guidance

4. Document TQ-04: Determinism
   - No sleep() calls (use polling)
   - No dependency on wall-clock time
   - Use frozen_time fixture for time-dependent tests
   - No random data without fixed seed
   - Async test patterns

5. Document TQ-05: Coverage Quality
   - Behavior-based testing (not line coverage chasing)
   - Property-based tests for stateful logic
   - Edge case and error path testing
   - Minimum coverage thresholds per file type
   - Coverage exemptions process

6. Include:
   - Test writing templates
   - Common test patterns
   - Mock usage guidelines
   - CI/CD integration examples
   - Troubleshooting guide
  </action>
  <verify>wc -l backend/docs/TEST_QUALITY_STANDARDS.md && grep -c "TQ-0[1-5]" backend/docs/TEST_QUALITY_STANDARDS.md</verify>
  <done>TEST_QUALITY_STANDARDS.md created (400+ lines, 5 quality gates defined)</done>
