# Phase 62, Plan 17: Unit Tests for Pure Logic

**Objective:** Write unit tests for pure logic functions (algorithms, validators, utilities) that don't require external dependencies. Use parametrize and Hypothesis for comprehensive edge case coverage.

**Purpose:** Integration tests cover DB/API interactions, but pure logic functions need focused unit tests with parametrized inputs for edge cases. Unit tests are fast and catch algorithmic bugs.

**Output:** 50-60 unit tests for pure logic, coverage increases to 50-55%.

## Research Findings

From 62-RESEARCH.md, on unit vs integration testing:

> "Unit Tests (40%):
> - Pure logic: governance_cache, algorithms, validators
> - Fast (<1 second per test)
> - No external dependencies"

And on parametrized tests:

> "Parametrized Tests (@pytest.mark.parametrize)
> - Test multiple inputs in one function
> - 10x more coverage per test written
> - Example: Test all HTTP status codes in one test"

And on property-based testing:

> "Hypothesis: 1 property test = 100 traditional tests
> - Generates 1000 examples
> - Covers all edge cases"

    ```python
    import pytest
    from core.validators import (
        validate_agent_name,
        validate_workflow_definition,
        validate_api_key
    )

    @pytest.mark.unit
    @pytest.mark.parametrize("name,is_valid", [
        ("Valid Agent", True),
        ("A", True),  # Minimum length
        ("A" * 100, True),  # Maximum length
        ("", False),  # Empty
        ("A" * 101, False),  # Too long
        ("Agent@123", False),  # Invalid chars
        ("<script>alert('xss')</script>", False),  # XSS attempt
    ])
    def test_agent_name_validation(name, is_valid):
        """Parametrized test for agent name validation"""
        result = validate_agent_name(name)
        assert result.is_valid == is_valid

    @pytest.mark.unit
    @pytest.mark.parametrize("workflow,expected_errors", [
        ({"steps": []}, 1),  # No steps
        ({"steps": [{"id": "s1"}]}, 0),  # Valid
        ({"steps": [{"id": ""}]}, 1),  # Empty step ID
        (None, 1),  # None workflow
    ])
    def test_workflow_validation(workflow, expected_errors):
        """Parametrized test for workflow validation"""
        result = validate_workflow_definition(workflow)
        assert len(result.errors) == expected_errors

    @pytest.mark.unit
    @pytest.mark.parametrize("api_key,is_valid", [
        ("sk-1234567890", True),
        ("sk-abc123def456", True),
        ("", False),
        ("invalid", False),
        ("sk-", False),  # Too short
    ])
    def test_api_key_validation(api_key, is_valid):
        """Parametrized test for API key validation"""
        result = validate_api_key(api_key)
        assert result.is_valid == is_valid

    @pytest.mark.unit
    def test_validation_error_messages():
        """Test validation error messages are helpful"""
        result = validate_agent_name("")
        assert "required" in result.errors[0].lower() or "empty" in result.errors[0].lower()
    ```
