# Phase 62, Plan 14: Register Missing API Routes

**Objective:** Register missing API routes in main_api_app.py so that existing API tests actually exercise production code instead of returning 404.

**Purpose:** 50 API route tests return 404 because routes aren't registered. These tests contribute zero coverage. Registering the routes unlocks ~5-8% coverage from tests that already exist.

**Output:** All API routes registered, tests return real responses, coverage increases to 30-35%.

From 62-VERIFICATION.md:

> "Unregistered Routes: API route tests fail because routes are not registered in the application
> - workspace_routes NOT registered → 9 tests return 404
> - token_routes NOT registered → 7 tests return 404
> - marketing_routes NOT registered → 7 tests return 404
> - operational_routes NOT registered → 6 tests return 404
> - user_activity_routes NOT registered → 10 tests return 404"

**Total:** ~50 tests returning 404 = zero coverage of real API code.
