# Phase 62, Plan 13: Fix Test Execution Blockers

**Objective:** Fix import errors, configuration problems, and test discovery issues that prevent existing tests from running and measuring coverage.

**Purpose:** Phase 62 created 567 tests but coverage stayed at 17.12% because tests can't execute due to import errors, wrong coverage thresholds, and integration tests being excluded. This plan fixes the blockers to unlock 25-30% coverage from existing tests.

**Output:** Working test suite, realistic coverage configuration, all 730 test files discoverable by pytest.

## Context

@.planning/phases/62-test-coverage-80pct/62-RESEARCH.md
@.planning/phases/62-test-coverage-80pct/62-VERIFICATION.md

## Research Findings

From 62-RESEARCH.md, Phase 62 failed because:

1. **Tests don't execute**: pytest only discovers 1 of 730 test files
2. **Import errors**: 92 tests blocked by import errors (services have different APIs than expected)
3. **Wrong coverage threshold**: fail_under=80.0 when actual coverage is 17.12%
4. **Integration tests excluded**: --ignore=tests/integration/ flag excludes ~172 tests
5. **Branch coverage disabled**: No --cov-branch flag

**Root Cause:** Tests were written without verifying they run, and coverage was only measured after 4 months.
