  <action>
Generate comprehensive coverage report:

1. Run pytest with coverage:
   ```bash
   pytest tests/ --cov=core --cov=api --cov=tools --cov=integrations \
          --cov-report=html --cov-report=term --cov-report=json \
          --ignore=tests/property_tests/ --ignore=tests/integration/ \
          -q 2>&1 | tee coverage_report.txt
   ```

2. Extract metrics:
   - Overall coverage percentage
   - Coverage by module (core, api, tools, integrations)
   - Total lines of code
   - Lines covered vs. uncovered
   - Top 20 files with lowest coverage

3. Generate HTML report for detailed analysis

4. Create COVERAGE_ANALYSIS.md with:
   - Current baseline metrics
   - Module-by-module breakdown
   - Priority ranking (files with >500 lines and <20% coverage)
   - Critical path coverage assessment
   - Recommendations for next steps

<task type="auto">
  <name>Identify high-priority testing targets</name>
  <files>backend/docs/COVERAGE_ANALYSIS.md</files>
  <action>
Create prioritized testing target list:

1. Analyze coverage report for high-impact files:
   Criteria:
   - >500 lines of code
   - <20% current coverage
   - Critical business logic (governance, agents, LLM, memory)

2. Rank files by (lines_of_code * (1 - coverage_percentage))
   - This prioritizes large files with low coverage

3. Categorize by effort level:
   - **Quick wins** (100-300 lines): 5-10 tests each
   - **Medium effort** (300-800 lines): 15-30 tests each
   - **Large modules** (800+ lines): 30-50 tests each

4. Create testing strategy:
   - Phase approach (Wave 1: Quick wins, Wave 2: Medium, Wave 3: Large)
   - Estimated effort per file
   - Expected coverage gain
   - Dependencies between modules

5. Document in COVERAGE_ANALYSIS.md
  </action>

  <done>At least 20 high-priority targets identified, testing strategy documented</done>
