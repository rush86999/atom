# Phase 62, Plan 18: Integration Tests for Tools and External Services

**Objective:** Write integration tests for tools (canvas, browser, device) and external integrations (Slack, Discord) using HTTP-level mocking (respx/VCR) instead of service-level mocks.

**Purpose:** Tools and external integrations have low coverage (<15%) but are important for agent capabilities. HTTP-level mocking tests real integration code while avoiding external dependencies.

**Output:** 30-40 integration tests for tools/integrations, coverage increases to 55-60%.

## Context

@.planning/phases/62-test-coverage-80pct/62-17-PLAN.md
@.planning/phases/62-test-coverage-80pct/62-RESEARCH.md

## Research Findings

From 62-RESEARCH.md:

> "Integration Tests Win When:
> - External Service Calls: Unit test mocks httpx (tests mock responses), Integration test uses Respx/vcr recording (tests real API contracts)
> - ROI: 2x better (catches API contract bugs)"

From coverage analysis in 62-VERIFICATION.md:
- Tools coverage: 10.8% (very low)
- Integrations coverage: 11.4% (very low)

These modules need focused integration testing with HTTP-level mocking.

    Pattern: Use respx to mock HTTP at protocol level

    ```python
    import pytest
    import respx
    from integrations.slack_enhanced_service import SlackEnhancedService
    from integrations.discord_service import DiscordService

    @pytest.mark.integration
    @respx.mock
    def test_slack_message_send():
        """Test Slack message sending with HTTP mock"""
        # Mock Slack API at HTTP level
        respx.post("https://slack.com/api/chat.postMessage").mock(
            return_value=httpx.Response(200, json={
                "ok": True,
                "channel": "C12345",
                "ts": "1234567890.123456"
            })
        )

        service = SlackEnhancedService(token="xoxb-test")
        result = service.send_message(channel="C12345", text="Test message")

        assert result["success"] == True
        assert result["message_ts"] == "1234567890.123456"

    @pytest.mark.integration
    @respx.mock
    def test_slack_error_handling():
        """Test Slack API error handling"""
        respx.post("https://slack.com/api/chat.postMessage").mock(
            return_value=httpx.Response(429, json={
                "ok": False,
                "error": "rate_limited"
            })
        )

        service = SlackEnhancedService(token="xoxb-test")
        result = service.send_message(channel="C12345", text="Test")

        assert result["success"] == False
        assert "rate_limited" in result["error"]

    @pytest.mark.integration
    @respx.mock
    @pytest.mark.parametrize("status_code,expected_success", [
        (200, True),
        (400, False),
        (401, False),
        (429, False),
        (500, False),
    ])
    def test_slack_http_status_codes(status_code, expected_success):
        """Parametrized test for HTTP status codes"""
        respx.post("https://slack.com/api/chat.postMessage").mock(
            return_value=httpx.Response(status_code, json={"ok": status_code == 200})
        )

        service = SlackEnhancedService(token="xoxb-test")
        result = service.send_message(channel="C12345", text="Test")
        assert result["success"] == expected_success

    @pytest.mark.integration
    @respx.mock
    def test_discord_webhook():
        """Test Discord webhook"""
        respx.post("https://discord.com/api/webhooks/123/token").mock(
            return_value=httpx.Response(204)
        )

        service = DiscordService(webhook_url="https://discord.com/api/webhooks/123/token")
        result = service.send_message(content="Test")
        assert result["success"] == True
    ```

    Target: 10-12 integration tests
