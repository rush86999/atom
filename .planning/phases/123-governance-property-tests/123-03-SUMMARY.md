1. **test_all_maturity_action_combinations**
   - Validates: 16 maturity×complexity pairs (4×4)
   - Strategy: sampled_from(maturities) × integers(1-4)
   - Invariant: Action permitted iff complexity <= maturity capability

2. **test_student_blocked_from_all_high_complexity**
   - Validates: STUDENT blocked from complexity 3-4 actions
   - Strategy: STUDENT agent × high_complexity_actions
   - Invariant: Low maturity can't bypass complexity gates

3. **test_autonomous_allowed_all_actions**
   - Validates: AUTONOMOUS allowed all 40+ actions
   - Strategy: AUTONOMOUS agent × all defined actions
   - Invariant: Highest maturity has no restrictions

4. **test_maturity_gates_enforced_for_all_actions**
   - Validates: Minimum maturity enforced for complexity 3-4
   - Strategy: Test actions below required maturity
   - Invariant: No agent below minimum can perform action
   - Note: Tests only complexity 3-4 to avoid substring matching bugs

5. **test_capability_matrix_completeness**
   - Validates: All 40+ actions have valid complexity 1-4
   - Strategy: Verify ACTION_COMPLEXITY dict
   - Invariant: No action has invalid complexity (0, 5+, None)

6. **test_cached_permission_does_not_bypass_maturity**
   - Validates: Cache doesn't bypass maturity gates
   - Strategy: Warm cache with low-complexity, try high-complexity
   - Invariant: Cached permissions don't override maturity

7. **test_action_case_insensitivity_for_all_actions**
   - Validates: Case-insensitive matching works
   - Strategy: Sample actions with case variants
   - Invariant: Action matching works regardless of case

8. **test_all_complexity_levels_have_actions**
   - Validates: All 4 levels have >= 1 action
   - Strategy: Verify ACTION_COMPLEXITY distribution
   - Invariant: Each complexity level (1-4) has actions
