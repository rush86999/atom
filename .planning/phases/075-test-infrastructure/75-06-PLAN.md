# Plan 75-06: Create Docker Compose Environment for E2E UI Testing

**Wave**: 1 (parallel)
**Dependencies**: []

## Overview
Create Docker Compose environment for E2E UI testing with isolated services (backend, frontend, PostgreSQL) using non-conflicting ports to avoid conflicts with development environment.

## Requirements Addressed
- INFRA-02: Docker Compose test environment with backend, frontend, PostgreSQL services

## Files Modified
- `docker-compose-e2e-ui.yml` - Docker Compose configuration
- `scripts/start-e2e-env.sh` - Start script
- `scripts/stop-e2e-env.sh` - Stop script

## Tasks

### Task 1: Create docker-compose-e2e-ui.yml with test services
**Files**:
- `docker-compose-e2e-ui.yml`

**Action**:
Create docker-compose-e2e-ui.yml with:
- postgres service (PostgreSQL 16-alpine, port 5434)
- backend service (atom-backend, port 8001, depends on postgres)
- frontend service (atom-frontend, port 3001, depends on backend)
- Networks: e2e_test_network (isolated from dev network)
- Volumes: postgres_data_e2e (separate from dev database)

**Verify**:
```bash
grep -q "version:" docker-compose-e2e-ui.yml
grep -q "postgres:" docker-compose-e2e-ui.yml
grep -q "backend:" docker-compose-e2e-ui.yml
grep -q "frontend:" docker-compose-e2e-ui.yml
```

**Done**: When docker-compose-e2e-ui.yml has 3 services

### Task 2: Configure PostgreSQL with test database and user
**Files**:
- `docker-compose-e2e-ui.yml`

**Action**:
Add postgres service configuration:
```yaml
postgres:
  image: postgres:16-alpine
  container_name: atom-postgres-e2e
  ports:
    - "5434:5432"  # Non-conflicting with dev (5432)
  environment:
    POSTGRES_DB: atom_test
    POSTGRES_USER: test_user
    POSTGRES_PASSWORD: test_pass
  volumes:
    - postgres_data_e2e:/var/lib/postgresql/data
```

**Verify**:
```bash
grep -q "5434:5432" docker-compose-e2e-ui.yml
grep -q "atom_test" docker-compose-e2e-ui.yml
```

**Done**: When postgres service uses port 5434 and test database

### Task 3: Configure backend service with test environment variables
**Files**:
- `docker-compose-e2e-ui.yml`

**Action**:
Add backend service configuration:
```yaml
backend:
  build:
    context: ./backend
    dockerfile: Dockerfile
  container_name: atom-backend-e2e
  ports:
    - "8001:8000"  # Non-conflicting with dev (8000)
  environment:
    DATABASE_URL: postgresql://test_user:test_pass@postgres:5432/atom_test
    PORT: 8000
    LOG_LEVEL: WARNING
  depends_on:
    postgres:
      condition: service_healthy
```

**Verify**:
```bash
grep -q "8001:8000" docker-compose-e2e-ui.yml
grep -q "DATABASE_URL" docker-compose-e2e-ui.yml
```

**Done**: When backend service uses port 8001 and test DATABASE_URL

### Task 4: Configure frontend service with test backend URL
**Files**:
- `docker-compose-e2e-ui.yml`

**Action**:
Add frontend service configuration:
```yaml
frontend:
  build:
    context: ./frontend-nextjs
    dockerfile: Dockerfile
  container_name: atom-frontend-e2e
  ports:
    - "3001:3000"  # Non-conflicting with dev (3000)
  environment:
    NEXT_PUBLIC_API_URL: http://backend:8000
  depends_on:
    backend:
      condition: service_started
```

**Verify**:
```bash
grep -q "3001:3000" docker-compose-e2e-ui.yml
grep -q "NEXT_PUBLIC_API_URL" docker-compose-e2e-ui.yml
```

**Done**: When frontend service uses port 3001 and backend URL

### Task 5: Add health checks for postgres and backend
**Files**:
- `docker-compose-e2e-ui.yml`

**Action**:
Add healthcheck to postgres:
```yaml
healthcheck:
  test: ["CMD-SHELL", "pg_isready -U test_user -d atom_test"]
  interval: 2s
  timeout: 5s
  retries: 5
```

Add healthcheck to backend:
```yaml
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:8000/health/live"]
  interval: 10s
  timeout: 5s
  retries: 3
```

**Verify**:
```bash
grep -q "healthcheck:" docker-compose-e2e-ui.yml
grep -q "pg_isready" docker-compose-e2e-ui.yml
```

**Done**: When health checks configured

### Task 6: Create start-e2e-env.sh script
**Files**:
- `scripts/start-e2e-env.sh`

**Action**:
Create start-e2e-env.sh:
```bash
#!/bin/bash
docker-compose -f docker-compose-e2e-ui.yml up -d
echo "Waiting for services to be healthy..."
./scripts/wait-for-healthy.sh
echo "E2E test environment ready!"
echo "Frontend: http://localhost:3001"
echo "Backend: http://localhost:8001"
```

Make executable:
```bash
chmod +x scripts/start-e2e-env.sh
```

**Verify**:
```bash
grep -q "docker-compose-e2e-ui.yml" scripts/start-e2e-env.sh
test -x scripts/start-e2e-env.sh
```

**Done**: When start-e2e-env.sh starts environment

### Task 7: Create stop-e2e-env.sh script
**Files**:
- `scripts/stop-e2e-env.sh`

**Action**:
Create stop-e2e-env.sh:
```bash
#!/bin/bash
docker-compose -f docker-compose-e2e-ui.yml down -v
echo "E2E test environment stopped and volumes cleaned."
```

Make executable:
```bash
chmod +x scripts/stop-e2e-env.sh
```

**Verify**:
```bash
grep -q "docker-compose-e2e-ui.yml" scripts/stop-e2e-env.sh
test -x scripts/stop-e2e-env.sh
```

**Done**: When stop-e2e-env.sh stops environment

## Success Criteria
1. docker-compose-e2e-ui.yml starts 3 services (postgres, backend, frontend)
2. PostgreSQL uses port 5434 (non-conflicting with dev on 5432)
3. Backend uses port 8001 (non-conflicting with dev on 8000)
4. Frontend uses port 3001 (non-conflicting with dev on 3000)
5. Health checks ensure services are ready before tests run
6. start-e2e-env.sh starts environment, stop-e2e-env.sh stops + cleans volumes

## Notes
- Non-conflicting ports: 5434 (db), 8001 (backend), 3001 (frontend)
- Separate volumes: postgres_data_e2e (doesn't affect dev database)
- Alpine images: Fast startup for postgres (16-alpine)
- Health checks: Prevent tests from running against unhealthy services
- Deferred: Playwright browser installation (75-07), test execution scripts (Phase 80)
