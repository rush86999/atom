# Plan 75-01: Create E2E UI Test Directory and Configuration

**Wave**: 1 (parallel)
**Dependencies**: []
**Autonomous**: true

## Overview
Create the foundational E2E UI test directory structure with pytest, Playwright, and project configuration files. This establishes the testing framework before creating fixtures.

## Requirements Addressed
- INFRA-01: Playwright Python 1.58.0 with pytest-playwright plugin
- INFRA-03: Playwright configuration with base URL, browsers, timeout settings

## Files Modified
- `backend/tests/e2e_ui/conftest.py` - Root pytest configuration with browser fixtures
- `backend/tests/e2e_ui/pyproject.toml` - Pytest and Playwright dependencies
- `backend/tests/e2e_ui/playwright.config.ts` - Playwright base URL and browser settings

## Tasks

### Task 1: Create E2E UI test directory structure
**Files**:
- `backend/tests/e2e_ui/__init__.py`
- `backend/tests/e2e_ui/tests/__init__.py`
- `backend/tests/e2e_ui/tests/test_example.py`


**Done**: When directories and __init__.py files exist

### Task 2: Create conftest.py with pytest and Playwright configuration
**Files**:
- `backend/tests/e2e_ui/conftest.py`

**Action**:
Create conftest.py with:
- pytest_configure hook for Playwright
- browser fixture (session-scoped, Chromium)
- browser_context fixture (accept downloads, bypass CSP)
- page fixture (from browser context)
- base_url fixture (http://localhost:3001 for E2E UI testing)

**Verify**:
```bash
grep -q "pytest_configure" backend/tests/e2e_ui/conftest.py
grep -q "@pytest.fixture(scope=\"session\")" backend/tests/e2e_ui/conftest.py
grep -q "base_url" backend/tests/e2e_ui/conftest.py
```

**Done**: When conftest.py has pytest hooks and browser fixtures

### Task 3: Create pyproject.toml with pytest and Playwright dependencies
**Files**:
- `backend/tests/e2e_ui/pyproject.toml`

**Action**:
Create pyproject.toml with:
- pytest >= 8.0.0
- pytest-playwright >= 1.58.0
- pytest-asyncio >= 0.23.0
- pytest-xdist >= 3.5.0 (for parallel execution)
- faker >= 22.0.0 (for test data generation)

**Verify**:
```bash
grep -q "pytest-playwright" backend/tests/e2e_ui/pyproject.toml
grep -q "1.58.0" backend/tests/e2e_ui/pyproject.toml
```

**Done**: When pyproject.toml has all required dependencies

### Task 4: Create playwright.config.ts with base URL and browser settings
**Files**:
- `backend/tests/e2e_ui/playwright.config.ts`

**Action**:
Create playwright.config.ts with:
- baseURL: "http://localhost:3001" (non-conflicting with dev frontend)
- browsers: chromium only (firefox, webkit deferred to v3.2)
- timeout: 30000ms (30 seconds for action timeout)
- retries: 0 (local), 2 (CI environment)
- workers: process.env.CI ? 2 : 4 (parallel execution)

**Verify**:
```bash
grep -q "baseURL:" backend/tests/e2e_ui/playwright.config.ts
grep -q "chromium" backend/tests/e2e_ui/playwright.config.ts
grep -q "timeout:" backend/tests/e2e_ui/playwright.config.ts
```

**Done**: When playwright.config.ts has base URL, browser config, and timeouts

## Success Criteria
1. pytest can discover tests in backend/tests/e2e_ui/tests/
2. Playwright browser fixture is available to tests
3. Base URL points to localhost:3001 (non-conflicting with dev)
4. playwright.config.ts has Chromium-only configuration
5. Example test runs (may fail if frontend not running, but framework works)

## Notes
- Uses non-conflicting port 3001 for E2E UI testing (dev frontend uses 3000)
- Playwright Python (not TypeScript) for pytest integration
- Chromium-only for v3.1 (Firefox/Safari deferred to v3.2)
- Session-scoped browser fixture for performance (reuse browser across tests)
