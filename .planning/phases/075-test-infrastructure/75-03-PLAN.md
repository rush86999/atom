# Plan 75-03: Create Test Data Factories for Unique Test Data

**Wave**: 1 (parallel)
**Dependencies**: []
**Autonomous**: true

## Overview
Create test data factories using factory_boy pattern for generating unique, realistic test data per worker. UUID v4 prevents parallel test collisions when multiple workers create data simultaneously.

## Requirements Addressed
- INFRA-05: Test data factories with unique data generation per worker

### Task 4: Create fixture for worker-specific data isolation
**Files**:
- `backend/tests/e2e_ui/fixtures/test_data_factory.py`

**Action**:
Add worker_id fixture:
```python
@pytest.fixture(scope="session")
def worker_id():
    """Return unique worker ID for this test worker (gw0, gw1, etc.)"""
    worker = os.getenv("PYTEST_XDIST_WORKER", "gw0")
    return worker
```

Add worker_seed fixture:
```python
@pytest.fixture(scope="function")
def worker_seed(worker_id):
    """Return unique seed for this worker + function combination"""
    return f"{worker_id}_{uuid4().hex[:8]}"
```

**Verify**:
```bash
grep -q "def worker_id" backend/tests/e2e_ui/fixtures/test_data_factory.py
grep -q "PYTEST_XDIST_WORKER" backend/tests/e2e_ui/fixtures/test_data_factory.py
```

**Done**: When worker_id and worker_seed fixtures exist

## Success Criteria
1. UserFactory generates unique email per call (UUID v4)
2. ProjectFactory generates unique name per call
3. worker_id fixture returns worker ID (gw0, gw1, etc.)
4. Factories create realistic data (names, emails, descriptions)
5. No data collisions when tests run in parallel (pytest-xdist)

## Notes
- UUID v4 ensures uniqueness across workers (gw0, gw1, gw2, gw3)
- factory_boy pattern: Declarative test data generation
- Faker library: Realistic fake data (names, emails, addresses)
- Worker-specific seeding: Each worker creates independent data sets
