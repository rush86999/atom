# Plan 75-05: Create API-First Setup Utilities for Fast Test Initialization

**Wave**: 1 (parallel)
**Dependencies**: []
**Autonomous**: true

## Overview
Create API-first setup utilities for fast test state initialization. Bypassing slow UI navigation (10-100x speedup) by calling API endpoints directly to create users, projects, skills, and other test data.

## Requirements Addressed
- INFRA-06: API-first test setup utilities for fast state initialization

## Tasks

### Task 1: Create api_setup.py with API client helper
**Files**:
- `backend/tests/e2e_ui/utils/api_setup.py`

**Action**:

## Success Criteria
1. APIClient can make HTTP requests to backend (port 8001)
2. create_test_user bypasses UI login (10-100x faster)
3. setup_test_user fixture returns authenticated user
4. Projects and skills can be created via API (no UI navigation)
5. Tests initialize state in <100ms vs 5-10s for UI navigation

## Notes
- API-first: Bypass UI for expensive state initialization
- Speedup: 10-100x faster (API: ~50ms vs UI login: 5-10s)
- Port 8001: Non-conflicting with dev backend (8000)
- JWT in localStorage: Sets authentication for Playwright page
