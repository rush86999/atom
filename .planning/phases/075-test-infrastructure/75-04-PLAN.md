### Task 2: Implement schema creation and teardown logic
**Files**:
- `backend/tests/e2e_ui/fixtures/database_fixtures.py`

**Action**:
Add schema creation:
```python
@pytest.fixture(scope="session", autouse=True)
def create_worker_schema(worker_id, get_engine):
    """Create worker-specific schema before test session"""
    schema = f"test_schema_{worker_id}"
    with get_engine.connect() as conn:
        conn.execute(sqlalchemy.text(f"CREATE SCHEMA IF NOT EXISTS {schema}"))
        conn.commit()
    yield schema
    # Teardown: drop schema after session
    with get_engine.connect() as conn:
        conn.execute(sqlalchemy.text(f"DROP SCHEMA IF EXISTS {schema} CASCADE"))
        conn.commit()
```

```python
def db_session(worker_schema, get_engine):
    """Create database session with transaction rollback"""
    from sqlalchemy.orm import sessionmaker
    engine = get_engine.execution_options(isolation_level="REPEATABLE READ")
    Session = sessionmaker(bind=engine)
    session = Session()
    session.execute(text(f"SET search_path TO {worker_schema}"))
    connection = session.connection()
    transaction = connection.begin()
    yield session
    # Rollback transaction after test
    session.rollback()
    session.close()
```

## Success Criteria
1. Each worker (gw0, gw1, gw2, gw3) has separate schema (test_schema_gw0, etc.)
2. Schemas are created before test session and dropped after
3. Transactions are rolled back after each test (no data pollution)
4. Tables are created in worker schema (not public schema)
5. Tests can run in parallel without data collisions

## Notes
- PostgreSQL schemas: Lightweight namespaces (not separate databases)
- Schema naming: test_schema_gw0, test_schema_gw1, test_schema_gw2, test_schema_gw3
- Transaction rollback: Faster than TRUNCATE for cleanup
- search_path: Directs queries to worker schema
- Deferred: Actual test data creation (Phase 76+), schema migration testing (Phase 76)
