    Create backend/tests/scripts/coverage_report_generator.py that:

    1. Runs pytest with coverage and generates all report formats:
       - HTML report with --cov-report=html
       - JSON report with --cov-report=json
       - Terminal report with --cov-report=term-missing

    2. Post-processes coverage.json for enhanced insights:
       - Identifies files with <50% coverage (high priority for improvement)
       - Lists top 10 files with most uncovered lines
       - Calculates branch coverage gap (total - covered)
       - Finds modules with zero coverage (untested modules)

    3. Generates summary report:
       - Overall coverage percentage
       - Modules below threshold (with current coverage)
       - Uncovered line count per module
       - Branch coverage statistics

    4. Outputs report to console and optionally to file

    Script structure:
    - run_coverage_tests(modules) -> subprocess to pytest
    - parse_coverage_json(json_path) -> returns coverage data
    - find_low_coverage_files(coverage, threshold=50) -> list of files
    - find_untested_modules(coverage) -> modules with 0% coverage
    - calculate_branch_gap(coverage) -> missing branch count
    - generate_summary_report(coverage) -> formatted output
    - main() with --modules, --threshold, --output-file options

    Use subprocess for pytest, json for parsing, argparse for CLI.
  </action>
