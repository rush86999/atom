  <name>Task 2: Create quality standards document</name>
  <files>backend/docs/QUALITY_STANDARDS.md</files>
  <action>
    Create backend/docs/QUALITY_STANDARDS.md with:

    1. Testing philosophy:
       - Tests are code, treat them with same quality standards
       - One assertion per test (single responsibility)
       - Tests should be independent and deterministic
       - Mock external dependencies, test internal logic

    2. Test patterns (with examples):
       - Arrange-Act-Assert (AAA) pattern
       - Given-When-Then for BDD-style tests
       - Factory pattern for test data
       - Fixture reuse for common setup

    3. Test naming conventions:
       - test_unitOfWork_underWhatConditions_returnsExpectedResult
       - test_errorCondition_throwsException
       - Property test: test_invariantName_underConditions

    4. Quality metrics:
       - Assertion density: 15+ assertions per 100 lines
       - Test independence: Random order must pass
       - Pass rate: 98%+ across suite
       - Execution time: <5 min for full suite

    5. Anti-patterns to avoid:
       - Testing implementation details
       - Brittle tests (over-specific assertions)
       - Shared state between tests
       - Mocking what you don't own
       - Test code duplication

    6. Code examples:
       - Good test vs bad test comparisons
       - Fixture usage examples
       - Mock patterns for external services

    Use Markdown with code examples, good/bad comparisons, cross-references.
  </action>

  <name>Task 3: Create quality runbook for troubleshooting</name>
  <files>backend/docs/QUALITY_RUNBOOK.md</files>
  <action>
    Create backend/docs/QUALITY_RUNBOOK.md with:

    1. Common issues and solutions:
       - Coverage below threshold: Identify gaps, add tests, use report drill-down
       - Flaky tests: Identify timing issues, add proper mocks, use fixtures
       - Slow tests: Profile with --durations, optimize setup, mock IO
       - Import errors: Check PYTHONPATH, verify conftest.py, isolate test modules
       - Database errors: Use db_session fixture, clean up data, check transactions

    2. Debugging techniques:
       - Run single test: pytest tests/test_module.py::test_function -v
       - Run with output: pytest -v -s (show print statements)
       - Run with debugger: pytest --pdb
       - Run with coverage: pytest --cov=module_name --cov-report=html

    3. CI failures:
       - Check logs for specific failure
       - Reproduce locally: Use same pytest command
       - Check environment: Verify test dependencies, Python version
       - Check for state issues: Run with --random-order

    4. Coverage issues:
       - View HTML report: Open htmlcov/index.html in browser
       - Find uncovered lines: JSON parse_coverage_json.py
       - Check branch coverage: Look for missing else/finally clauses
       - Profile: Use pytest-cov --cov-context=test

    5. Performance optimization:
       - Parallel execution: pytest -n auto
       - Test splitting: Run subsets by marker
       - Fixture caching: Use session-scoped fixtures
       - Mock slow operations: External API calls, database queries

    6. Getting help:
       - Check documentation: TEST_COVERAGE_GUIDE.md, QUALITY_STANDARDS.md
       - Ask in team chat: Include error logs, reproduction steps
       - Create issue: Template for bug reports

    Use Markdown with troubleshooting flowcharts, code snippets, cross-references.
