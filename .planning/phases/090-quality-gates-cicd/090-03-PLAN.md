  <name>Task 1: Create coverage trend generation script</name>
  <files>backend/tests/scripts/generate_coverage_trend.py</files>
  <action>
    Create backend/tests/scripts/generate_coverage_trend.py that:

    1. Reads current coverage.json and existing trending.json
    2. Appends new entry to coverage_history array with:
       - date (ISO format)
       - phase, plan identifiers
       - coverage_percent, files_covered, files_total
       - branches_covered, branches_total
       - new_files_added, modified_files (from git diff)
    3. Calculates trend metrics:
       - 7-day average, 30-day average
       - Week-over-week change percentage
       - Regression detection (drop >5% triggers alert)
       - Linear regression to predict 80% target date
    4. Generates trend_analysis section with insights
    5. Writes updated trending.json
    6. Optionally generates HTML report with charts

    Script structure:
    - load_current_coverage() -> parses coverage.json
    - load_trending_data() -> parses existing trending.json or creates new
    - get_git_metrics() -> returns file counts from git diff
    - calculate_trend_metrics(history) -> averages, changes, predictions
    - detect_regression(current, baseline) -> alerts if drop >5%
    - predict_target_date(history, target=80) -> estimated completion
    - generate_html_report(trending, output_path) -> creates HTML with charts
    - main() with --coverage-json, --trending-json, --html-output options

    Use json for data parsing, argparse for CLI, datetime for timestamps, statistics module for linear regression.
  </action>

  </verify>
