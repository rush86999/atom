# Phase 225.1 Plan 01: BYOKHandler to LLMService Gap Analysis

**Created:** March 22, 2026
**Purpose:** Identify gaps between BYOKHandler features used by agents and LLMService exposed methods
**Status:** Complete

---

## Executive Summary

This gap analysis documents all BYOKHandler methods used by agent classes and compares them against LLMService's exposed interface. The analysis reveals that **LLMService already exposes equivalent methods for all BYOKHandler functionality used by agents**, enabling a straightforward migration path.

**Key Finding:** No new methods need to be added to LLMService. The migration can proceed immediately using existing LLMService methods.

## Gap Analysis

### Missing Methods Analysis

**Question:** Which BYOKHandler methods used by agents are NOT exposed in LLMService?

**Answer:** **NONE** - All BYOKHandler methods used by agents have equivalent LLMService methods.

### Method Mapping Table

| BYOKHandler Method (used by agents) | LLMService Equivalent | Status |
|-------------------------------------|----------------------|--------|
| `analyze_query_complexity()` | Available via `handler.analyze_query_complexity()` (access through `llm_service.handler`) | ✅ Available |
| `generate_structured_response()` | `generate_structured()` | ✅ Equivalent |
| `generate_response()` | `generate()` or `generate_completion()` | ✅ Equivalent |
| `get_optimal_provider()` | `get_optimal_provider()` | ✅ Same signature |
| `get_ranked_providers()` | `get_ranked_providers()` | ✅ Same signature |
| `classify_cognitive_tier()` | `classify_tier()` | ✅ Same signature |

### Exceptions Found

**analyze_query_complexity()**: This method is NOT directly exposed as a top-level LLMService method, but IS accessible via:
```python
llm_service.handler.analyze_query_complexity(prompt, task_type)
```

**Migration Decision:** Either:
1. **Option A:** Add `analyze_query_complexity()` as a top-level LLMService method (simple delegation)
2. **Option B:** Access via `llm_service.handler.analyze_query_complexity()` (requires no code changes)

**Recommendation:** Option A - Add top-level method for cleaner API

## Migration Strategy

### Recommended Approach

**Strategy:** "Add Missing Method + Migrate All Services"

1. **Phase 1: Add Missing Method to LLMService** (225.1-02)
   - Add `analyze_query_complexity()` as top-level LLMService method
   - Simple delegation: `return self.handler.analyze_query_complexity(prompt, task_type)`

2. **Phase 2: Migrate GenericAgent** (225.1-03)
   - Replace `self.llm = BYOKHandler(...)` with `self.llm = LLMService(...)`
   - Update 3 method calls to use LLMService interface
   - Test all 53 GenericAgent tests

3. **Phase 3: Migrate Other Services** (225.1-04)
   - AgentExecutionService: Replace `BYOKHandler()` with `LLMService()`
   - EpisodeSegmentationService: Accept `LLMService` in constructor
   - CanvasSummaryService: Accept `LLMService` in constructor

### Methods to Add to LLMService

**Single Method Required:**

```python
def analyze_query_complexity(self, prompt: str, task_type: Optional[str] = None) -> QueryComplexity:
    """
    Analyze query complexity for optimal provider routing.

    Args:
        prompt: The user query to analyze
        task_type: Optional task type hint (code, chat, analysis, etc.)

    Returns:
        QueryComplexity enum value (SIMPLE, MODERATE, COMPLEX, ADVANCED)

    Example:
        >>> service = LLMService()
        >>> complexity = service.analyze_query_complexity("Write Python code")
        >>> print(complexity)  # QueryComplexity.ADVANCED
    """
    return self.handler.analyze_query_complexity(prompt, task_type)
```

### Migration Order (Simplest to Most Complex)

1. **CanvasSummaryService** (LOW complexity)
   - Single dependency injection change
   - No method signature changes
   - Tests: 13 tests in `test_canvas_summary_service.py`

2. **EpisodeSegmentationService** (LOW complexity)
   - Single dependency injection change

   - Tests: Part of episodic memory test suite

3. **AgentExecutionService** (LOW complexity)
   - Replace `BYOKHandler()` instantiation with `LLMService()`
   - Update 2 method calls (analyze_query_complexity, get_optimal_provider)
   - Tests: Integration tests for agent execution

4. **GenericAgent** (MEDIUM complexity)
   - Replace `self.llm = BYOKHandler(...)` with `self.llm = LLMService(...)`
   - Update 3 method calls
   - Verify instructor library still works through LLMService
   - Tests: 53 tests in `test_generic_agent_coverage.py`

## Risk Assessment

### Migration Risks

| Risk | Severity | Likelihood | Mitigation |
|------|----------|------------|------------|
| Breaking instructor integration | HIGH | LOW | LLMService.generate_structured() uses same instructor wrapper |
| Performance regression | MEDIUM | LOW | LLMService delegates to BYOKHandler (no overhead) |
| Test failures | MEDIUM | LOW | All method signatures are compatible |
| Missing configuration | LOW | LOW | LLMService accepts same workspace_id parameter |

### Key Considerations

1. **Instructor Library**: LLMService.generate_structured() already wraps BYOKHandler.generate_structured_response(), so no breaking changes
2. **Backward Compatibility**: Adding top-level `analyze_query_complexity()` doesn't break existing code
3. **Performance**: Zero overhead - LLMService is a thin wrapper around BYOKHandler
4. **Testing**: All existing tests can run with LLMService instead of BYOKHandler
