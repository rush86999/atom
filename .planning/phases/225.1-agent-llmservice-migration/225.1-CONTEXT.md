# Phase 225.1: Agent LLMService Migration - Context

**Gathered:** March 22, 2026
**Status:** Ready for planning

## Guiding Vision

**LLMService is the single source of truth for ALL LLM interactions** — Agent classes should use LLMService wrapper, not BYOKHandler directly. This unifies the architecture and makes LLMService easier to manage.

Previous architecture (verified in 225-02) had agent classes using BYOKHandler (Layer 2) while API routes used LLMService (Layer 3). This created two integration patterns. New vision: Everything uses LLMService.

## Phase Boundary

Migrate agent classes from BYOKHandler to LLMService wrapper, exposing any missing methods on LLMService that agents need (cognitive tiers, instructor integration, etc.).

**Files in scope:**
- `backend/core/generic_agent.py` — Primary target
- Other agent classes that use BYOKHandler directly (to be discovered during research)

**Success Criteria:**
1. generic_agent.py uses LLMService instead of BYOKHandler
2. LLMService exposes all methods agents need (or agent code adapts)
3. All agent tests pass with LLMService integration
4. No regression in agent execution or capabilities
5. Single unified pattern: all code uses LLMService

## Implementation Decisions

### Architectural Change
- **Previous decision** (225-02): Agent classes use BYOKHandler directly for full feature access
- **New decision** (user request): LLMService is the single source of truth — migrate agents to LLMService
- **Rationale:** Simpler architecture, easier management, unified observability

### Migration Strategy
- **Research first**: Understand what features agents use from BYOKHandler
- **Gap analysis**: Identify missing methods on LLMService
- **Expose or adapt**: Add missing methods to LLMService OR adapt agent code
- **Sequential migration**: generic_agent.py first, then other agent classes

### Claude's Discretion
- Phase numbering (225.1 vs 226+)
- Whether to expose methods on LLMService or create adapter pattern
- How to handle instructor library integration (used by generic_agent.py)
