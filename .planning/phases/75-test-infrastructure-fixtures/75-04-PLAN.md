  <action>
Create `backend/tests/e2e_ui/fixtures/database_fixtures.py` with database isolation:

Fixture: `postgres_dsn()` - Session-scoped
- Returns PostgreSQL DSN from environment or default
- Default: "postgresql://e2e_tester:test_password@localhost:5433/atom_e2e_test"
- Reads DATABASE_URL environment variable if set

Fixture: `db_engine(postgres_dsn)` - Session-scoped
- Creates SQLAlchemy engine with postgres_dsn
- Pool size: 5, max_overflow: 10 (for parallel workers)
- Pool pre-ping enabled (connection validation)
- Echo: False (set True for SQL debugging)
- Returns Engine instance

Fixture: `worker_schema(worker_id)` - Function-scoped
- Returns schema name based on worker: f"test_schema_{worker_id}"
- For master/single worker: "test_schema_master"
- For gw0: "test_schema_gw0", gw1: "test_schema_gw1", etc.
- IMPORTANT: Different naming than existing E2E to avoid conflicts

Fixture: `create_worker_schema(engine, worker_schema)` - Function-scoped (autouse before db_session)
- Connects to PostgreSQL
- Creates schema if not exists: CREATE SCHEMA IF NOT EXISTS {worker_schema}
- Sets search_path to worker_schema
- Runs Base.metadata.create_all with schema=worker_schema
- Handles "already exists" errors gracefully

Fixture: `drop_worker_schema(engine, worker_schema)` - Function-scoped (finalizer)
- Drops schema after all tests complete: DROP SCHEMA IF NOT EXISTS {worker_schema} CASCADE
- Runs in teardown (not per-test, for efficiency)

Fixture: `db_session(engine, worker_schema)` - Function-scoped
- Creates session with schema in search_path
- Executes: SET search_path TO {worker_schema}, public
- Begins transaction for rollback
- Yields session
- On teardown: session.rollback(), session.close()
- IMPORTANT: Uses rollback, not commit (for fast cleanup)

Fixture: `db_connection(db_session)` - Function-scoped
- Returns raw connection for SQL execution
- For direct SQL when ORM not needed
- Auto-closes on teardown

Helper: `get_worker_id()` - Function
- Returns os.getenv("PYTEST_XDIST_WORKER_ID", "master")
- Normalizes: "gw0" -> "gw0", "master" -> "master"

Helper: `init_schema_for_worker(engine, schema_name)` - Function
- Creates all tables in specified schema
- Called by create_worker_schema fixture

IMPORTANT: Use SET search_path for schema isolation. Do NOT drop tables between tests (use rollback). Use CASCADE when dropping schema. Session-scoped engine shared across tests in worker.
