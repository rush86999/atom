# Skill Execution Context
- Execution endpoint: POST /api/skills/execute with skill_id, inputs, agent_id
- Execution returns: {success: bool, result: {}, error: str, execution_id: str}
- Long-running skills show progress indicator (spinner, progress bar)
- Output formats: text (plain), JSON (formatted), canvas (chart/form)
- Execution errors displayed with: error message, suggested fix, retry option
- Governance: STUDENT blocked from Python skills, INTERN needs approval for some actions
- Execution history stored in SkillExecution table with status (pending, running, success, failed)
- WebSocket or polling for real-time updates on long-running skills
</context>
