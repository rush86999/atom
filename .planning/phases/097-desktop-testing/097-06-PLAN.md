    **Property tests:**

    1. **File Path Validation Invariant** (2-3 properties):
       - Property: Paths with directory traversal (..) should be rejected
       - Property: Valid path segments should be accepted

       - Settings: { numRuns: 100 }

    2. **Command Parameter Validation Invariant** (2-3 properties):
       - Property: Empty command names should be rejected
       - Property: Valid command parameters should be accepted
       - Use: fc.constantFrom for valid commands, fc.string() for invalid
       - Settings: { numRuns: 100 }

    3. **Shell Command Whitelist Invariant** (2-3 properties):
       - Property: Shell commands must match whitelist pattern
       - Property: Whitelisted commands should execute successfully
       - Use: fc.asciiString(1, 20) for command generation
       - Whitelist from main.rs: ['ls', 'pwd', 'cat', 'grep', 'head', 'tail', 'echo', 'find', 'ps', 'top']
       - Settings: { numRuns: 100 }

    4. **Session State Consistency Invariant** (2-3 properties):
       - Property: Set session followed by get session should return same values
       - Property: Session token format should be valid
       - Use: fc.uuid(), fc.asciiString(1, 50), fc.email()
       - Settings: { numRuns: 50 } (IO-bound)

    5. **Notification Parameter Validation Invariant** (2-3 properties):
       - Property: Notification title should not be empty
       - Property: Notification sound values should be valid ("default" or "none")
       - Use: fc.string() for title, fc.constantFrom("default", "none") for sound
       - Settings: { numRuns: 100 }

    6. **File Content Round-trip Invariant** (1-2 properties):
       - Property: Write followed by read should return same content
       - Use: fc.string() for content, fc.asciiString(1, 32) for filename
       - Settings: { numRuns: 50 } (IO-bound)

    Note: Mock Tauri invoke for CI testing (GUI not required for command validation tests).
    Use vi Jest mocks or conditional test skipping for actual Tauri commands.

    Follow patterns from mobile queueInvariants.test.ts and backend Hypothesis tests.
