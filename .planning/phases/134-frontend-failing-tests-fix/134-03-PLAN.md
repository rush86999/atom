    Current state (lines 33-41):
    ```typescript
    // Establish API mocking before all tests (only if server loaded)
    if (server) {
      beforeAll(() => server.listen({ onUnhandledRequest: 'error' }));
      // Reset any request handlers that we may add during the tests,
      // so they don't affect other tests
      afterEach(() => server.resetHandlers());
      // Clean up after the tests are finished
      afterAll(() => server.close());
    }
    ```

    While the if (server) block exists, the closures inside beforeAll/afterEach/afterAll still reference `server` directly. If `server` becomes undefined between setup and test execution (rare edge case), this causes errors.

    Add defensive null checks inside each hook:
    ```typescript
    // Establish API mocking before all tests (only if server loaded)
    if (server) {
      beforeAll(() => server?.listen({ onUnhandledRequest: 'error' }));
      // Reset any request handlers that we may add during the tests,
      // so they don't affect other tests
      afterEach(() => server?.resetHandlers());
      // Clean up after the tests are finished
      afterAll(() => server?.close());
    }
    ```

    Using optional chaining (`?.`) ensures that if server somehow becomes undefined between setup and execution, the hooks will no-op rather than throw.
