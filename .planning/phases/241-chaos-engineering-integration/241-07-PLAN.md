```yaml
name: Chaos Engineering Tests (Weekly)

on:
  schedule:
    # Run every Sunday at 2 AM UTC (Saturday night in US)
    - cron: '0 2 * * 0'
  workflow_dispatch:  # Allow manual trigger

jobs:
  chaos-tests:
    name: Run Chaos Engineering Tests
    runs-on: ubuntu-latest
    timeout-minutes: 90

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: |
          cd backend
          pip install -e .
          pip install pytest pytest-timeout psutil

      - name: Install chaos testing dependencies
        run: |
          # Install toxiproxy-python for network chaos
          pip install toxiproxy-python || echo "Toxiproxy not available, will skip network chaos tests"

          # Install memory-profiler for memory chaos tests
          pip install memory-profiler || echo "memory-profiler not available"

      - name: Start Toxiproxy (if available)
        run: |
          # Try to start Toxiproxy via Docker
          docker run -d --name toxiproxy -p 8474:8474 ghcr.io/shopify/toxiproxy:2.5.0 || echo "Toxiproxy Docker not available"
          docker ps || echo "Docker not available"

      - name: Verify test environment
        run: |
          cd backend
          # Verify we're in test environment
          export ENVIRONMENT=test
          export DATABASE_URL=sqlite:///./test_chaos.db

          # Verify no production access
          if echo "$DATABASE_URL" | grep -i "production"; then
            echo "ERROR: Production database detected!"
            exit 1
          fi

          echo "✓ Test environment verified"

      - name: Run network latency chaos tests
        run: |
          cd backend
          pytest tests/chaos/test_network_latency_chaos.py -v --tb=short -m chaos --timeout=120 || echo "Network chaos tests failed"

      - name: Run database drop chaos tests
        run: |
          cd backend
          pytest tests/chaos/test_database_drop_chaos.py -v --tb=short -m chaos --timeout=120 || echo "Database chaos tests failed"

      - name: Run memory pressure chaos tests
        run: |
          cd backend
          pytest tests/chaos/test_memory_pressure_chaos.py -v --tb=short -m chaos --timeout=120 || echo "Memory chaos tests failed"

      - name: Run service crash chaos tests
        run: |
          cd backend
          pytest tests/chaos/test_service_crash_chaos.py -v --tb=short -m chaos --timeout=120 || echo "Service crash tests failed"

      - name: Run blast radius control tests
        run: |
          cd backend
          pytest tests/chaos/test_blast_radius_controls.py -v --tb=short -m chaos --timeout=120 || echo "Blast radius tests failed"

      - name: Run recovery validation tests
        run: |
          cd backend
          pytest tests/chaos/test_recovery_validation.py -v --tb=short -m chaos --timeout=120 || echo "Recovery validation tests failed"

      - name: Run all chaos tests (comprehensive)
        run: |
          cd backend
          pytest tests/chaos/ -v --tb=short -m chaos --timeout=120 --junitxml=chaos-test-results.xml

      - name: Upload test results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: chaos-test-results
          path: backend/chaos-test-results.xml
          retention-days: 30

      - name: File bugs for failures
        if: failure()
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          GITHUB_REPOSITORY: ${{ github.repository }}
        run: |
          cd backend
          # Use bug filing service if GitHub token available
          if [ -n "$GITHUB_TOKEN" ]; then
            python -m pytest tests/chaos/ -v --tb=short -m chaos --bug-filing || echo "Bug filing completed"
          fi

      - name: Cleanup Toxiproxy
        if: always()
        run: |
          docker stop toxiproxy || true
          docker rm toxiproxy || true

      - name: Notify on failure
        if: failure()
        run: |
          echo "Chaos engineering tests failed!"
          echo "Run: pytest tests/chaos/ -v -m chaos"
          echo "CI: ${{ github.server_url }}/${{ github.repository }}/actions/runs/${{ github.run_id }}"
```

# Chaos Engineering Tests

Controlled failure injection tests for system resilience validation.

## Purpose

Chaos engineering tests validate system resilience to:
- Network latency (slow 3G simulation with Toxiproxy)
- Database connection drops (connection pool exhaustion)
- Memory pressure (heap exhaustion handling)
- Service crashes (LLM provider failures, Redis crashes)

## Safety

**Blast Radius Controls:**
- ✅ Test databases only (never production)
- ✅ Isolated test network (localhost only)
- ✅ 60-second failure duration limit
- ✅ Environment validation (ENVIRONMENT=test required)
- ✅ Database URL validation (no production endpoints)
- ✅ Hostname validation (no production hosts)

## Troubleshooting

### Toxiproxy Not Available

**Error:** `toxiproxy-python not installed`

**Solution:**
```bash
pip install toxiproxy-python
docker run -d --name toxiproxy -p 8474:8474 ghcr.io/shopify/toxiproxy:2.5.0
```

### Tests Skipped Due to Missing Dependencies

**Error:** `psutil not installed` or `memory-profiler not installed`

**Solution:**
```bash
pip install psutil memory-profiler
```

### Blast Radius Check Failed

**Error:** `Unsafe: Database URL appears to be production`

**Solution:**
```bash
export DATABASE_URL=sqlite:///./test_chaos.db
export ENVIRONMENT=test
```

### Test Timeout

**Error:** `Timeout >60s`

**Solution:** Chaos tests have 60s duration limit. If test legitimately needs more time, adjust in pytest.ini or use `@pytest.mark.timeout(120)`.

### Redis Not Available

**Error:** `Redis connection refused`

**Solution:** Redis tests use mock if Redis not available. To use actual Redis:
```bash
docker run -d --name redis -p 6379:6379 redis:alpine
```

## Writing New Chaos Tests

Follow the template in `backend/tests/bug_discovery/TEMPLATES/CHAOS_TEMPLATE.md`.

**Key Requirements:**
1. Use `@pytest.mark.chaos` marker
2. Use `@pytest.mark.timeout(60)` for duration cap
3. Use `ChaosCoordinator` for orchestration
4. Include blast radius checks (`assert_blast_radius()`)
5. Verify graceful degradation (no crash)
6. Verify recovery (return to baseline)
7. Verify data integrity (no data loss/corruption)

**Example:**
```python
@pytest.mark.chaos
@pytest.mark.timeout(60)
def test_new_chaos_scenario(chaos_coordinator, chaos_db_session):
    """Test system resilience to [failure scenario]."""

    # Create test data
    agent = AgentRegistry(id="test-agent", name="test", maturity_level="STUDENT")
    chaos_db_session.add(agent)
    chaos_db_session.commit()

    # Define failure injection
    def inject_failure():
        return failure_context_manager

    # Define graceful degradation check
    def verify_graceful(metrics):
        assert metrics["cpu_percent"] < 100, "System crashed"

    # Run experiment
    results = chaos_coordinator.run_experiment(
        experiment_name="test_new_chaos_scenario",
        failure_injection=inject_failure,
        verify_graceful_degradation=verify_graceful,
        blast_radius_checks=[assert_blast_radius]
    )

    assert results["success"]
```
