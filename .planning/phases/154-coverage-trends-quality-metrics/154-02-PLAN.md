  <name>Task 1: Create assert-to-test ratio tracker script</name>
  <files>backend/tests/scripts/assert_test_ratio_tracker.py</files>
  <action>
Create assert_test_ratio_tracker.py script that counts assert statements per test using AST parsing:

1. Import required modules (ast, sys, pathlib, argparse - all stdlib, no new dependencies)
2. Define AssertCountVisitor class extending ast.NodeVisitor:
   - __init__: Initialize test_functions list and current_function dict
   - visit_FunctionDef: Check if function name starts with "test_"
     - If test function, create current_function dict with name, file, assert_count=0
     - Call generic_visit to traverse function body
     - Append current_function to test_functions list
     - Reset current_function to None
   - visit_Assert: Increment assert_count in current_function if active
3. Define analyze_test_file(file_path) function:
   - Read source code from file
   - Parse with ast.parse(source, filename=str(file_path))
   - Create AssertCountVisitor and visit(tree)
   - Add file path to each test function result
   - Return list of test functions with assert counts
4. Define calculate_assert_ratio(tests) function:
   - Calculate total_asserts = sum(t["assert_count"] for t in tests)
   - Calculate avg_asserts_per_test = total_asserts / len(tests)
   - Identify low_quality_tests = [t for t in tests if t["assert_count"] < 2]
   - Return (avg_asserts_per_test, low_quality_tests)
5. Define find_test_files(test_path) function:
   - Use Path.rglob("test_*.py") to find all test files recursively
   - Return list of Path objects
6. Define main() function with CLI:
   - argparse setup:
     - positional: test_path (Path to tests directory)
     - --min-ratio (default: 2.0, minimum asserts per test)
     - --output (optional JSON output file)
     - --format (text|json, default: text)
   - Find all test files via find_test_files()
   - Analyze each file and collect all tests
   - Calculate ratio via calculate_assert_ratio()
   - Print summary:
     - Total tests, Total asserts, Average asserts per test
     - Low-quality tests count (<min-ratio asserts)
     - List top 10 low-quality tests with file::test_name (assert_count)
   - If low-quality tests found:
     - Print warning: "⚠️ Low-Quality Tests Detected:"
     - List each low-quality test with assert count
     - Exit with code 1 if avg_ratio < min_ratio
   - Otherwise exit 0

Handle parameterized tests (pytest.mark.parametrize):
- Exclude from ratio calculation OR count as N asserts where N = number of test cases
- Add --include-parameterized flag to control behavior

Reference RESEARCH.md section "Pattern 2: Assert-to-Test Ratio Tracking via AST" for implementation details.
