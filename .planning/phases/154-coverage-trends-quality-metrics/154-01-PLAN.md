Create generate_pr_trend_comment.py script that generates PR comment markdown with trend indicators:

1. Import required modules (json, sys, pathlib, argparse)
2. Define calculate_platform_delta(current, previous) function:
   - Calculate delta = current - previous
   - Determine trend indicator: ↑ if delta > 1.0, ↓ if delta < -1.0, → if stable (±1%)
   - Determine severity: 🔴 CRITICAL if delta < -5.0, 🟡 WARNING if delta < -1.0, ✅ OK otherwise
3. Define generate_pr_comment(trending_data) function:
   - Load history from trending_data (require at least 2 entries)
   - For each platform (backend, frontend, mobile, desktop):
     - Extract current and previous coverage from history[-1] and history[-2]
     - Call calculate_platform_delta() to get indicator and severity
     - Build markdown table row with Previous | Current | Delta | Status
   - Add legend explaining indicators (↑↓→) and severity levels
   - Add historical context (baseline, current, target, remaining)
   - Return markdown string
4. Add CLI interface with argparse:
   - --trending-file PATH (required, default: tests/coverage_reports/metrics/cross_platform_trend.json)
   - --output PATH (optional, default: pr_comment.md)
5. In main():
   - Load trending JSON from file
   - Generate PR comment via generate_pr_comment()
   - Write to output file
   - Print to stdout for GitHub Action consumption
   - Exit 0 if successful, 1 if insufficient history
