  <action>
    Create MEASUREMENT_METHODOLOGY.md documenting:

    1. **Standard Coverage Measurement Command:**
       ```bash
       cd backend
       pytest tests/ --cov=core --cov=api --cov=tools \
         --cov-report=json:tests/coverage_reports/metrics/coverage.json \
         --cov-report=html:tests/coverage_reports/html \
         --cov-report=term-missing:skip-covered
       ```

    2. **Scope Definition:**
       - Include: core/, api/, tools/ (production code only)
       - Exclude: tests/, __pycache__, migrations/
       - Rationale: Test coverage should measure production code, not test code itself

    3. **Baseline Tracking:**
       - Each phase creates phase_{NN}_baseline.json
       - Baseline = starting coverage before adding tests
       - Final = ending coverage after all phase tests added

    4. **Gap Calculation:**
       - gap = target - baseline (e.g., 80% - 26.15% = 53.85 pp)
       - improvement = final - baseline
       - remaining = target - final

    5. **Reporting Format:**
       ```json
       {
         "phase": "127",
         "baseline_percentage": 26.15,
         "final_percentage": 26.15,
         "target_percentage": 80.0,
         "gap_percentage_points": 53.85,
         "improvement_percentage_points": 0.0,
         "measurement_scope": "core/, api/, tools/",
         "files_measured": 528
       }
       ```

    Reference pytest.ini [coverage:run] and [coverage:report] sections.
