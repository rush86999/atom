## Open Questions

1. **80% Target Feasibility:** Is 80% achievable given the current codebase?
   - **Risk:** Some complex components may be difficult to test (ReactFlow, heavy third-party components)
   - **Mitigation:** Focus on high-impact, testable code. Accept lower coverage for complex components if needed.

2. **Test Execution Time:** 400-500 new tests may take >4 minutes to run
   - **Recommendation:** Use parallel execution (maxWorkers=4)
   - **Alternative:** Split tests into suites (unit, integration, e2e)

3. **Flaky Async Tests:** Edge case tests with timing issues may be flaky
   - **Recommendation:** Use waitFor, findBy* queries, mock timers
   - **Pattern:** `await waitFor(() => expect(element).toBeVisible(), { timeout: 5000 })`

4. **Accessibility Testing:** jest-axe may have false positives
   - **Recommendation:** Combine automated tests with manual accessibility audit
   - **Tools:** axe DevTools, NVDA screen reader, keyboard-only navigation
