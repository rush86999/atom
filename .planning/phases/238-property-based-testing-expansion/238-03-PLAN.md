    Test 1: `test_segments_are_contiguous_no_gaps`
    - Invariant: Segments cover full episode timeline with no gaps
    - Strategy: st.lists(st.datetimes(), min_size=2, max_size=50) for message timestamps
    - Settings: HYPOTHESIS_SETTINGS_CRITICAL (max_examples=200)
    - Act: Create episode from messages, segment it
    - Assert: segment timestamps cover entire episode range (first segment start = first message, last segment end >= last message)

    Test 2: `test_segments_do_not_overlap`
    - Invariant: No two segments have overlapping time ranges
    - Strategy: st.lists(st.datetimes(), min_size=2, max_size=50) for message timestamps
    - Settings: HYPOTHESIS_SETTINGS_CRITICAL (max_examples=200)
    - Act: Segment episode, check all segment pairs
    - Assert: For any two segments, either segment_a.end <= segment_b.start OR segment_b.end <= segment_a.start

    Test 3: `test_segmentation_on_time_gaps`
    - Invariant: Time gaps > threshold cause segment split
    - Strategy: st.lists(st.datetimes(), min_size=5, max_size=50) with st.just(timedelta(minutes=30)) for gap_threshold
    - Settings: HYPOTHESIS_SETTINGS_STANDARD (max_examples=100)
    - Act: Create messages with gaps, segment
    - Assert: Segments split at gaps > threshold

    Test 4 (optional): `test_segmentation_preserves_message_order`
    - Invariant: Messages within segments maintain original order
    - Strategy: st.lists(st.integers(0, 1000), min_size=5, max_size=100) for message_sequence_ids
    - Settings: HYPOTHESIS_SETTINGS_STANDARD (max_examples=100)
    - Assert: Within each segment, message_sequence_ids are monotonically increasing

    - Invariant: Temporal retrieval returns episodes sorted by timestamp (newest first)
    - Strategy: st.lists(st.datetimes(), min_size=5, max_size=50) for episode timestamps
    - Settings: HYPOTHESIS_SETTINGS_STANDARD (max_examples=100)
    - Act: Create episodes with varying timestamps, temporal retrieve
    - Assert: Results are sorted by timestamp descending

    Test 3: `test_retrieval_results_size_within_limit`
    - Invariant: Retrieval never returns more than requested limit
    - Strategy: st.integers(1, 100) for limit, st.lists(st.uuids(), min_size=10, max_size=200) for episodes
    - Settings: HYPOTHESIS_SETTINGS_IO (max_examples=50, involves DB queries)
    - Assert: len(retrieved_episodes) <= limit

    Test 4 (optional): `test_contextual_retrieval_combines_temporal_semantic`
    - Invariant: Contextual retrieval balances recency and semantic similarity
    - Strategy: st.tuples(st.text(), st.datetimes(), st.floats(0.0, 1.0)) for (query, timestamp, similarity_weight)
    - Settings: HYPOTHESIS_SETTINGS_STANDARD (max_examples=100)

    **test_lifecycle_transitions.py** (4 tests with RuleBasedStateMachine):
    Test 1: `test_episode_lifecycle_is_valid_dag` (state machine test)
    - Invariant: Episode lifecycle follows DAG (ACTIVE → ARCHIVED → DELETED, no cycles)
    - Use hypothesis.stateful.RuleBasedStateMachine
    - States: ACTIVE, ARCHIVED, DELETED
    - Rules: archive() (ACTIVE → ARCHIVED), delete() (ARCHIVED → DELETED), restore() (ARCHIVED → ACTIVE)
    - @invariant(): No cycles exist (can't reach DELETED → ACTIVE)
    - Settings: HYPOTHESIS_SETTINGS_STANDARD (max_examples=100)

    Test 2: `test_lifecycle_transitions_are_valid`
    - Invariant: Only valid transitions allowed per current state
    - Strategy: st.sampled_from(["ACTIVE", "ARCHIVED", "DELETED"]) for current_state
    - Settings: HYPOTHESIS_SETTINGS_STANDARD (max_examples=100)
    - Assert: Valid transitions: ACTIVE→ARCHIVED, ARCHIVED→ACTIVE, ARCHIVED→DELETED; Invalid: ACTIVE→DELETED, DELETED→*

    Test 3: `test_archived_episodes_preserve_data`
    - Invariant: Archiving preserves all episode data (segments, feedback)
    - Strategy: st.dictionaries(st.text(), st.integers()) for episode_metadata
    - Settings: HYPOTHESIS_SETTINGS_IO (max_examples=50, DB operations)

    Test 4: `test_deleted_episodes_are_soft_deleted`
    - Invariant: Deleted episodes are marked deleted, not removed from DB
    - Strategy: st.uuids() for episode_ids
    - Settings: HYPOTHESIS_SETTINGS_IO (max_examples=50)
    - Assert: Deleted episodes exist in DB with deleted_at timestamp

    DOCUMENT INVARIANTS FIRST in each test docstring (PROP-05 requirement).
    Use RuleBasedStateMachine for lifecycle DAG test (PROP-03 state machine requirement).
