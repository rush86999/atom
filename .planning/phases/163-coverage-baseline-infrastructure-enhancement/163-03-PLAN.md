    ## Coverage Measurement Methodology

    ### Principle: Actual Line Coverage, Not Estimates
    Atom uses actual line coverage from coverage.py JSON output. Service-level
    estimates caused false confidence in Phase 160-162 (74.6% estimated vs
    8.50% actual) and are no longer used.

    ### Measurement Command
    ```bash
    pytest tests/ \
      --cov=core \
      --cov=api \
      --cov=tools \
      --cov-branch \
      --cov-report=json \
      --cov-report=term-missing \
      --cov-report=html
    ```

    ### Output Files
    - coverage.json: Machine-readable line + branch coverage (source of truth)
    - html/: Browser-based line-by-line coverage report
    - Terminal: Missing lines shown in output

    ### Key Metrics
    - percent_covered: Line coverage (primary metric)
    - percent_covered_branch: Branch coverage (secondary metric)
    - num_statements: Total executable lines
    - covered_lines: Lines executed by tests

    ### Scope
    Entire backend production code:
    - core/: Services, models, business logic
    - api/: FastAPI routes, endpoints
    - tools/: Browser automation, device capabilities
    - Excludes: tests/, migrations/, __pycache__/

    ### Progressive Targets
    - Phase 1: 70% line coverage, 60% branch coverage
    - Phase 2: 75% line coverage, 65% branch coverage
    - Phase 3: 80% line coverage, 70% branch coverage

    ### Anti-Patterns
    ❌ Aggregating service-level estimates (caused false confidence)
    ❌ Using file count weighted averages
    ❌ Measuring single files and extrapolating
    ❌ Excluding large code sections without justification

    ### Reading Coverage Reports
    1. Check totals.percent_covered for overall line coverage
    2. Check files for per-file breakdown
    3. Use html/ report for line-by-line missing coverage
    4. Compare to backend_163_baseline.json for trend detection

    ### Baseline Reference
    - File: backend/tests/coverage_reports/metrics/backend_163_baseline.json
    - Generated: Phase 163
    - Methodology: pytest --cov-branch (actual execution)
    - Scope: Full backend (core/, api/, tools/)
