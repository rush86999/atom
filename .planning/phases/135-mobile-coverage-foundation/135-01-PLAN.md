- Mock configuration issues (35%): Expo module mocks not resolving
- Root cause: Expo async functions return synchronously from mocks

Fixes:
1. Ensure Expo module mocks return Promises (not sync values)
2. Add proper mock return values for camera, location, notifications
3. Fix mock import order (jest.mock() BEFORE imports)
4. Update jest.setup.js if global mock enhancement needed

For socket.io-client mocks: Ensure `io()` returns proper mock socket with `connected: false` property

Example:
```typescript
// BAD (sync return)
jest.mock('expo-camera', () => ({
  requestCameraPermissionsAsync: jest.fn().mockReturnValue({ status: 'granted' })
}));

// GOOD (async return)
jest.mock('expo-camera', () => ({
  requestCameraPermissionsAsync: jest.fn().mockResolvedValue({ status: 'granted' })
}));
```
