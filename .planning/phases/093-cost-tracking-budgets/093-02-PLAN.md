  Category definitions (standard cost categories):
  ```python
  STANDARD_CATEGORIES = {
      "llm_tokens": "LLM API usage (OpenAI, Anthropic, etc.)",
      "compute": "Cloud compute costs (AWS, GCP, Azure)",
      "storage": "Database and object storage",
      "network": "Bandwidth, CDN, DNS",
      "labor": "Human labor costs (hours * rate)",
      "software": "SaaS subscriptions and licenses",
      "infrastructure": "DevOps, monitoring, logging",
      "support": "Customer support operations",
      "sales": "Sales and marketing expenses",
      "other": "Miscellaneous costs"
  }
  ```

  Custom exceptions:
  ```python
  class CostAttributionError(Exception):
      """Base exception for cost attribution"""
      pass

  class UncategorizedCostError(CostAttributionError):
      """Raised when cost has no category"""
      pass

  class InvalidCategoryError(CostAttributionError):
      """Raised when category is not recognized"""
      pass
  ```

  CostAttributionService class:
  1. `__init__(self, db: Session)`: Initialize with database session

  2. `attribute_cost(self, amount: Decimal, category: str, project_id: str = None, description: str = None) -> Transaction`:
     - Validate category is in STANDARD_CATEGORIES (or allow custom with warning)
     - Raise UncategorizedCostError if category is None or empty
     - Create Transaction record with category
     - If project_id provided, link to Project for budget tracking
     - Return created Transaction

  3. `get_budget_attribution(self, project_id: str) -> Dict[str, Any]`:
     - Query all Transactions for project, grouped by category
     - Calculate: total_spend, by_category (dict), uncategorized (should be 0)
     - Return attribution report

  4. `validate_categorization(self, project_id: str = None) -> Dict[str, Any]`:
     - Check for uncategorized transactions (should be 0)
     - Return validation status and any issues found

  5. `allocate_cost(self, amount: Decimal, category: str, allocations: List[Dict]) -> List[Transaction]`:
     - Split cost across multiple budgets/projects
     - Verify sum of allocations equals original amount
     - Create Transactions for each allocation
     - Return list of created Transactions

  6. `get_category_breakdown(self, start_date: date = None, end_date: date = None) -> Dict[str, Decimal]`:
     - Aggregate spend by category across all projects
     - Optional date range filtering
     - Return category totals sorted by amount

  IMPORTANT: All amount calculations use Decimal (no float). Sum of allocations must exactly equal original amount (validation).
  </action>
