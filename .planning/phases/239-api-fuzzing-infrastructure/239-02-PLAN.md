Create test_auth_api_fuzzing.py with FastAPI TestClient fuzzing:

1. **Imports**:
   - pytest, atheris, sys
   - FastAPI TestClient from fastapi.testclient
   - Session from sqlalchemy.orm
   - main:app (FastAPI app)
   - core.database:get_db
   - tests.e2e_ui.fixtures.auth_fixtures:authenticated_user, test_user
   - tests.e2e_ui.fixtures.database_fixtures:db_session

2. **test_login_endpoint_fuzzing(db_session: Session)**:
   - Marker: @pytest.mark.fuzzing, @pytest.mark.slow, @pytest.mark.timeout(300)
   - Override database: app.dependency_overrides[get_db] = lambda: db_session
   - Create TestClient(app)
   - Define fuzz_one_input(data):
     - Use FuzzedDataProvider: from atheris import fp; fdp = fp.FuzzedDataProvider(data)
     - Generate fuzzed payload: email (100 chars), password (100 chars)
     - Call client.post("/api/auth/mobile/login", json={email, password, device_token, platform})
     - Assert status in [200, 400, 401, 422] (no crashes)
     - Catch ValueError, JSONDecodeError (expected), re-raise Exception (potential bug)
   - Run: atheris.Setup([-atheris.FuzzedDataProviderFlag] + sys.argv, fuzz_one_input); atheris.Fuzz()

3. **test_signup_endpoint_fuzzing(db_session: Session)**:
   - Similar to login, fuzz POST /api/auth/signup
   - Fuzzed fields: email, password, confirm_password, name
   - Status codes: [200, 400, 409, 422]

4. **test_mobile_login_fuzzing(db_session: Session, authenticated_user)**:
   - Use authenticated_user fixture for valid token
   - Fuzz POST /api/auth/mobile/login with device_token, platform variations
   - Fuzzed fields: device_token (100 chars), platform (ios/android/random), device_info (dict)

Use TestClient pattern (NOT httpx - faster, no network overhead).
Set FUZZ_CRASH_DIR environment variable for crash artifacts.
DO NOT fuzz production URLs (use TestClient with database override).
Reuse db_session fixture for database isolation (worker-aware, transaction rollback).
