2. **start_campaign(target_endpoint, test_file, duration_seconds=3600, iterations=10000)**:
   - Generate campaign_id: f"{target_endpoint.replace('/', '-')}_{datetime.utcnow().isoformat()}"
   - Create campaign_crash_dir: self.crash_dir / campaign_id

3. **stop_campaign(campaign_id)**:
   - Get process from self.running_campaigns[campaign_id]
   - Send SIGTERM (graceful shutdown), SIGKILL after 10s timeout
   - Remove from running_campaigns
   - Return: {"campaign_id", "status": "stopped"}

4. **monitor_campaign(campaign_id)**:
   - Read campaign_crash_dir for crash files (*.input, *.log)
   - Count executions (from pytest output or Atheris stats)
   - Count crashes (number of *.input files)
   - Return: {"campaign_id", "status", "executions", "crashes", "duration_seconds"}

Use subprocess.Popen for process management, tempfile for crash artifacts, pathlib.Path for directory operations.

  <name>Task 2: Create CrashDeduplicator for error signature hashing</name>
  <files>backend/tests/fuzzing/campaigns/crash_deduplicator.py</files>
  <action>
Create CrashDeduplicator class with SHA256-based crash deduplication:

1. **deduplicate_crashes(crash_dir)**:
   - Iterate crash_dir.glob("*.input") files
   - For each crash_file: read crash_log_file = crash_file.with_suffix(".log")
   - Extract stack trace using _extract_stack_trace(crash_log)
   - Generate SHA256 hash using _generate_signature_hash(stack_trace)
   - Group crashes: Dict[signature_hash, List[Path]]
   - Return: crashes_by_signature dict

2. **_extract_stack_trace(crash_log)**:
   - Split crash_log by "\n"
   - Find "Traceback" line to start extraction
   - Collect lines until non-indented line (end of traceback)
   - Return: stack_trace string with line numbers

3. **_generate_signature_hash(stack_trace)**:
   - Use hashlib.sha256(stack_trace.encode())
   - Return: signature_hash.hexdigest()

4. **get_unique_crashes(crashes_by_signature)**:
   - Return first crash file from each signature group
   - Use for bug filing (one bug per unique crash)

Import hashlib for SHA256, pathlib.Path for file operations, re for stack trace parsing.
Include unit test examples in docstring showing deduplication behavior.

DO NOT duplicate crash deduplication logic from other parts of codebase (check Phase 237 patterns).

   - For each (signature_hash, crash_files) in crashes_by_signature:
     - Read crash_input (binary) from crash_files[0]
     - Read crash_log from crash_log_file
     - Call bug_filing_service.file_bug() with metadata:
       - test_name: f"fuzzing_{target_endpoint}"
       - error_message: f"Crash in {target_endpoint}: {crash_log[:200]}"
       - metadata: {
           "test_type": "fuzzing",
           "target_endpoint": target_endpoint,
           "crash_input": crash_input.hex()[:1000],
           "crash_log": crash_log[:500],
           "signature_hash": signature_hash,
           "related_crashes": len(crash_files)
         }
     - Collect bug_result: {"status", "issue_url", "issue_number"}
   - Return: filed_bugs list

2. **run_campaign_with_bug_filing(target_endpoint, test_file, duration_seconds=3600)**:
   - Start campaign: start_campaign(target_endpoint, test_file, duration_seconds)

   - Stop campaign: stop_campaign(campaign_id)
   - Deduplicate crashes: CrashDeduplicator.deduplicate_crashes(campaign_crash_dir)
   - File bugs: file_bugs_for_crashes(target_endpoint, crashes_by_signature)
   - Return: {"campaign_id", "executions", "crashes", "bugs_filed"}

Use time.sleep() for campaign duration, subprocess.wait() for process completion.
