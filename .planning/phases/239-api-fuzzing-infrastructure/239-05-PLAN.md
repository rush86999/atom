  <files>backend/tests/fuzzing/scripts/run_fuzzing_campaigns.py</files>
  <action>
Create run_fuzzing_campaigns.py script for campaign orchestration:

1. **run_campaign(target_endpoint, test_file, duration_seconds=3600)**:
   - Import FuzzingOrchestrator from tests.fuzzing.campaigns.fuzzing_orchestrator
   - Initialize orchestrator with GITHUB_TOKEN, GITHUB_REPOSITORY from env
   - Start campaign: orchestrator.start_campaign(target_endpoint, test_file, duration_seconds)
   - Monitor every 60 seconds: orchestrator.monitor_campaign(campaign_id)
   - Stop after duration: orchestrator.stop_campaign(campaign_id)
   - Deduplicate crashes: CrashDeduplicator().deduplicate_crashes(campaign_crash_dir)
   - File bugs: orchestrator.file_bugs_for_crashes(target_endpoint, crashes_by_signature)
   - Return: campaign_summary dict

2. **run_all_campaigns(duration_seconds=3600)**:
   - Define campaign_configs list:
     - {"target": "POST /api/auth/login", "test_file": "tests/fuzzing/test_auth_api_fuzzing.py::test_login_endpoint_fuzzing"}
     - {"target": "POST /api/agents/{id}/run", "test_file": "tests/fuzzing/test_agent_api_fuzzing.py::test_agent_run_fuzzing"}
     - {"target": "POST /api/canvas/present", "test_file": "tests/fuzzing/test_canvas_presentation_fuzzing.py::test_canvas_present_fuzzing"}
     - {"target": "POST /api/workflows", "test_file": "tests/fuzzing/test_workflow_api_fuzzing.py::test_workflow_create_fuzzing"}
     - {"target": "POST /api/skills/import", "test_file": "tests/fuzzing/test_skill_installation_fuzzing.py::test_skill_import_fuzzing"}
   - For each config: run_campaign(config["target"], config["test_file"], duration_seconds)
   - Collect results: total_executions, total_crashes, bugs_filed
   - Return: overall_summary dict

3. **main()**:
   - Parse args: --duration (default 3600), --campaign (optional specific campaign)
   - Get FUZZ_CAMPAIGN_DURATION from env (override if provided)
   - If --campaign specified: run single campaign
   - Else: run_all_campaigns(duration_seconds)
   - Print summary to stdout (JSON format for CI parsing)
   - Exit code: 0 if success, 1 if crashes found, 2 if errors

Use argparse for CLI args, os.getenv for environment variables, json for summary output.
Set FUZZ_ITERATIONS env var based on duration (10000 iterations for 3600s, scale proportionally).
Include error handling for orchestrator failures (log error, continue with next campaign).

2. **Quick Start**:
   - Install dependencies: pip install atheris
   - Run single fuzz test: pytest tests/fuzzing/test_auth_api_fuzzing.py -v -m fuzzing
   - Run all fuzz tests: pytest tests/fuzzing/ -v -m fuzzing
   - Run campaign: python tests/fuzzing/scripts/run_fuzzing_campaigns.py --duration 60

3. **Fuzzing Infrastructure**:
   - FuzzingOrchestrator: Campaign management (start, stop, monitor, file bugs)
   - CrashDeduplicator: SHA256-based crash deduplication
   - BugFilingService: Automated GitHub issue filing
   - Corpus management: Interesting inputs for re-seeding

4. **Fuzzing Test Structure**:
   - Test file pattern: test_*_fuzzing.py
   - Markers: @pytest.mark.fuzzing, @pytest.mark.slow, @pytest.mark.timeout(300)
   - Fixtures: db_session, authenticated_user (reused from e2e_ui)
   - Crash artifacts: FUZZ_CRASH_DIR environment variable

5. **Coverage**:
   - Auth endpoints: login, signup, JWT validation, password reset (FUZZ-03)
   - Agent endpoints: run, status, delete, streaming (FUZZ-04)
   - Workflow/skill endpoints: create, trigger, import, execute (FUZZ-05)

6. **CI/CD Integration**:
   - Weekly pipeline: bug-discovery-weekly.yml
   - Duration: 1 hour per campaign
   - NOT in PR pipeline (FUZZ-07 requirement)
   - Manual trigger: workflow_dispatch

7. **Corpus Management**:
   - What is corpus? Interesting inputs discovered during fuzzing
   - Where: tests/fuzzing/campaigns/corpus/
   - Re-seeding: Use corpus to speed up future campaigns
   - Subdirectories: auth/, agent/, workflow/

8. **Crash Analysis**:
   - Crash artifacts: tests/fuzzing/campaigns/crashes/
   - Deduplication: SHA256 hash of error signature
   - Bug filing: Automated via BugFilingService
   - Reports: tests/fuzzing/scripts/aggregate_crash_reports.py

9. **Best Practices**:
   - Start with 10000 iterations (~5-10 min), scale based on crash discovery
   - Use TestClient pattern (NOT httpx for non-streaming)
   - Reuse fixtures from e2e_ui (db_session, authenticated_user)
   - NEVER fuzz production URLs (use TestClient with database override)

10. **Troubleshooting**:
    - Fuzzing hangs: Reduce iterations, check timeout config
    - No crashes found: Increase iterations, expand input space
    - Duplicate bugs: Check deduplication logic, verify signature hash
    - CI timeout: Reduce campaign duration, run fewer endpoints

Include examples, diagrams, and links to relevant docs (RESEARCH.md, existing fuzz tests).
