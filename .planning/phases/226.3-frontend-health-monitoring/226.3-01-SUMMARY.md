## API Endpoints

### Provider Registry Endpoints

**GET /api/ai/providers/registry**
- Query params: active_only (bool), include_inactive (bool)
- Returns: List of providers with model counts
- Response: { success, providers: [{ provider_id, name, model_count, ... }], count }

**GET /api/ai/providers/registry/{provider_id}**
- Returns: Single provider with all models
- Response: { success, provider: {...}, models: [...], model_count }
- Error: 404 if provider not found

**GET /api/ai/providers/registry/{provider_id}/models**
- Query params: supports_vision (bool), min_quality (int), max_cost (float)
- Returns: Filtered models for provider
- Response: { success, models: [...], count }

**POST /api/ai/providers/registry/sync**
- Returns: Sync operation ID
- Response: { success, message, sync_id }
- Behavior: Triggers background sync from LiteLLM/OpenRouter

**GET /api/ai/providers/registry/sync/status**
- Returns: Current sync status
- Response: { success, syncing, last_sync }

### API Key Endpoints

**POST /api/ai/providers/{provider_id}/keys**
- Request body: { api_key: str, key_name: str }
- Returns: Key storage confirmation
- Response: { success, message, provider_id, key_name, key_preview, key_id }
- Validation: api_key min_length=10, key_name alphanumeric
- Error: 400 for invalid provider, 422 for short keys

## Security Improvements

### Before (Vulnerable)
```typescript
// BAD - API key in URL
const url = `/api/ai/providers/${provider}/keys?api_key=${apiKey}`;
fetch(url);
```

### After (Secure)
```typescript
// GOOD - API key in POST body
fetch(`/api/ai/providers/${provider}/keys`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ api_key: apiKey, key_name: 'default' })
});
```

### Security Benefits
- ✅ API keys not in browser URL bar or history
- ✅ API keys not logged by web servers or proxies
- ✅ API keys not captured by analytics platforms
- ✅ API keys not visible inreferer headers
- ✅ Proper Pydantic validation (min_length, alphanumeric)
