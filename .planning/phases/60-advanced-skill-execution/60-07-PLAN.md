    # Advanced Skill Execution

    **Phase 60** introduces advanced skill execution features that build on Atom's Community Skills infrastructure (Phase 14), Python Package Support (Phase 35), and npm Package Support (Phase 36).

    ## Features Overview

    ### 1. Skill Marketplace

    The Skill Marketplace provides a centralized hub for discovering, rating, and installing community skills.

    - **Discovery**: Search and browse skills by category, type, and popularity
    - **Ratings**: Submit 1-5 star ratings with reviews
    - **Installation**: One-click installation with automatic dependency management
    - **API**: REST endpoints for marketplace operations

    ### 2. Dynamic Skill Loading

    Load and reload skills at runtime without service restart.

    - **Hot-Reload**: Skills update within 1 second of file changes
    - **Module Cache**: Proper sys.modules management prevents stale code
    - **File Monitoring**: Optional watchdog-based file watching
    - **Performance**: Target < 1 second for skill loading

    ### 3. Skill Composition

    Chain multiple skills into complex workflows with DAG validation.

    - **DAG Validation**: Automatic cycle detection
    - **Data Passing**: Output from one skill becomes input to next
    - **Rollback**: Failed workflows automatically roll back
    - **Conditional Execution**: Skip steps based on conditions

    ### 4. Auto-Installation

    Automatic dependency resolution and installation.

    - **Conflict Detection**: Version conflict detection before installation
    - **Batch Installation**: Install multiple skills efficiently
    - **Distributed Locking**: Prevent concurrent build conflicts
    - **Rollback**: Cleanup on installation failure

    ### 5. Supply Chain Security

    Comprehensive protection against real-world supply chain attacks.

    - **Typosquatting Detection**: Low download count warnings
    - **Dependency Confusion**: Internal package protection
    - **Postinstall Malware**: Script analysis blocks cryptojackers
    - **Audit Trail**: All security decisions logged

    ## Quick Start

    ### Browse the Marketplace

    \`\`\`bash
    curl "http://localhost:8000/marketplace/skills?page=1&page_size=20"
    \`\`\`

    ### Install a Skill

    \`\`\`bash
    curl -X POST "http://localhost:8000/marketplace/skills/{skill_id}/install" \\
      -H "Content-Type: application/json" \\
      -d '{"agent_id": "my-agent", "auto_install_deps": true}'
    \`\`\`

    ### Create a Workflow

    \`\`\`python

        SkillStep("fetch", "http_get", {"url": "api.example.com"}, []),

    ### Skill Installation Fails

    **Problem**: Package installation fails with "conflict detected"

    **Solution**:
    1. Check dependency versions in skill metadata
    2. Resolve conflicts manually or contact skill author
    3. Use `auto_install_deps=false` to skip auto-installation

    ### Hot-Reload Not Working

    **Problem**: Skill changes don't appear after hot-reload

    **Solution**:
    1. Verify watchdog is installed: `pip install watchdog`
    2. Check file permissions on skill directory
    3. Ensure `enable_monitoring=True` in SkillDynamicLoader

    ### Workflow Validation Fails

    **Problem**: Workflow marked as "invalid" or "cyclic"

    **Solution**:
    1. Review dependency graph for cycles
    2. Ensure all dependencies reference existing steps
    3. Use `/composition/validate` endpoint for detailed errors

    # Skill Composition Patterns

    Skill composition enables chaining multiple skills into complex workflows with automatic dependency resolution and error handling.

    ## Core Concepts

    ### DAG Workflows

    Workflows are **Directed Acyclic Graphs (DAGs)**:
    - **Directed**: Steps flow in one direction
    - **Acyclic**: No circular dependencies
    - **Graph**: Steps are nodes, dependencies are edges

    ### Step Execution Order

    Steps execute in **topological order**:
    1. Steps with no dependencies execute first
    2. Dependent steps execute after their dependencies
    3. Multiple branches execute in parallel when possible

    ## Basic Patterns

    ### Linear Pipeline

    Sequential steps where each depends on the previous:

    \`\`\`python
    from core.skill_composition_engine import SkillCompositionEngine, SkillStep

    steps = [
        SkillStep("fetch", "http_get", {"url": "api.example.com/data"}, []),
        SkillStep("parse", "json_parse", {}, ["fetch"]),
        SkillStep("save", "database_insert", {"table": "results"}, ["parse"])
    ]
    \`\`\`

    **Flow**: fetch → parse → save

    ### Fan-Out / Fan-In

    One step produces data used by multiple steps:

    \`\`\`python
    steps = [
        SkillStep("fetch", "http_get", {"url": "api.example.com"}, []),
        SkillStep("analyze", "sentiment_analysis", {}, ["fetch"]),
        SkillStep("extract", "entity_extraction", {}, ["fetch"]),
        SkillStep("merge", "combine_results", {}, ["analyze", "extract"])
    ]
    \`\`\`

    **Flow**:
    ```
         fetch
         /    \
    analyze  extract
         \    /
          merge
    ```

    ### Conditional Branching

    Execute steps based on conditions:

    \`\`\`python
    steps = [
        SkillStep("fetch", "http_get", {"url": "api.example.com"}, []),
        SkillStep(
            "notify",
            "send_email",
            {},
            ["fetch"],
            condition="fetch.get('success') == True"
        )
    ]
    \`\`\`

    **Flow**: fetch → (if success) → notify

    ### Error Handling

    Failed workflows automatically roll back:

    \`\`\`python
    # If "process" fails, "fetch" is rolled back
    steps = [
        SkillStep("fetch", "http_get", {}, []),
        SkillStep("process", "transform", {}, ["fetch"]),
        SkillStep("save", "database_insert", {}, ["process"])
    ]
    \`\`\`

    **Rollback**: Executes compensation actions in reverse order

    ## Advanced Patterns

    ### Map-Reduce

    Process data in parallel then aggregate:

    \`\`\`python
    steps = [
        SkillStep("split", "split_data", {"chunk_size": 100}, []),
        SkillStep("process1", "transform_chunk", {}, ["split"]),
        SkillStep("process2", "transform_chunk", {}, ["split"]),
        SkillStep("process3", "transform_chunk", {}, ["split"]),
        SkillStep("aggregate", "combine_chunks", {}, ["process1", "process2", "process3"])
    ]
    \`\`\`

    ### Retry Pattern

    Add retry policy to steps:

    \`\`\`python
    steps = [
        SkillStep(
            "fetch",
            "http_get",
            {"url": "flaky-api.com"},
            [],
            retry_policy={"max_retries": 3, "backoff": "exponential"}
        )
    ]
    \`\`\`

    ### Timeout Handling

    Set timeout for individual steps:

    \`\`\`python
    steps = [
        SkillStep(
            "slow_operation",
            "heavy_computation",
            {},
            [],
            timeout_seconds=300  # 5 minutes
        )
    ]
    \`\`\`

    ## Data Passing

    ### Automatic Output Merging

    Step outputs are automatically merged into dependent step inputs:

    \`\`\`python
    steps = [
        SkillStep("step1", "skill_a", {"x": 1}, []),
        SkillStep("step2", "skill_b", {"y": 2}, ["step1"])
    ]
    \`\`\`

    `step2` receives: `{"x": 1, "y": 2, "step1_output": {...}}`

    ### Explicit Data Passing

    Reference specific step outputs:

    \`\`\`python
    # In step2 inputs, use step1 output
    steps = [
        SkillStep("step1", "get_user", {"user_id": 123}, []),
        SkillStep("step2", "send_email", {
            "to": "{{step1.user_email}}",
            "template": "welcome"
        }, ["step1"])
    ]
    \`\`\`

    ## API Usage

          },
          {
            "step_id": "process",
            "skill_id": "json_parse",
            "inputs": {},
            "dependencies": ["fetch"]
          }
        ]
      }'
    \`\`\`

    Check workflow without executing:

    \`\`\`

    }
    \`\`\`

    \`\`\`

    ## Common Workflows

    ### ETL Pipeline

    Extract, Transform, Load pattern:

    etl_steps = [
        SkillStep("extract", "api_fetch", {"endpoint": "/data"}, []),
        SkillStep("transform", "data_clean", {}, ["extract"]),
        SkillStep("validate", "schema_check", {"schema": "v1"}, ["transform"]),
        SkillStep("load", "database_bulk_insert", {}, ["validate"])
    ]
    \`\`\`

    ### Data Enrichment

    Fetch data, enrich from multiple sources:

    \`\`\`python
    enrichment_steps = [
        SkillStep("base", "fetch_user", {"user_id": 123}, []),
        SkillStep("enrich1", "fetch_profile", {}, ["base"]),
        SkillStep("enrich2", "fetch_activity", {}, ["base"]),
        SkillStep("enrich3", "fetch_preferences", {}, ["base"]),
        SkillStep("merge", "merge_user_data", {}, ["enrich1", "enrich2", "enrich3"])
    ]
    \`\`\`

    ### Multi-Channel Notification

    Notify via multiple channels:

    \`\`\`python
    notification_steps = [
        SkillStep("prepare", "format_message", {"template": "alert"}, []),
        SkillStep("email", "send_email", {}, ["prepare"]),
        SkillStep("slack", "slack_post", {}, ["prepare"]),
        SkillStep("sms", "send_sms", {}, ["prepare"])
    ]
    \`\`\`

    ## Best Practices

    ### Design Principles

    1. **Idempotency**: Steps should produce same output for same input
    2. **Isolation**: Each step should do one thing well
    3. **Fail Fast**: Validate inputs early in workflow
    4. **Compensation**: Design rollback actions for each step

    ### Performance

    1. **Minimize Dependencies**: Reduce workflow depth
    2. **Parallel Branches**: Fan-out for independent operations
    3. **Batch Operations**: Process multiple items together
    4. **Cache Results**: Store expensive computations

    ### Error Handling

    1. **Define Rollback**: Each step needs compensation action
    2. **Retry Intelligently**: Only retry transient failures
    3. **Log Context**: Include step ID in error messages
    4. **Set Timeouts**: Prevent runaway steps

    ## Troubleshooting

    ### Workflow Validation Fails

    **Problem**: "Workflow contains cycles"

    **Solution**:
    1. Review step dependencies
    2. Ensure no step transitively depends on itself
    3. Break cycles by introducing intermediate steps
