    ```python
    import json, sys

    with open("tests/coverage_reports/metrics/coverage.json") as f:
        data = json.load(f)

    errors = []

    # Check meta
    if "meta" not in data:
        errors.append("MISSING: meta")
    if "files" not in data:
        errors.append("MISSING: files")
    if "totals" not in data:
        errors.append("MISSING: totals")

    if "files" in data:
        for fname, fdata in data["files"].items():
            if "executed_lines" not in fdata:
                errors.append(f"MISSING executed_lines in {fname}")
            if "summary" not in fdata:
                errors.append(f"MISSING summary in {fname}")
            else:
                if "percent_covered" not in fdata["summary"]:
                    errors.append(f"MISSING summary.percent_covered in {fname}")
                if "missing_lines" not in fdata:
                    errors.append(f"MISSING missing_lines in {fname}")

    # Check totals has a percentage
    if "totals" in data and "percent_covered" not in data["totals"]:
        errors.append("MISSING totals.percent_covered")

    if errors:
        print(f"VALIDATION FAILED: {len(errors)} errors")
        for e in errors:
            print(f"  - {e}")
        sys.exit(1)
    else:
        print(f"VALIDATION PASSED")
        print(f"Backend coverage: {data['totals']['percent_covered']:.2f}%")
        print(f"Files measured: {len(data['files'])}")
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pytest/jest output -> coverage.json | Coverage measurement tools writing to filesystem |
| coverage.json -> validation script | Coverage data parsed by validation logic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-292-01 | Spoofing | coverage.json generation | accept | Coverage data is derived from test execution, not user input. Falsification would require test modification which is in-scope work. ASVS L1 does not require coverage report integrity controls. |
| T-292-02 | Information Disclosure | coverage.json files | accept | Coverage data contains file paths and line numbers -- no PII, secrets, or credentials. Metadata includes source code line counts but not actual source content. Low sensitivity. |
| T-292-03 | Denial of Service | test execution | accept | Test execution failure would prevent baseline capture. This is detected by non-zero exit code and caught by structural validation. Mitigation: run in CI pipeline with retry; manual re-run is trivial. |
</threat_model>
