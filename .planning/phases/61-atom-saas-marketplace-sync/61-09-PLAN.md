**Sections**:

1. **Overview**:
   - Purpose: Define Atom SaaS platform requirements for local Atom sync
   - Architecture: Local Atom ←→ Atom SaaS Cloud Platform
   - Sync modes: Polling (HTTP), Real-time (WebSocket)

2. **Required HTTP API Endpoints**:

   ```
   GET /api/v1/skills
   Description: Fetch skills from marketplace
   Query Parameters:
     - page: int (default 1)
     - page_size: int (default 100, max 1000)
   Response:
     {
       "skills": [
         {
           "skill_id": "uuid",
           "name": "Skill Name",
           "description": "Description",
           "version": "1.0.0",
           "code": "skill code",
           "command": "python",
           "python_packages": ["package==1.0.0"],
           "npm_packages": [],
           "tags": ["tag1"],
           "metadata": {},
           "category": "automation",
           "created_at": "2026-01-01T00:00:00Z",
           "updated_at": "2026-01-01T00:00:00Z"
         }
       ],
       "page": 1,
       "page_size": 100,
       "total": 1000
     }
   Authentication: Bearer token (ATOM_SAAS_API_TOKEN)
   Rate limit: 100 requests/minute
   ```

   ```
   GET /api/v1/categories
   Description: Fetch skill categories
   Response:
     {
       "categories": [
         {
           "name": "automation",
           "description": "Automation skills",
           "skill_count": 150
         }
       ]
     }
   ```

   ```
   POST /api/v1/ratings
   Description: Submit skill rating
   Body:
     {
       "skill_id": "uuid",
       "rating": 5,
       "comment": "Great skill!",
       "user_id": "user@example.com"
     }
   Response:
     {
       "rating_id": "uuid",
       "created_at": "2026-01-01T00:00:00Z"
     }
   ```

3. **Required WebSocket Endpoint**:

   ```
   WS wss://api.atomsaas.com/ws
   Authentication: Query param ?token=ATOM_SAAS_API_TOKEN
   Message Format (Server → Client):
     {
       "type": "skill_update" | "category_update" | "rating_update" | "skill_delete",
       "data": { ... }
     }
   Heartbeat: Ping every 30 seconds, expect Pong within 10 seconds
   Reconnection: Client handles with exponential backoff
   ```

4. **Authentication**:
   - API Token: Bearer token in Authorization header
   - Token Format: UUID v4 or JWT
   - Token Rotation: Support token refresh without service interruption

5. **Rate Limiting**:
   - API: 100 requests/minute per instance
   - WebSocket: 100 messages/second
   - Retry-After header included on 429 responses

6. **Error Handling**:
   - 401: Invalid/expired token
   - 429: Rate limit exceeded (include Retry-After)
   - 500: Server error (client should retry with backoff)
   - 503: Maintenance mode

7. **Monitoring Requirements**:
   - Health check endpoint: GET /health
   - Metrics endpoint: GET /metrics (Prometheus format)
   - Response time P95: <200ms for skill fetch

8. **Production Deployment Checklist**:
   - [ ] Atom SaaS API deployed and accessible
   - [ ] Generate ATOM_SAAS_API_TOKEN for each Atom instance
   - [ ] Configure ATOM_SAAS_API_URL (production URL)
   - [ ] Configure ATOM_SAAS_WS_URL (WebSocket URL)
   - [ ] Verify API endpoints respond correctly
   - [ ] Test WebSocket connection
   - [ ] Configure firewall/rules for API access
   - [ ] Set up monitoring for Atom SaaS availability

9. **Testing Procedures**:
   - API connectivity test: curl with Bearer token
   - WebSocket connection test: wscat or websocat
   - Sync verification: Check SkillCache/CategoryCache populated
   - Metrics verification: Check sync metrics

10. **Fallback/Mock Options**:
    - If Atom SaaS not available: Local marketplace only mode
    - Mock server option for development: Use wiremock or similar
    - Local marketplace mode: Disable sync, use local skills only

Target: 500+ lines
