## Task 2: Create health check endpoint for sync subsystem
**Objective**: Add health check for Kubernetes probes

**Implementation**:
- Create `backend/core/sync_health_monitor.py` with SyncHealthMonitor class
- Add health check endpoint: GET /health/sync
- Health check logic:
  - Check SyncState: Is last_sync within expected interval? (default 30 minutes)
  - Check WebSocket: Is connected? (if enabled)
  - Check errors: Are there recent errors? (within last 5 minutes)
  - Check scheduler: Is scheduler running?
- Return JSON:
  ```json
  {
    "status": "healthy" | "degraded" | "unhealthy",
    "last_sync": "2026-02-19T10:00:00Z",
    "sync_age_minutes": 5,
    "websocket_connected": true,
    "scheduler_running": true,
    "recent_errors": 0,
    "details": {...}
  }
  ```
- HTTP status codes:
  - 200: All checks passed
  - 503: Any check failed (unhealthy)
  - 200 with degraded status: Some checks failed but not critical

**Acceptance criteria**:

- Returns accurate status
- Checks last sync age
- Checks WebSocket connection
- Checks scheduler status

# Edge cases to handle
- Health check called during sync: Return previous sync status (don't wait)
- Metrics endpoint called before first sync: Return zeros for all metrics
- Admin endpoint called by STUDENT agent: Return 403 Forbidden
- Manual sync triggered while sync in progress: Return 503 Service Unavailable
- WebSocket disabled while connected: Graceful disconnect, update status
- Bulk resolve with invalid conflict IDs: Skip invalid, resolve valid, return partial success

# Performance considerations
- Health check completes within 50ms (cached SyncState)
- Metrics endpoint completes within 100ms (in-memory counters)
- Admin endpoints don't block sync operations
- Audit trail logged asynchronously (non-blocking)
- Pagination uses efficient queries (LIMIT/OFFSET)

# Security considerations
- All admin endpoints require AUTONOMOUS maturity
- Audit trail records user_id for all actions
- Health check endpoint has no authentication (public for Kubernetes)
- Metrics endpoint has no authentication (public for Prometheus)
- Rate limiting on admin endpoints (100 req/min per user)
- Input validation on all parameters (prevent injection)
