# Edge cases to handle
- No pending ratings: Sync should complete gracefully with 0 uploaded
- Network timeout during upload: Retry with exponential backoff
- Duplicate remote ratings: Use remote_rating_id to update instead of create
- User rates same skill multiple times: Keep newest rating based on timestamp
- Atom SaaS API returns 404 (skill deleted): Mark local rating as failed, don't retry
- Concurrent sync requests: Return 503 if sync already in progress
- Rating deleted locally: Don't push to Atom SaaS (skip if not found in DB)

# Performance considerations
- Batch upload reduces API calls (100 ratings per request)
- Parallel uploads with asyncio.gather (max 10 concurrent)
- Efficient query for pending ratings (uses synced_to_saas index)
- Dead letter query includes pagination (100 per page)

# Security considerations
- Admin endpoints require AUTONOMOUS maturity
- User_id validated before uploading to Atom SaaS (prevent impersonation)
- Rating values validated (1-5 range) before upload
- Audit trail for all manual sync triggers
