  <name>Create error-mapping.ts utility module</name>
  <files>frontend-nextjs/lib/error-mapping.ts</files>
  <action>
Create frontend-nextjs/lib/error-mapping.ts with:

1. getUserFriendlyErrorMessage(error: any): string
Maps technical errors to user-friendly messages:
- ENOTFOUND/ECONNREFUSED → "Unable to connect to the server. Please check your internet connection."
- ETIMEDOUT/ECONNABORTED → "Request timed out. Please try again."
- 401 → "Your session has expired. Please log in again."
- 403 → "You do not have permission to perform this action. Please contact your administrator."
- 404 → "The requested resource was not found. Please check the URL or contact support."
- 500 → "Something went wrong on our end. Our team has been notified. Please try again later."
- 503 → "Our service is temporarily unavailable. Please try again in a few moments."
- 504 → "The request took too long to process. Please try again."
- 429 → "Too many requests. Please wait before trying again."
- Default → "An unexpected error occurred. Please try again."

2. getErrorAction(error: any): string | null
Returns suggested user action:
- 401 → "Log in again"
- 5xx → "Retry"
- Network errors → "Check connection"
- 429 → "Wait a moment"
- Others → null (no specific action)

3. getErrorSeverity(error: any): 'info' | 'warning' | 'error'
Maps error to severity level:
- 4xx (except 429) → 'warning'
- 429, 5xx, network → 'error'
- Others → 'info'

4. isRetryableError(error: any): boolean
Returns true if error should trigger automatic retry:
- 5xx errors → true
- Network errors (ECONNABORTED, ETIMEDOUT, ECONNRESET, ENOTFOUND) → true
- 408 Request Timeout → true
- Others → false

Add JSDoc comments for each function with examples
